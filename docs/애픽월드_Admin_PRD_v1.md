## 1. 어드민 개요

### 목적

1인 개발 환경에서 서비스를 효율적으로 운영·관리하기 위한 내부 관리 도구.

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **최소 기능 우선** | Phase 1에서는 신고 처리 + 작품 승인 + 회원 관리만 |
| **자동화 중심** | 반복 작업(신고 감지, 매물 만료 등)은 코드로 자동 처리 |
| **확장 가능** | Phase 2에서 판매 관리, 수익 분석 등으로 확장 |
| **단일 관리자** | 현재 관리자 = 개발자 1인. 다중 관리자는 Phase 3+ |

### 어드민 사용자

| Phase | 관리자 | 설명 |
|-------|--------|------|
| Phase 1~2 | 개발자 본인 (슈퍼 어드민) | 전체 기능 접근 |
| Phase 3+ | 운영 관리자 추가 가능 | 역할 기반 권한(RBAC) |

## 2. 1인 개발 원칙 — 어드민 편

> **핵심: "관리자 UI 개발에 시간을 낭비하지 않는다"**

| 전략 | 설명 |
|------|------|
| 서비스 내 /admin 라우트 | 별도 프로젝트 대신 Next.js App Router에 /admin/* 추가 |
| 최소한의 UI | 기능 중심. 미려한 디자인보다 데이터 확인·조작이 빠르게 가능할 것 |
| Prisma Studio 병행 | 초기에는 Prisma Studio로 직접 DB 조작도 병행 |
| API 재사용 | 서비스 API에 admin guard 추가하여 재사용 |
| 테이블 컴포넌트 | 재사용 가능한 범용 테이블(검색, 정렬, 페이지네이션) 하나로 대부분 커버 |

## 3. 어드민 접근 & 인증

### 인증 방식

| 항목 | Phase 1 | Phase 3+ |
|------|---------|----------|
| 로그인 | 기존 회원 계정 + admin role 체크 | 별도 어드민 로그인 |
| 권한 확인 | DB users 테이블의 role = 'ADMIN' | RBAC (역할 기반 접근 제어) |
| 접근 제한 | NestJS Guard + Next.js middleware + 2FA(TOTP/이메일 OTP) | IP 화이트리스트 + 디바이스 신뢰 정책 |
| 세션 | JWT (서비스와 동일) | 별도 어드민 토큰 |

### 접근 보안

- `/admin/*` 라우트는 `role = 'ADMIN'` 인 경우에만 접근 허용
- Next.js middleware에서 서버사이드 세션 체크 → 미인증 시 리다이렉트
- NestJS API에서 `@AdminGuard()` 데코레이터로 어드민 전용 엔드포인트 보호
- Phase 1부터 어드민 계정 2FA 필수 적용 (TOTP 또는 이메일 OTP)
- 어드민 로그인 실패 횟수 제한(예: 5회) + 잠금 해제 절차 운영
- 어드민 행동 로그 기록 (누가, 언제, 무엇을 했는지)

### Phase 3+ 확장: 역할 기반 접근 제어 (RBAC)

| 역할 | 권한 | 비고 |
|------|------|------|
| SUPER_ADMIN | 전체 기능 | 개발자 |
| MODERATOR | 신고 처리, 매물 관리, 후기 관리 | 운영자 |
| CONTENT_ADMIN | 작품/캐릭터 승인, 공지 관리 | 콘텐츠 관리자 |

> 운영 영향 메모 (2026-03-11): 프론트에서 교환 제안 작성 진입이 EXCH02 바텀시트 중심에서 EXCH04 전용 페이지로 분리되었고, 제안 정책이 `동시 pending 3건 + 재시도 제한(24시간 3회/7일 6회)`으로 강화됨. 어드민은 abuse 모니터링을 위해 제한 초과 로그와 상태 변경 권한 위반 로그(owner/proposer 권한 위배)를 조회할 수 있어야 함.

> 추가 메모 (2026-03-11): 보정 수령은 참여자 1인당 최대 +1만 허용된다. 잔여 미달(`rebalance_failed_partial`) 발생 시 진행자의 후속 처리(`recruit_again/organizer_takeover/cancelled`) 선택 여부를 어드민이 감사해야 한다.

### 제재 처리 워크플로우

```
[신고 접수]
  ↓ 자동: 접수 상태로 등록 + 관리자에게 알림 (FCM/이메일)
[관리자: 검토]
  ↓ 채팅 기록, 매물 정보, 사용자 이력 확인
[처리 결정]
  ├── 경고 부여 → 사용자에게 경고 알림
  ├── 일시 정지 (7일/30일/90일) → 계정 기능 제한 + 사용자 알림
  ├── 영구 정지 → 계정 차단 + 동일 번호 차단 + 사용자 알림
  ├── 매물 삭제/숨김 → 해당 매물 비노출 처리
  ├── 공구 강제 종료 → 참여자 전원에게 알림
  └── 반려 (정상) → 신고자에게 결과 안내
[처리 기록 저장]
  ↓ 누가, 언제, 어떤 조치를 했는지 로그
[신고자에게 처리 결과 알림]
```

### 제재 기준 가이드라인

| 위반 유형 | 1회 | 2회 | 3회 | 4회 |
|-----------|-----|-----|-----|-----|
| 교환 미발송 | 경고 | 7일 정지 | 30일 정지 | 영구 정지 |
| 동행 무단 취소 | 경고 | 30일 정지 | - | - |
| 허위 매물 | 매물 삭제 + 경고 | 7일 정지 | 30일 정지 | - |
| 욕설/비하 | 경고 | 30일 정지 | 90일 정지 | - |
| 사진 도용 | 매물 삭제 + 경고 | 7일 정지 | 30일 정지 | - |
| **공구 먹튀** | **즉시 영구 정지 + 번호 차단** | - | - | - |
| **사기 (고의)** | **즉시 영구 정지 + 번호 차단** | - | - | - |

## 16. DB 스키마 — 어드민 관련

> 서비스 DB에 어드민 관련 테이블/컬럼을 추가하는 방식.

### users 테이블 추가 필드

```prisma
model User {
  // ... 기존 필드
  role          Role      @default(USER)    // USER | ADMIN
  status        UserStatus @default(ACTIVE)  // ACTIVE | WARNING | SUSPENDED | DORMANT | DELETED
  suspendedAt   DateTime?
  suspendedUntil DateTime?
  suspendReason  String?
  warningCount   Int       @default(0)
  sanctions      Sanction[]
}

enum Role {
  USER
  ADMIN
}

enum UserStatus {
  ACTIVE
  WARNING
  SUSPENDED
  DORMANT
  DELETED
}
```

### 신고 (Report) 테이블

```prisma
model Report {
  id            Int          @id @default(autoincrement())
  type          ReportType   // EXCHANGE_NO_SEND | GROUP_BUY_SCAM | COMPANION_NO_SHOW | FAKE_LISTING | HARASSMENT | PHOTO_THEFT | OTHER
  status        ReportStatus @default(PENDING) // PENDING | REVIEWING | RESOLVED | REJECTED
  description   String
  attachments   String[]     // 스크린샷 URL 배열

  reporterId    Int
  reporter      User         @relation("ReportsMade", fields: [reporterId], references: [id])

  targetUserId  Int
  targetUser    User         @relation("ReportsReceived", fields: [targetUserId], references: [id])

  listingId     Int?         // 관련 매물 (nullable)
  chatRoomId    String?      // 관련 채팅방 (nullable)

  resolvedBy    Int?         // 처리한 관리자 ID
  resolvedAt    DateTime?
  resolution    String?      // 처리 결과 설명

  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt
}

enum ReportType {
  EXCHANGE_NO_SEND
  GROUP_BUY_SCAM
  COMPANION_NO_SHOW
  FAKE_LISTING
  HARASSMENT
  PHOTO_THEFT
  OTHER
}

enum ReportStatus {
  PENDING
  REVIEWING
  RESOLVED
  REJECTED
}
```

### 제재 이력 (Sanction) 테이블

```prisma
model Sanction {
  id            Int            @id @default(autoincrement())
  type          SanctionType   // WARNING | SUSPEND_7D | SUSPEND_30D | SUSPEND_90D | PERMANENT_BAN | NICKNAME_CHANGE | UNSUSPEND
  reason        String
  reportId      Int?           // 관련 신고 (nullable)

  userId        Int
  user          User           @relation(fields: [userId], references: [id])

  adminId       Int            // 처리한 관리자
  admin         User           @relation("SanctionsByAdmin", fields: [adminId], references: [id])

  expiresAt     DateTime?      // 정지 만료일 (영구 정지는 null)
  createdAt     DateTime       @default(now())
}

enum SanctionType {
  WARNING
  SUSPEND_7D
  SUSPEND_30D
  SUSPEND_90D
  PERMANENT_BAN
  NICKNAME_CHANGE
  UNSUSPEND
}
```

### 관리자 활동 로그 (AdminLog) 테이블

```prisma
model AdminLog {
  id          Int       @id @default(autoincrement())
  action      String    // "user.suspend" | "listing.delete" | "report.resolve" | "work.approve" 등
  targetType  String    // "user" | "listing" | "report" | "work" | "character"
  targetId    Int
  details     Json?     // 조치 상세 (변경 전/후 데이터)

  adminId     Int
  admin       User      @relation(fields: [adminId], references: [id])

  createdAt   DateTime  @default(now())
}
```

### 공지사항 (Notice) 테이블

```prisma
model Notice {
  id          Int           @id @default(autoincrement())
  title       String
  content     String        // Markdown
  category    NoticeCategory // SERVICE | MAINTENANCE | EVENT
  isPinned    Boolean       @default(false)
  publishedAt DateTime?     // 예약 발행
  authorId    Int
  author      User          @relation(fields: [authorId], references: [id])

  createdAt   DateTime      @default(now())
  updatedAt   DateTime      @updatedAt
}

enum NoticeCategory {
  SERVICE
  MAINTENANCE
  EVENT
}
```
