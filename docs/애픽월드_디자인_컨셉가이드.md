> **"사랑하는 걸, 여기서 Pick!"**
>
> 신뢰감 있는 클린 마켓플레이스 디자인으로 C2C 거래의 안심을 전달하는 디자인 시스템

## 1. 컨셉 방향

### 왜 클린 마켓플레이스인가?

| 이유 | 설명 |
|------|------|
| **신뢰도 확보** | C2C 거래는 플랫폼 신뢰가 핵심. 깔끔하고 정돈된 UI가 "안전한 거래" 인상을 줌 |
| **거래 정보 가독성** | 상품 상태, 가격, 교환 조건 등 핵심 정보가 즉시 눈에 들어와야 함 |
| **팬덤 콘텐츠 강조** | 배경이 심플해야 굿즈 사진·팬아트 배너가 돋보임 |
| **다양한 팬덤 포용** | 애니·만화·소설·아이돌·아티스트·배우·무대·게임 모든 팬덤에 거부감 없는 중립적 디자인 |
| **1인 개발 효율** | 복잡한 커스텀 에셋 없이 컴포넌트 기반으로 빠르게 구현 가능 |

### 레퍼런스

| 레퍼런스 | 참고 포인트 |
|---------|-----------|
| **번개장터** | 카드 레이아웃, 상품 그리드, 신뢰 뱃지 시스템 |
| **크림 (KREAM)** | 미니멀 화이트 기반, 깔끔한 상품 상세 페이지 |
| **포카마켓** | 굿즈 특화 카테고리, 교환 UI |
| **당근마켓** | 지역 기반 거래, 온도/신뢰 시스템 시각화 |
| **Depop** | 젊은 층 타겟 C2C, 감성적이면서도 정돈된 레이아웃 |
| **Mercari** | 일본 C2C, 카테고리 필터링, 상태 태그 |

### 컨셉 키워드

> **Clean Trust × 덕질 마켓플레이스**
>
> 깔끔한 · 신뢰감 · 직관적 · 정돈된 · 따뜻한 · 안심 거래

## 2. 컬러 팔레트

### 디자인 원칙

## 4. 스페이싱 & 레이아웃

### 그리드 시스템

| 항목 | 값 |
|------|------|
| **기본 단위** | 4px (모든 간격은 4의 배수) |
| **페이지 좌우 패딩** | 모바일 16px / 태블릿 24px / 데스크톱 auto (max-width 1200px 중앙) |
| **카드 그리드** | 모바일 2열 / 태블릿 3열 / 데스크톱 4열 |
| **카드 간격** | 12px (모바일) / 16px (데스크톱) |
| **섹션 간격** | 32px ~ 48px |

### 반응형 브레이크포인트

> **모바일 퍼스트, 2 브레이크포인트 전략.**
> 패드(태블릿)는 별도 레이아웃 없이, 세로는 모바일 / 가로는 데스크톱이 자연스럽게 적용.
> 향후 하이브리드 앱(웹뷰) 전환 시 모바일 레이아웃을 그대로 사용.

| 이름 | 범위 | Tailwind | 레이아웃 |
|------|------|----------|---------|
| **Mobile** | 0~767px | 기본 (접두사 없음) | 1컬럼, 풀폭, 하단 네비게이션 |
| **Desktop** | 768px~ | `md:` | 중앙 max-width 1200px, 사이드바/그리드 확장 |

#### 화면별 반응형 전략

| 화면 | Mobile | Desktop |
|------|--------|---------|
| 홈 | 1컬럼 풀폭, 가로 스크롤 섹션 | 중앙 1200px, 그리드 2~3열 |
| 교환/공구 목록 | 2열 그리드 | 3~4열 그리드 |
| 상세 페이지 | 풀폭 | 중앙 카드 형태 (max-width 680px) — 당근마켓 PC 참고 |
| 채팅 | 풀스크린 | 좌측 목록 + 우측 채팅 (2패널) |
| 마이페이지 | 1컬럼 | 좌측 메뉴 + 우측 컨텐츠 |
| 등록 폼 | 풀폭 스텝 폼 | 중앙 카드 (max-width 600px) |

#### 네비게이션 전환

| 요소 | Mobile | Desktop |
|------|--------|---------|
| 메인 네비 | 하단 BottomNav (5탭) | 상단 TopBar + 좌측 사이드 |
| FAB | 우하단 고정 | 동일 (또는 상단 [등록] 버튼) |
| 바텀시트 | 하단에서 슬라이드 업 | 중앙 모달로 전환 |
| 스와이프 | 좌우 스와이프 | 좌우 화살표 버튼 |

#### 하이브리드 앱 대비

- 웹뷰에서는 항상 Mobile 레이아웃 강제 적용 (viewport 375px 기준)
- 앱 전환 시 BottomNav → 네이티브 탭바로 교체 (웹뷰에서는 숨김)
- SafeArea 대응: env(safe-area-inset-*) CSS 변수 사전 적용
- 딥링크 대비: 모든 페이지 URL이 /exchange/[id] 형태로 유지

### Tailwind CSS 반응형 구현

```css
/* Mobile First - 기본 스타일이 모바일 */
.container { @apply px-4 max-w-full; }
.card-grid { @apply grid grid-cols-2 gap-3; }

/* Desktop (md: 768px~) */
@screen md {
  .container { @apply px-6 max-w-[1200px] mx-auto; }
  .card-grid { @apply grid-cols-3 lg:grid-cols-4 gap-4; }
  .detail-page { @apply max-w-[640px] mx-auto; }
}
```

### 네비게이션 반응형

**Mobile (기본)**
- Bottom Navigation: 5개 탭 (홈, 교환, 이벤트, 채팅, 마이페이지)
- 화면 하단에 고정
- 각 탭: 아이콘 + 레이블

**Desktop (768px 이상)**
- Top Navigation Bar: 로고 + 메뉴 + 검색 + 알림 + 프로필
- 화면 상단에 고정
- 가로형 레이아웃으로 확장

**네비게이션 전환**
- 768px 이상에서 Bottom Navigation 숨김 (`hidden md:flex`)
- 768px 미만에서 Top Navigation 숨김 (`flex md:hidden`)
- 부드러운 전환으로 UX 일관성 유지

## 5. UI 컴포넌트

### 5.1 상품 카드

가장 핵심적인 컴포넌트. 깔끔하고 정보 밀도가 적절해야 함.

```
┌──────────────────────┐
│  ┌──────────────────┐ │
│  │                  │ │  ← 상품 이미지 (1:1 비율)
│  │    굿즈 사진      │ │
│  │                  │ │
│  └──────────────────┘ │
│  [교환] [아이돌/아티스트]  │  ← 태그 (거래유형 + 카테고리)
│  하이큐 피규어 히나타   │  ← 상품명 (최대 2줄)
│  작품명 · 캐릭터명     │  ← 메타 정보 (Gray 400)
│  ─────────────────── │
│  [Star] 4.8 · 거래 12회  │  ← 판매자 신뢰도
└──────────────────────┘
```

**스타일 규칙:**
- 카드 배경: White, border: 1px solid Gray 200, border-radius: 12px
- 이미지: object-fit: cover, border-radius: 12px 12px 0 0
- 호버: 그림자(shadow-md) + 살짝 올라감(translateY -2px)
- 태그: 작은 pill 형태, 배경 컬러 + 텍스트 (카테고리별 accent 컬러)

### 5.2 교환 태그 유형

| 유형 | 배경 | 텍스트 | 레이블 |
|------|------|--------|--------|
| **교환** | `#FFF7ED` | `#EA580C` | 교환 |
| **공구** | `#EFF6FF` | `#2563EB` | 공구 |
| **동행** | `#F0FDF4` | `#16A34A` | 동행 |
| **세트공구** | `#FAF5FF` | `#7C3AED` | 세트공구 |

### 5.3 버튼 시스템

| 유형 | 스타일 | 용도 |
|------|--------|------|
| **Primary** | bg: Blue, text: White, rounded-lg | 핵심 CTA (교환 제안, 공구 참여) |
| **Secondary** | bg: White, border: Blue, text: Blue | 보조 액션 (찜, 공유) |
| **Ghost** | bg: transparent, text: Gray 600 | 텍스트 버튼 (더보기, 취소) |
| **Danger** | bg: Error, text: White | 삭제, 신고 |
| **Disabled** | bg: Gray 100, text: Gray 400 | 비활성 |

**사이즈:**
- **Large**: h-48px, px-24px, text-16px (메인 CTA)
- **Medium**: h-40px, px-16px, text-14px (일반)
- **Small**: h-32px, px-12px, text-12px (카드 내, 태그 옆)

### 5.4 신뢰도 표시

번개장터의 "번개파워"나 당근마켓의 "매너온도"처럼, 애픽 월드만의 신뢰 시스템 시각화.

```
┌─────────────────────────────────┐
│  [User] 닉네임           Lv.3 [Gem]  │
│  [Star x4] 4.2 · 거래 28회            │
│  ━━━━━━━━━━━━━━░░░░░  78%          │  ← 신뢰도 게이지 (Blue 그라데이션)
│  [MapPin] 서울 · [Clock] 최근 활동 2시간 전  │
└─────────────────────────────────┘
```

**신뢰 등급:**
| 등급 | 조건 | 아이콘 | 컬러 |
|------|------|--------|------|
| Lv.1 새싹 | 가입 | `Sprout` | Gray 400 |
| Lv.2 일반 | 거래 5회+ | `Star` | Info |
| Lv.3 우수 | 거래 20회+ & 평점 4.0+ | `Gem` | Blue |
| Lv.4 VIP | 거래 50회+ & 평점 4.5+ | `Crown` | Warning (Gold) |

### 아이콘 시스템

| 항목 | 규칙 |
|------|------|
| **라이브러리** | Lucide React (오픈소스, 일관된 스타일) |
| **선 두께** | 1.5px (기본) |
| **크기** | 16px (인라인), 20px (네비게이션), 24px (강조) |
| **컬러** | 단색. 기본 Gray 600, 활성 Blue |

### 상태 아이콘

| 상태 | 아이콘 | 설명 |
|------|--------|------|
| 교환 대기 | `Clock` | 상대방 응답 대기 |
| 교환 성사 | `CheckCircle` | 매칭 완료 |
| 공구 진행 | `Package` | 모집 중 |
| 공구 마감 | `Lock` | 목표 달성/마감 |
| 새 알림 | Red dot badge | 빨간 도트 뱃지 |

### 빈 상태 (Empty State)

화면에 데이터가 없을 때 보여주는 일러스트. 심플한 라인 일러스트 스타일.

```
       ┌───┐
       │ ? │
       └─┬─┘
     ╱   │   ╲
    ╱    │    ╲
   ○     ○     ○

  아직 등록된 교환 상품이 없어요
  [교환 등록하기] 버튼으로 시작해보세요!
```

- 일러스트: 단색 라인 드로잉 (Gray 300)
- 메시지: 친근한 톤 (Body M, Gray 400)
- CTA 버튼: 항상 포함

## 8. 모션 & 인터랙션

### 기본 원칙

- 부드럽고 빠르게: 150ms~300ms
- 자연스러운 easing: `ease-out` 기본
- 과하지 않게: 모든 요소가 움직일 필요 없음

### 트랜지션 가이드

| 요소 | 속성 | 듀레이션 | 이징 |
|------|------|----------|------|
| 버튼 호버 | background, transform | 150ms | ease-out |
| 카드 호버 | shadow, transform | 200ms | ease-out |
| 페이지 전환 | opacity | 200ms | ease-in-out |
| 모달 등장 | opacity, transform(scale) | 250ms | ease-out |
| 바텀시트 | transform(translateY) | 300ms | ease-out |
| 스켈레톤 | opacity(pulse) | 1500ms | ease-in-out |
| 토스트 | transform(translateY), opacity | 300ms | ease-out |

### 마이크로인터랙션

| 인터랙션 | 효과 |
|---------|------|
| **찜 하트** | 클릭 시 하트 아이콘 스케일 바운스 (1 → 1.3 → 1) + 컬러 변경 |
| **교환 매칭** | 두 카드가 중앙으로 슬라이드 → 연결 효과 → 체크 표시 |
| **공구 달성** | 프로그레스 바 fill + 컨페티 효과 (subtle) |
| **풀투리프레시** | 상단에서 로딩 스피너 내려옴 |
| **스와이프 삭제** | 좌측 스와이프로 삭제 옵션 노출 |

## 14. 디자인 원칙 요약

1. **Content First** — 굿즈 사진과 거래 정보가 주인공. UI는 프레임 역할
2. **Trust by Design** — 신뢰 뱃지, 평점, 거래 횟수를 항상 노출하여 안심 거래 유도
3. **Less is More** — 과한 장식 지양. 여백과 정돈으로 고급스러움 표현
4. **Warm & Approachable** — 딱딱한 비즈니스가 아닌, 덕질 친구 같은 따뜻함
5. **Consistent** — 컬러, 타이포, 스페이싱, 컴포넌트 규칙을 일관되게 유지
6. **Mobile First** — 모바일에서 가장 잘 동작하도록 설계, 데스크톱은 확장
