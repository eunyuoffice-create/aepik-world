# 애픽 월드 (Aepik World) — 전체 문서 검토 및 보완 사항 v1.0

> **검토일**: 2026-03-06
> **대상 문서**: PRD v5.0, 이용약관 초안, 개인정보처리방침 초안, Admin PRD v1.0, 디자인 컨셉 가이드 v1.0

---

## 1. PRD v5.0 검토 결과

### ✅ 잘 되어 있는 부분

- Phase 1 범위가 교환/공구로 명확하게 한정됨
- 법적 포지션 구분 (Phase별 통신판매중개업 대응) 명확
- 교환 매칭 로직 (1:1, 다수:1, 차액) 상세 기술
- 세트 공구(캐릭터 선점) 플로우 잘 정의됨
- 팬 이벤트 캘린더 + 생일 배너 광고 신규 기능 반영 완료
- 수익 모델이 Phase별로 현실적으로 설계됨

### ⚠️ 보완 필요 사항

| # | 항목 | 현재 상태 | 보완 방향 | 우선도 |
|---|------|----------|----------|--------|
| 1 | **교환 매칭 알고리즘 상세** | 우선순위만 기술 | 매칭 스케줄러 동작 방식 명확화 (실시간 vs 배치, cron 주기) | 중 |
| 2 | **알림 시스템 과부하 방지** | 알림 종류만 나열 | 알림 쓰로틀링/그룹핑 정책 필요 (같은 매물 반복 알림 방지) | 중 |
| 3 | **이미지 가이드라인** | 파이프라인만 기술 | 금지 이미지(성인물, 폭력, 실사 합성 등) 명시 필요 | 높음 |
| 4 | **공구 최소 인원** | "최소 인원 설정 가능"만 기재 | 최소 인원 미달 시 처리 플로우 명확화 (자동 취소 vs 진행자 선택) | 중 |
| 5 | **세트 공구 랜덤 굿즈 대응** | 미언급 | 트레이딩 카드 등 랜덤 굿즈는 "원하는 캐릭터가 나오지 않을 수 있음" 고지 필요 | 중 |
| 6 | **오프라인 직거래 안전 가이드** | 간략 언급 | 안전 거래 가이드라인 문서화 (공공장소 권장, 야간 비추천 등) | 중 |
| 7 | **Admin PRD와 이벤트/배너 연동** | Admin PRD에 이벤트·배너 관리 섹션 없음 | Admin PRD v1.1에 이벤트 캘린더 관리 + 배너 검수 관리 추가 필요 | 높음 |
| 8 | **에러 핸들링** | 미기술 | 주요 플로우별 에러 케이스 (네트워크 오류, 동시 선점 충돌 등) 정의 필요 | 중 |
| 9 | **데이터 백업 전략** | 미기술 | PostgreSQL 백업 주기, 복구 절차 명시 필요 | 중 |
| 10 | **접근성(A11y)** | 미기술 | 최소 WCAG AA 기준 명시 (디자인 컨셉 가이드에는 있으나 PRD에는 없음) | 낮음 |

## 2. 이용약관 검토 결과

### ✅ 잘 되어 있는 부분

- 서비스 성격 (커뮤니티 플랫폼, 거래 당사자 아님) 명확히 규정
- 교환, 차액 보충, 직거래, 세트 공구 각각 별도 조항
- 팬 이벤트 캘린더, 배너 광고 신규 조항 반영 완료
- 저작권 조항에 Notice & Takedown 절차 포함
- 기록 보관 기간 명시 (채팅 6개월, 거래 3년)

### ⚠️ 보완 필요 사항

| # | 항목 | 보완 방향 | 우선도 |
|---|------|----------|--------|
| 1 | **미성년자 보호 조항 강화** | 만 14세 미만 법정대리인 동의 외에, 미성년자 금전 거래(차액) 시 주의사항 추가 필요 | 중 |
| 2 | **분쟁 해결 절차 구체화** | 현재 "당사자 간 해결 원칙"만 있음 → 단계별 분쟁 해결 절차 (1차 당사자 협의 → 2차 플랫폼 중재 → 3차 외부 기관) 명시 필요 | 높음 |
| 3 | **환불 정책 통합** | 배너 광고, 이벤트 프로모션 각각 환불 규정이 있으나 통합 환불 조항 없음 | 중 |
| 4 | **서비스 이용 제한 공지 방법** | 제재 시 "사전 통지" 방법(이메일, 푸시, 앱 내 알림) 구체화 필요 | 낮음 |
| 5 | **제3자 서비스(Firebase, S3 등) 장애 면책** | 외부 서비스 장애로 인한 서비스 중단 시 면책 조항 추가 필요 | 중 |
| 6 | **개인정보 제3자 제공 근거** | 분쟁 시 상대방 연락처 제공에 대한 동의 절차 명확화 필요 | 높음 |

## 5. 문서 간 정합성 이슈

| # | 이슈 | 관련 문서 | 조치 |
|---|------|----------|------|
| 1 | Admin PRD에 이벤트/배너 관리 섹션 미반영 | PRD v5 ↔ Admin PRD | Admin PRD v1.1 업데이트 필요 |
| 2 | PRD의 상품 사진 "최대 5장" vs 교환글 등록의 "최대 10장" | PRD 13조 vs 8조 | 5장으로 통일 (Phase 1 MVP) |
| 3 | 이용약관 부칙 시행일 미기입 | 이용약관 | 출시일 확정 시 기입 |
| 4 | 개인정보처리방침과 이용약관의 보관 기간 교차 확인 필요 | 이용약관 25조 ↔ 개인정보처리방침 | 정합성 확인 필요 |
| 5 | 디자인 가이드의 네비게이션 "이벤트" 탭이 PRD 목차와 매칭 확인 | 디자인 가이드 ↔ PRD | 정합 확인 완료 (이벤트 캘린더 = 11조) |

## 6. 우선 조치 사항 (액션 아이템)

### 즉시 조치 (Phase 1 출시 전)

1. **PRD**: 상품 사진 수량 통일 (5장 vs 10장 → 5장으로 결정 또는 명확화)
2. **이용약관**: 분쟁 해결 절차 구체화 (단계별)
3. **이용약관**: 개인정보 제3자 제공 동의 절차 명확화
4. **Admin PRD**: 이벤트 캘린더 관리 + 배너 검수 관리 섹션 추가

### Phase 1.5 전 조치

5. **PRD**: 알림 쓰로틀링/그룹핑 정책 정의
6. **PRD**: 이미지 가이드라인 (금지 콘텐츠 목록) 명시
7. **이용약관**: 환불 정책 통합 조항 추가
8. **이용약관**: 제3자 서비스 장애 면책 조항 추가

### Phase 2 전 조치

9. **법률 자문**: 이용약관 전체 + 통신판매중개업 전환 대비
10. **PRD**: 데이터 백업/복구 전략 문서화
11. **이용약관**: 미성년자 금전 거래 주의사항 추가

---

> **결론**: 전체적으로 기획 문서의 완성도가 높음. 핵심 비즈니스 로직, 법적 대응, 기술 스택이 잘 정리되어 있으며, 1인 개발에 맞는 현실적 범위 설정이 잘 됨. 상기 보완 사항은 대부분 "좋은 것을 더 좋게" 수준이며, 당장 블로커가 되는 이슈는 없음. Phase 1 출시 전 즉시 조치 4건만 반영하면 개발 착수에 무리 없음.
