## 2. 회원가입 & 인증 플로우

### 2-0. 로그인 [AUTH01]

```
[앱/웹 진입]
    │
    ▼
[AUTH01-A 로그인 메인]
    │
    ├── [카카오 로그인] → 카카오 OAuth 팝업/리다이렉트
    │       ├── 기존 회원 → JWT 발급 → HOME01
    │       └── 신규 회원 → AUTH02 (회원가입)
    │
    ├── [이메일 로그인] → AUTH01-B 이메일 로그인 모드
    │       ├── 이메일 + 비밀번호 입력
    │       ├── [로그인 버튼] → API 호출
    │       │     ├── 성공 → JWT 발급 → HOME01
    │       │     ├── 실패 → 인라인 에러 "이메일 또는 비밀번호가 일치하지 않습니다"
    │       │     └── 5회 초과 실패 → 30분 계정 잠금
    │       ├── [비밀번호를 잊으셨나요?] → AUTH01-C 비밀번호 찾기
    │       │     ├── 이메일 입력 → 재설정 링크 발송 (유효 1시간)
    │       │     └── 성공 → "메일이 전송되었습니다" 안내
    │       └── [회원가입] → AUTH02
    │
    ├── [로그인 없이 둘러보기] → 비로그인 세션 → HOME01 (검색/열람만 가능)
    │
    └── [약관 안내] → 이용약관/개인정보처리방침 링크 → STAT01
```

**토큰 갱신/세션 만료 처리**
```
[accessToken 만료]
    │
    ▼
[/api/auth/refresh 호출]
    ├── 성공 → access/refresh 재발급 (refresh rotation) → 원래 API 재시도
    └── 실패(만료/재사용 탐지) → 세션 무효화 → AUTH01 재로그인
```

**비로그인 상태 기능 제한 시 → 로그인 유도 바텀시트 [AUTH01-D]**
```
[교환글 등록/공구 참여/채팅/찜/신고 시도]
    │
    ▼
[AUTH01-D 로그인 유도 바텀시트]
    ├── "로그인이 필요한 기능입니다" 안내
    ├── [카카오 로그인] → OAuth 진행
    ├── [이메일 로그인] → AUTH01-B
    └── [나중에] → 바텀시트 닫기, 현재 화면 유지
```

**특수 계정 상태 처리**
```
[로그인 시도]
    ├── 휴면 계정 (1년 미접속) → 휴면 해제 안내 팝업 → 본인인증 후 복구
    ├── 정지 계정 → 정지 안내 + 사유 + 해제일 표시, 기능 차단
    └── OAuth 에러 → 에러 페이지 + "다시 시도" / "이메일로 로그인" 안내
```

### 2-1. 회원가입 [AUTH02]

```
[AUTH01에서 진입]
    ├── 카카오 신규 → Step 1 → (Step 2 건너뜀) → Step 3 → Step 4
    └── "회원가입" 링크 → Step 1 → Step 2 → Step 3 → Step 4
        │
        ▼
[AUTH02-A Step 1: 약관 동의]
    ├── [전체 동의] → 모든 체크박스 일괄 선택/해제
    ├── [필수] 이용약관 동의 + "보기" (STAT01)
    ├── [필수] 개인정보 수집 및 이용 동의 + "보기"
    ├── [선택] 마케팅 정보 수신 동의
    ├── [필수] 만 14세 이상 확인
    └── [다음] → 필수 항목 모두 체크 시 활성화
        │
        ▼
[AUTH02-B Step 2: 계정 정보 입력] (이메일 가입만, 카카오 경유 시 건너뜀)
    ├── 이메일 입력 → 형식 검증 + 자동 중복 확인
    ├── 비밀번호 입력 (8자+, 영문+숫자+특수문자) → 강도 바 (약함/보통/강함)
    ├── 비밀번호 확인 → 일치 여부 실시간 표시
    └── [다음] → 모든 유효성 통과 시 활성화
        │
        ▼
[AUTH02-C Step 3: 프로필 설정]
    ├── 프로필 이미지 (선택, 80px 원형, 미등록 시 기본 아바타)
    ├── 닉네임 입력 (2~12자, 한글/영문/숫자)
    │     ├── debounce 500ms 후 중복 확인
    │     └── 금칙어 필터 (비속어, 관리자 사칭 차단)
    └── [다음] → 닉네임 유효 + 중복확인 통과 시 활성화
        │
        ▼
[AUTH02-D Step 4: 관심 카테고리 선택] (선택사항)
    ├── 애니/만화/소설 (Purple) | 아이돌/아티스트 (Pink)
    ├── 배우/무대 (Amber) | 게임 (Cyan)
    ├── 복수 선택 가능, 미선택도 허용
    ├── [시작하기] → 회원가입 완료
    └── [나중에 설정할게요] → 카테고리 미선택 상태로 완료
        │
        ▼
[AUTH02-E 가입 완료]
    ├── "{닉네임}님, 환영합니다!" 메시지
    ├── [시작하기] → JWT 발급 → HOME01
    └── (24시간 내 웰컴 푸시 알림)
```

### 2-2. 본인인증 (Just-in-time)

```
[거래성 액션 첫 시도]
    ├── 교환/공구/동행 등록
    ├── 교환/공구/동행 신청
    └── 휴면 해제
    │
    ▼
[본인인증 필요 안내 바텀시트]
    │
    ├── [인증하기] → PortOne 본인인증 팝업
    │                    │
    │                    ├── 성공 → [인증 완료 뱃지 부여] → 원래 작업 계속
    │                    └── 실패 → 재시도 안내
    │
    └── [나중에] → 원래 화면으로 복귀 (기능 제한 유지)
```

## 3. 메인 홈 플로우

```
[홈 화면]
    │
    ├── [로그인 여부 확인]
    │       ├── 로그인 + 관심 카테고리 있음 → 관심 카테고리 우선 홈 구성
    │       ├── 로그인 + 관심 카테고리 없음 → 기본 랭킹(인기순) + 카테고리 설정 유도
    │       └── 비로그인 → 개인화 없이 기본 정렬(최신순/인기순)
    │
    ├── [검색바 탭] ──────────────────→ 3-1. 검색 플로우
    │
    ├── [배너 캐러셀 탭] ─────────────→ 배너 상세 (배너 주인공 관련 매물 목록)
    │
    ├── [다가오는 이벤트 섹션]
    │       ├── [이벤트 카드 탭] ────→ 이벤트 상세 페이지
    │       └── [더보기] ────────────→ 이벤트 캘린더 페이지
    │
    ├── [인기 교환 섹션]
    │       ├── [상품 카드 탭] ──────→ 교환 상세 페이지
    │       └── [더보기] ────────────→ 교환 목록 페이지
    │
    ├── [진행 중 공구 섹션]
    │       ├── [공구 카드 탭] ──────→ 공구 상세 페이지
    │       └── [더보기] ────────────→ 공구 목록 페이지
    │
    ├── [동행 구함 섹션]
    │       ├── [동행 탭] ──────────→ 동행글 상세
    │       └── [더보기] ────────────→ 동행글 목록 페이지
    │
    ├── [`Bell` 알림 아이콘] ────────────→ 알림 목록 페이지
    │
    ├── [서비스 소개 · 운영정책] ───────→ STAT01 (/about)
    │
    └── [☰ 메뉴 아이콘] ────────────→ 사이드 메뉴
```

### 3-0. 홈 섹션 기본 정렬 규칙

- 인기 교환: 인기순 (최근 7일 조회수 + 찜 수)
- 진행 중 공구: 마감 임박순
- 동행 모아보기: 최신순
- 비로그인: 위 기본 정렬만 적용 (개인화 없음)

### 3-1. 검색 플로우

```
[검색바 탭] (홈 상단 검색바 / 하단 네비 돋보기)
    │
    ▼
[검색 입력 모드 — SRCH01-A]
    │
    ├── [인기 검색어] → 탭 시 즉시 검색 실행
    │
    ├── [최근 검색어] → 탭 시 재검색 / [X] 개별 삭제 / [전체 삭제]
    │
    └── [텍스트 입력]
            │
            ├── 2글자 이상 → 실시간 자동완성 (작품명 > 캐릭터명 우선, 300ms 디바운스)
            │     └── 자동완성 항목 탭 → 즉시 검색 실행
            │
            └── [검색 버튼 / 엔터]
                    │
                    ▼
                [검색 결과 모드 — SRCH01-B]
                    │
                    ├── [탭 바] 전체 / 교환 / 공구 / 동행 / 이벤트
                    │
                    ├── [필터 칩 바] 작품 ▾ | 캐릭터 ▾ | 필터 (아이콘)
                    │     │
                    │     └── [필터 아이콘 탭]
                    │             │
                    │             ▼
                    │         [필터 바텀시트 — SRCH01-C]
                    │             ├── 카테고리 (4대 칩)
                    │             ├── 굿즈 타입 (다중 선택)
                    │             ├── 상품 상태 (S/A/B/C급)
                    │             ├── 거래 방법 (택배/직거래/둘 다)
                    │             ├── 정렬 (최신/인기/가까운)
                    │             ├── 추가 옵션 (차액 협의 가능만)
                    │             │
                    │             ├── [초기화] → 모든 필터 리셋
                    │             └── [N건 보기] → 필터 적용 + 바텀시트 닫기
                    │
                    ├── [정렬] (최신순 / 인기순 / 가까운순)
                    │
                    ├── [결과 카드 탭] → 해당 상세 페이지 (EXCH02/GRPB02/WANT02(동행글)/EVNT02)
                    │
                    └── [검색 결과 없음] → 빈 상태 + 추천 검색어 제안
```

## 4. 교환 플로우

### 4-1. 교환글 등록

```
[FAB(+) 버튼 또는 등록하기 버튼]
    │
    ▼
[거래 유형 선택] ── 교환 / 공구 / 동행
    │
    │ (교환 선택)
    ▼
[교환글 등록 폼 — Step 1: 보유 굿즈]
    ├── 사진 업로드 (최대 5장)
    ├── 작품 검색/선택
    ├── 캐릭터 선택
    ├── 굿즈 타입 선택
    ├── 상품 상태 (S/A/B/C급)
    ├── 보유 수량
    └── [다음]
        │
        ▼
[교환글 등록 폼 — Step 2: 원하는 굿즈]
    ├── 원하는 작품/캐릭터/굿즈타입
    ├── 원하는 수량
    ├── 차액 협의 가능 여부 (체크박스)
    │     └── (체크 시) 희망 차액 금액 입력
    └── [다음]
        │
        ▼
[교환글 등록 폼 — Step 3: 거래 방식]
    ├── 택배 / 직거래 / 둘 다
    │     └── (직거래 선택 시) 지도에서 위치 선택 + 설명 입력
    ├── 추가 조건 (텍스트, 선택)
    └── [등록하기]
        │
        ▼
[본인인증 체크] ── 미인증 시 → 2-2. 본인인증 플로우
    │
    ▼
[등록 완료]
    ├── 교환글 상세 페이지로 이동
    └── (백그라운드) 매칭 알고리즘 실행 → 매칭 시 양쪽 푸시 알림
```

### 4-2. 교환 매칭 & 진행

```
[매칭 알림 수신] (FCM 푸시)
    │
    ▼
[알림 탭] → 매칭된 교환글 상세 확인
    │
    ▼
[교환 상세 페이지]
    │
    ├── [`MessageCircle` 채팅] → 채팅방 개설/입장
    │       │
    │       ▼
    │   [채팅에서 조건 협의]
    │       ├── 교환 수량 확인
    │       ├── 차액 금액 협의 (있는 경우)
    │       ├── 거래 방식 확인 (택배/직거래)
    │       └── 양쪽 합의
    │           │
    │           ▼
    │       [교환 확정 버튼 클릭] (양쪽 모두)
    │           │
    │           ▼
    │       ── 택배 거래 ──              ── 직거래 ──
    │           │                           │
    │           ▼                           ▼
    │       [차액 정산]               [일정/장소 확정]
    │       (직접 송금)                     │
    │           │                           ▼
    │           ▼                       [만남 & 교환]
    │       [각자 발송]                     │
    │       (운송장 번호 입력)               ▼
    │           │                       [교환 완료 버튼]
    │           ▼                       (양쪽 모두)
    │       [수취 확인]                     │
    │       (양쪽 모두)                     │
    │           │                           │
    │           └──────────┬────────────────┘
    │                      │
    │                      ▼
    │              [교환 완료 처리]
    │                      │
    │                      ▼
    │              [후기 작성 요청 알림]
    │                      │
    │                      ▼
    │              [후기 작성] (7일 이내)
    │                      │
    │                      ├── 긍정 / 부정
    │                      └── 텍스트 후기 (선택)
    │
    ├── [`Repeat` 교환 제안]
    │       │
    │       ▼
    │   [교환 제안 페이지 EXCH04]
    │       ├── 제안 방식 선택
    │       │     ├── 내 교환글 매칭 (`proposal_mode=listing`)
    │       │     └── 빠른 제안 등록 (`proposal_mode=manual`)
    │       ├── 거래 조건 입력
    │       │     ├── 거래 구조: 1:1 / 1:N / N:1
    │       │     ├── 수량: 제공 수량 / 희망 수량
    │       │     ├── 차액: 없음 / +금액 / -금액
    │       │     └── 오늘 직교환(이벤트 근처): 이벤트명/시간대/위치 (당일 한정)
```

```
[7일 미응답 자동 만료]
    │
    ├── 등록자가 제안에 7일간 미응답
    │     │
    │     ▼
    │   [제안 상태 → expired] → 제안자에게 "제안이 만료되었습니다" 알림

          │     │

          │   해당 글의 남은 `pending` 제안 일괄 `expired` 처리
          │     └── 각 제안자에게 "교환글이 마감되었습니다" 알림 (`XCircle`, Blue → EXCH05)
```

```
[상태 변경 요청]
    ├── accept/reject: 등록자(owner)만 가능 + 현재 상태 pending 필수
    ├── cancel: 제안자(proposer)만 가능 + 현재 상태 pending 필수
    ├── 종단 상태(accepted/rejected/cancelled/expired) 재변경 불가
    └── accept 동시 요청 시 최초 1건만 성공(트랜잭션), 나머지는 409 처리

    │           └── 안내: "받은 제안 이력이 있어 수정할 수 없어요. 제안이 한 번이라도 도착한 글은 수정할 수 없어요."
    └── [삭제]
          ├── 받은 제안 이력 0건 → 삭제 가능 (삭제 확인 팝업 후 실행)
          └── 받은 제안 이력 1건 이상(상태 무관) → 삭제 불가
                └── 안내: "받은 제안 이력이 있어 삭제할 수 없어요. 분쟁 방지를 위해 마감으로 종료해 주세요."
```
