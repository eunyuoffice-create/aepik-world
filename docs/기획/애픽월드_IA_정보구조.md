## 1. 사이트맵 (전체 구조)

```

│

## 2. 페이지별 상세 IA

### 2-1. 홈 (/)

```
[Header]
  ├── 로고 (애픽 월드)
  ├── 알림 아이콘 (🔔) + 뱃지
  └── 메뉴 아이콘 (☰)

[검색바]
  └── 플레이스홀더: "상품명, 작품, 캐릭터 검색"

[배너 캐러셀]
  ├── 배너 이미지 (1200×300)
  ├── 배너 제목 + 설명
  └── 인디케이터 (● ○ ○)

[다가오는 이벤트 섹션]
  ├── 섹션 헤더: "📅 다가오는 이벤트" + [더보기]
  └── 가로 스크롤 카드
        ├── 날짜
        ├── 이벤트명
        ├── 장소
        └── 카테고리 태그

[인기 교환 섹션]
  ├── 섹션 헤더: "🔥 인기 교환" + [더보기]
  └── 2열 그리드 카드
        ├── 상품 이미지 (1:1)
        ├── 거래유형 태그 + 카테고리 태그
        ├── 상품명 (2줄)
        ├── 작품 · 캐릭터
        └── ★ 평점 · 거래 횟수

[진행 중 공구 섹션]
  ├── 섹션 헤더: "📦 진행 중 공구" + [더보기]
  └── 가로 스크롤 카드
        ├── 상품 이미지
        ├── 공구유형 태그 (일반/세트)
        ├── 상품명
        ├── 참여현황 (N/M명)
        └── 프로그레스 바 (%)

[구함글 섹션]
  ├── 섹션 헤더: "🔍 구함글 모아보기" + [더보기]
  └── 리스트 카드
        ├── [구함] 태그 + 구함글 제목
        └── 작성자 · 시간

[Bottom Navigation]
  ├── 홈 (🏠)
  ├── 교환 (🔄)
  ├── 이벤트 (📅)
  ├── 채팅 (💬)
  └── 마이 (👤)
```

### 2-2. 교환 목록 (/exchange)

```
[Header]
  ├── 페이지 제목: "교환"
  └── 필터 아이콘

[필터 바] (가로 스크롤 칩)
  ├── 작품 ▾
  ├── 캐릭터 ▾
  ├── 굿즈 타입 ▾
  ├── 상품 상태 ▾
  ├── 거래 방법 ▾
  └── 차액 여부 ▾

[정렬]
  └── 최신순 / 인기순 / 가까운순

[교환글 그리드] (2열)
  └── 상품 카드 (동일 구조)

[FAB] (+) → 교환글 등록
```

### 2-3. 교환 상세 (/exchange/[id])

```
[Header]
  ├── ← 뒤로가기
  └── 공유 / 신고

[이미지 슬라이더]
  ├── 상품 이미지 (스와이프)
  └── 인디케이터 (● ○ ○)

[거래유형 태그] + [카테고리 태그]

[상품 정보]
  ├── 상품명
  ├── 작품 · 캐릭터
  └── 등록일 · 조회수

[상세 정보 박스]
  ├── 상태: S급 (미개봉)
  ├── 희망 교환: OO 캐릭터 OO 굿즈
  ├── 차액: ±O,OOO원 가능
  └── 거래 방식: 택배/직거래

[(직거래 시) 지도]
  └── 카카오맵 — 거래 희망 위치

[판매자 정보 박스]
  ├── 프로필 이미지 + 닉네임
  ├── 신뢰 등급 + 뱃지
  ├── ★ 평점 · 거래 횟수
  └── 최근 활동

[하단 고정 CTA]
  ├── [❤️ 찜]
  ├── [💬 채팅]
  └── [🔄 교환 제안]
```

### 2-4. 교환글 등록 (/exchange/new)

```
[Header]
  ├── ← 뒤로가기
  └── 페이지 제목: "교환글 등록"

[스텝 인디케이터]
  └── ① 보유 굿즈 → ② 원하는 굿즈 → ③ 거래 방식

--- Step 1: 보유 굿즈 ---

[사진 업로드]
  └── 최대 5장, 드래그 정렬

[작품 선택]
  └── 검색 인풋 + 자동완성 + "직접 추가" 버튼

[캐릭터 선택]
  └── 작품 선택 후 드롭다운 활성화

[굿즈 타입 선택]
  └── 포토카드 / 아크릴 / 캔뱃지 / 피규어 / ...

[상품 상태]
  └── S급 / A급 / B급 / C급 라디오

[보유 수량]
  └── 수량 입력 (기본 1)

[다음 버튼]

--- Step 2: 원하는 굿즈 ---

[원하는 작품/캐릭터]
  └── 검색 선택

[원하는 굿즈 타입]
  └── 드롭다운

[원하는 수량]

[차액 협의]
  ├── 체크박스: "차액 협의 가능"
  └── (체크 시) 희망 차액 금액 인풋 (+/- 선택)

[다음 버튼]

--- Step 3: 거래 방식 ---

[거래 방법]
  └── 택배 / 직거래 / 둘 다 가능

[(직거래 선택 시)]
  ├── 지도 — 위치 선택 (카카오맵)
  └── 설명 텍스트 인풋

[추가 조건]
  └── 텍스트 인풋 (선택)

[등록하기 버튼]
```

### 2-5a. 구함글 상세 (/wanted/[id])

```

### 2-5b. 공구 상세 — 세트 공구 (/group-buy/[id])

```
[Header]
  ├── ← 뒤로가기
  └── 공유 / 신고

[가격/진행도 박스]
  ├── 세트 가격: OO,OOO원
  ├── 인당 가격: O,OOO원
  ├── 참여현황: N/M명

  ├── 프로필 + 닉네임 + 신뢰 등급
  └── ★ 평점 · 거래 횟수

[상세 설명]

[하단 고정 CTA]
  └── [📦 공구 참여하기] (캐릭터 선택 필수)
```

```
[Header]
  ├── ← 채팅 목록

[입력 영역]
  ├── 텍스트 인풋

  └── 전송 버튼

### 2-9. 마이페이지 (/my)

```
[프로필 영역]
  ├── 프로필 이미지 (원형)

  ├── 교환글 N
  ├── 공구 N
  └── 구함글 N

[메뉴 리스트]
  ├── 📋 내 교환글

  ├── 🔍 내 구함글

  ├── 🎨 내 배너 신청

### 2-10. 배너 신청 (/banner/apply)

```

[신청 폼]
  ├── 대상 캐릭터/아이돌/아티스트/배우 (검색 선택)

  │     └── [가이드라인 전문 보기] 링크

```

## 6. 데이터 흐름 요약

```
[클라이언트]                     [서버]                      [DB/외부]
    │                              │                            │
    ├─ 상품 카드 렌더링 ────────→ GET /api/listings ────────→ PostgreSQL
    ├─ 교환글 등록 ─────────────→ POST /api/exchange ────────→ PostgreSQL
    │                              │                            │
    │                              ├─ 매칭 알고리즘 실행 ──────→ PostgreSQL
    │                              └─ 매칭 결과 ──────────────→ FCM (푸시)
    │                              │                            │
    ├─ 채팅 메시지 ─────────────→ Firebase Realtime DB ─────→ Firebase
    ├─ 이미지 업로드 ───────────→ POST /api/upload ──────────→ S3 + CloudFront
    ├─ 본인인증 ────────────────→ PortOne API ─────────────→ 통신사
    └─ 지도 ────────────────────→ 카카오맵 API ────────────→ 카카오
```
