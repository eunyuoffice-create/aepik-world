## 화면 개요

> 사용자가 받은 모든 알림을 시간순으로 관리하는 허브 화면. 알림 유형별 필터링, 타입별 탭 전환, 읽음/미읽음 상태 관리, 개별 삭제 및 일괄 읽음 처리를 제공한다. FCM 푸시 알림과 인앱 알림이 자동으로 등록되며, 90일 초과 알림은 자동 삭제된다.

---

## 진입 경로

- 앱 헤더의 Bell(`Bell`) 아이콘 탭
- FCM 푸시 알림 탭 (앱 내 진입)
- 마이페이지(USER01) > 알림 설정에서 "알림 센터로 가기"

---

## 화면 구성 요소

### ① Header

- 좌측: 뒤로가기 화살표 (또는 생략, 메인 화면으로 이동)
- 중앙: 페이지 제목 "알림"
- 우측: 설정 아이콘(`Gear`) — 탭 시 USER03 알림 설정으로 이동

### ② 알림 필터 탭바

5개 탭 (가로 스크롤 칩 형태)

| 탭명 | 아이콘 | 필터 대상 | 비고 |
|------|--------|----------|------|
| 전체 | — | 모든 알림 | 기본 활성 |
| 교환 | `Repeat` | 교환 매칭, 교환 제안, 제안 결과 | |
| 공구 | `Package` | 공구 알림, 목표 달성, 마감 임박 | |
| 이벤트 | `Calendar` | 팬 이벤트, 이벤트 리마인더 | |
| 시스템 | `Info` | 배너 검수 결과, 신고 결과, 공지사항 | |

- 탭 전환 시 해당 필터 알림만 표시 (GET /api/notifications?type=...)
- 현재 탭 밑줄 강조 (Blue #2563EB)

### ③ 모두 읽음 버튼

- 헤더 하단 우측에 텍스트 버튼 "모두 읽음"
- 미읽음 알림이 있을 때만 표시
- 탭 시 모든 알림을 읽음 처리 (PATCH /api/notifications/read-all)
- 성공 시 토스트: "모든 알림을 읽음으로 표시했습니다"

### ④ 알림 카드 리스트

각 알림 아이템 구성:

| 영역 | 스펙 |
|------|------|
| **좌측 (미읽음 표시)** | 미읽음: 파란 점(6px) + 배경 #F0F7FF; 읽음: 배경 #fff |
| **중좌 (아이콘)** | 40px 원형, 알림 유형별 아이콘 + 배경색 (카테고리별) |
| **중앙 (텍스트)** | 제목(14px Bold, 1줄 ellipsis) + 본문(12px Regular, 2줄 ellipsis, #666) |
| **시간 표시** | 12px Gray 400, "방금" / "N분 전" / "N시간 전" / "어제" / "MM.DD" |
| **우측 (썸네일)** | 40px 정사각형, 관련 매물 이미지 (있으면), border-radius: 4px |

- 카드 전체 탭 시: 해당 상세 페이지로 이동 + 읽음 처리 (PATCH /api/notifications/[id]/read)
- 좌측 스와이프 시: 삭제 버튼 표시 → 탭 시 "삭제하시겠습니까?" 확인 후 삭제 (DELETE /api/notifications/[id])

### ⑤ 알림 그룹핑

시간 섹션 디바이더로 구분:

- **오늘** — 당일 자정 이후 알림
- **어제** — 전일
- **이번 주** — 2일~6일 전
- **이전** — 7일 이상 전

각 섹션 헤더: 14px Bold #333, padding 12px 16px

### ⑥ 무한 스크롤

- 화면 하단 도달 시 다음 20개 알림 자동 로드
- 로딩 중: 스켈레톤 UI (알림 카드 2~3개 형태)
- 더 이상 알림 없음: 메시지 표시 "더 이상 알림이 없습니다"

### ⑦ 빈 상태

- 조건: 알림이 전혀 없거나, 선택된 탭 필터에 해당 알림이 없음
- 일러스트: Bell 아이콘 (큼)
- 메시지: "아직 알림이 없어요"
- CTA 없음 (수동 진입만)

### ⑧ 하단 네비게이션 (BottomNav)

5개 탭: 홈(`Home`) | 교환(`Repeat`) | 이벤트(`Calendar`) | 채팅(`MessageCircle`) | 마이(`User`)

## 데이터 필드

### API: GET /api/notifications

**Request (Query Parameters):**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| type | string | X | 필터 유형 (all/exchange/groupbuy/event/system) |
| page | number | X | 페이지 번호 (기본 1) |
| limit | number | X | 페이지당 개수 (기본 20) |
| read_status | string | X | 읽음 상태 (all/unread/read) |

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| items | array | 알림 목록 |
| items[].id | string | 알림 ID |
| items[].type | string | 알림 유형 (exchange_matching, proposal_received 등) |
| items[].title | string | 알림 제목 |
| items[].body | string | 알림 본문 (2줄까지 표시) |
| items[].icon | string | 아이콘 이름 (Lucide) |
| items[].icon_bg_color | string | 배경색 (hex) |
| items[].thumbnail_url | string | 관련 이미지 URL (nullable) |
| items[].related_id | string | 관련 컨텐츠 ID (exchange_id, user_id 등) |
| items[].related_type | string | 관련 컨텐츠 타입 (exchange, groupbuy, event 등) |
| items[].target_url | string | 이동 대상 URL (/exchange/[id] 등). 제안 관련 알림은 `/exchange/my-proposals?highlight=[proposal_id]` 형태로 딥링크 |
| items[].is_read | boolean | 읽음 여부 |
| items[].created_at | string | 생성일시 (ISO 8601) |
| unread_count | number | 미읽음 알림 개수 |
| total | number | 전체 알림 개수 |
| has_next | boolean | 다음 페이지 존재 여부 |

### API: PATCH /api/notifications/read-all

모든 알림을 읽음 처리

**Response:**

```json
{
  "success": true,
  "message": "모든 알림을 읽음으로 표시했습니다"
}
```

### API: PATCH /api/notifications/[id]/read

특정 알림을 읽음 처리

**Response:**

```json
{
  "success": true,
  "is_read": true
}
```

### API: DELETE /api/notifications/[id]

특정 알림 삭제

**Response:**

```json
{
  "success": true,
  "message": "알림이 삭제되었습니다"
}
```
