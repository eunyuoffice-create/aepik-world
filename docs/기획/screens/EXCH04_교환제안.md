## 화면 개요

> 교환 상세(EXCH02)에서 진입하는 전용 교환 제안 작성 화면. `내 교환글 매칭(listing)`과 `새 제안 등록(manual)` 두 방식 중 하나를 선택해 거래 구조/수량/차액/오늘 직교환 조건을 입력하고 제안을 발송한다.

---

## 진입 경로

- 교환 상세(EXCH02) > [교환 제안] 탭
- 교환 신청함(EXCH05) > 거절/만료된 보낸 제안 > [다시 제안] 탭 (선택)
- 알림 > 교환 제안 작성 유도 딥링크 (선택)

---

## 선행 조건

- 로그인 필수
- 본인 글에는 제안 작성 불가
- 동일 대상 글에 동시 pending 최대 3건 제안 제한
- 재시도 제한(rate limit): 동일 대상 글 기준 24시간 3회 / 7일 6회

---

## 화면 구성 요소

### ① Header
- 좌측: ← 뒤로가기
- 중앙: "교환 제안"
- 우측: 닫기(×)

### ② 대상 교환글 요약
- 상대 교환글 썸네일 + 상품명 + 작품/캐릭터
- 희망 교환 조건 핵심 요약 (수량/차액 여부/거래 방식)

### ③ 제안 방식 탭
- 탭 2종
  - 내 교환글 매칭 (`proposal_mode=listing`)
  - 새 제안 등록 (`proposal_mode=manual`)
- 탭 전환 시 하위 폼 영역 교체

### ④ 내 교환글 매칭 영역 (listing)
- 내 교환글 목록 카드 (대표이미지, 상품명, 상태, 작품/캐릭터)
- 단일 선택(라디오)
- 이미 해당 대상에 제안한 매물은 `제안 중` 뱃지 + 비활성
- 내 등록글이 없으면 Empty + `교환글 등록하기` CTA

### ⑤ 새 제안 등록 영역 (manual)
- 사진 업로드 1~3장
- 굿즈명 입력
- 상태 선택 (S/A/B/C)
- 수량 입력
- 임시 제안 데이터로 저장되며 공개 교환글 자동 생성 없음

### ⑥ 거래 구조/수량
- 거래 구조: `1:1` / `1:N` / `N:1`
- 제공 수량 (`offer_quantity`)
- 희망 수량 (`want_quantity`)

### ⑦ 차액 제안
- 없음 / `+금액` / `-금액`
- 금액 입력 (숫자, 원 단위)

### ⑧ 오늘 직교환(이벤트 근처)
- 토글 ON/OFF
- ON 시 입력:
  - 이벤트 선택 또는 이벤트명 직접 입력
  - 가능 시간대
  - 대략 위치/랜드마크
- 당일만 활성

### ⑨ 메시지
- 선택 입력 (최대 100자)
- 예: "오늘 8시 이후 합정역 직교환 가능해요"

### ⑩ 하단 CTA
- [교환 제안 보내기] (primary)
- 필수값 미충족 시 비활성
- 성공 시 EXCH05 반영 + 완료 토스트

---

## 화면 상태

- **기본 상태(매칭 탭)**: 내 교환글 선택 리스트 노출
- **기본 상태(새 제안 탭)**: 사진/상태/수량 입력 폼 노출
- **내 교환글 없음**: Empty + `교환글 등록하기` 버튼
- **제안 한도 초과**: CTA 비활성 + "동일 글에 대기중 제안은 최대 3건까지 가능해요"
- **제안 재시도 제한**: "요청이 너무 많아요. 잠시 후 다시 시도해 주세요(24시간 3회/7일 6회)"
- **오늘 직교환 비활성**: 당일이 아니면 토글 비활성 + 안내 문구
- **제출 중**: 로딩 + 중복 제출 방지
- **제출 완료**: 토스트 후 EXCH05 또는 이전 상세(EXCH02) 복귀

## 데이터 필드

### API: GET /api/exchange/[id]/proposal/options

| 필드 | 타입 | 설명 |
|------|------|------|
| target_exchange | object | 대상 교환글 요약 {id, title, image, work, character} |
| my_exchange_items | array | 내 교환글 목록 {id, title, image, condition, work, character, already_proposed} |
| can_propose | boolean | 제안 가능 여부 |
| reason | string | 제안 불가 사유 (한도초과/본인글/차단 등) |
| available_instant_meetup | boolean | 오늘 직교환 옵션 사용 가능 여부 |

### API: POST /api/exchange/[id]/proposal

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| proposal_mode | string | O | `listing` / `manual` |
| my_exchange_id | string | 조건부 | `listing`일 때 필수 |
| manual_offer.images | string[] | 조건부 | `manual`일 때 필수, 최대 3 |
| manual_offer.title | string | 조건부 | `manual`일 때 필수 |
| manual_offer.condition | string | 조건부 | `manual`일 때 필수 (S/A/B/C) |
| manual_offer.quantity | number | 조건부 | `manual`일 때 필수 |
| trade_structure | string | O | `one_to_one` / `one_to_many` / `many_to_one` |
| offer_quantity | number | O | 제공 수량 |
| want_quantity | number | O | 희망 수량 |
| cash_adjustment | object | X | {enabled, direction, amount} |
| instant_meetup | object | X | {enabled, event_name, time_window, meetup_spot(대략 위치/랜드마크)} |
| message | string | X | 첨부 메시지 (100자) |

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 제안 한도 | 동일 대상 글에 사용자당 동시 pending 최대 3건 |
| 2 | 제안 재시도 제한 | 동일 대상 글 기준 24시간 3회 / 7일 6회 제안 생성 시도 제한 |
| 3 | 본인 글 차단 | 본인 작성 글에는 제안 작성 불가 |
| 4 | 방식 분기 검증 | `listing`은 `my_exchange_id` 필수, `manual`은 `manual_offer` 필수 |
| 5 | 빠른 제안 등록 정책 | `manual` 데이터는 EXCH01 공개글로 자동 생성하지 않음 |
| 6 | 다수거래 허용 | `1:1`/`1:N`/`N:1` + 수량 입력 허용 |
| 7 | 차액 방향 | `+`/`-` 방향과 금액을 제안 단위로 저장 |
| 8 | 오늘 직교환 정책 | 당일만 활성, 이벤트/시간대/대략 위치(역/랜드마크) 입력 필수 |
| 9 | 위치 정밀도 제한 | 오늘 직교환은 정확 주소 입력 금지, 대략 위치만 허용 |
| 10 | 성공 후 반영 | 제안 생성 성공 시 EXCH05 받은/보낸 목록에 즉시 반영 |

## 인터랙션

| # | 트리거 | 동작 | 비고 |
|---|--------|------|------|
| 1 | ← 탭 | EXCH02로 복귀 | 작성 중이면 이탈 확인 |
| 2 | 탭 전환(listing/manual) | 입력 영역 전환 | 각 탭 입력값 유지 |
| 3 | 내 교환글 카드 탭 | 선택 상태 변경 | 단일 선택 |
| 4 | 사진 업로드 탭 | 이미지 선택기 오픈 | 최대 3장 |
| 5 | 거래 구조 칩 탭 | 구조 변경 | 수량 입력값 유지 |
| 6 | 오늘 직교환 토글 ON | 이벤트/시간/대략 위치 입력 영역 확장 | 당일만 가능 |
| 7 | [교환 제안 보내기] 탭 | 검증 → POST 호출 → 성공 토스트 | 중복 탭 방지 |
| 8 | 내 교환글 없음 상태 CTA | EXCH03 등록 화면 이동 | 신규 작성 후 복귀 가능 |
