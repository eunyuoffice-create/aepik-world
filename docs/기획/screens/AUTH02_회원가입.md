## 화면 개요

> 이메일 또는 카카오 OAuth 경유 신규 사용자의 회원가입 화면. 스텝 기반 온보딩으로 닉네임 설정, 약관 동의, 관심 카테고리 선택을 순서대로 진행한다. 본인인증(휴대폰)은 가입 시 강제하지 않고, 첫 교환/공구 시도 시 Just-in-time으로 요청한다.

---

## 진입 경로

- AUTH01 카카오 로그인 성공 (신규 사용자)
- AUTH01 이메일 로그인 모드 > "회원가입" 링크 탭

---

## 화면 구성

### 스텝 프로그레스

| 단계 | 명칭 | 필수 여부 |
|------|------|-----------|
| Step 1 | 약관 동의 | 필수 |
| Step 2 | 계정 정보 입력 | 필수 (이메일 가입만) |
| Step 3 | 프로필 설정 | 필수 |
| Step 4 | 관심 카테고리 | 선택 |

---

### A. Step 1 — 약관 동의

| ID | 요소 | 설명 |
|----|------|------|
| A-1 | 헤더 | 좌측: X 닫기(가입 취소), 중앙: "회원가입" |
| A-2 | 스텝 인디케이터 | 4단계 중 1단계 활성 (COMM02 스텝퍼 사용) |
| A-3 | 안내 문구 | "서비스 이용을 위해 약관에 동의해주세요" |
| A-4 | 전체 동의 체크박스 | "전체 동의" — 아래 모든 항목 일괄 체크/해제 |
| A-5 | 이용약관 동의 | [필수] 이용약관 동의 + "보기" 링크 (STAT01 이동) |
| A-6 | 개인정보처리방침 동의 | [필수] 개인정보 수집 및 이용 동의 + "보기" 링크 |
| A-7 | 마케팅 수신 동의 | [선택] 마케팅 정보 수신 동의 |
| A-8 | 만 14세 이상 확인 | [필수] "만 14세 이상입니다" 체크박스 |
| A-9 | 다음 버튼 | Primary Large 풀폭, 필수 항목 모두 체크 시 활성화 |

---

### B. Step 2 — 계정 정보 입력 (이메일 가입만)

> 카카오 OAuth 경유 시 이 단계는 건너뜁니다 (이메일/비밀번호가 이미 카카오에서 제공).

| ID | 요소 | 설명 |
|----|------|------|
| B-1 | 헤더 | 좌측: <- 뒤로가기, 중앙: "회원가입" |
| B-2 | 스텝 인디케이터 | 4단계 중 2단계 활성 |
| B-3 | 이메일 입력 필드 | placeholder: "이메일 주소 입력", 실시간 형식 검증 |
| B-4 | 이메일 중복 확인 | 입력 완료 시 자동 체크, 중복이면 인라인 에러 |
| B-5 | 비밀번호 입력 필드 | placeholder: "비밀번호 (8자 이상, 영문+숫자+특수문자)", 표시/숨김 토글 |
| B-6 | 비밀번호 강도 표시 | 약함(Red) / 보통(Amber) / 강함(Green) 바 |
| B-7 | 비밀번호 확인 필드 | placeholder: "비밀번호 확인", 일치 여부 실시간 표시 |
| B-8 | 다음 버튼 | Primary Large 풀폭, 모든 유효성 통과 시 활성화 |

---

### C. Step 3 — 프로필 설정

| ID | 요소 | 설명 |
|----|------|------|
| C-1 | 헤더 | 좌측: <- 뒤로가기, 중앙: "프로필 설정" |
| C-2 | 스텝 인디케이터 | 4단계 중 3단계 활성 |
| C-3 | 프로필 이미지 | 80px 원형 기본 아바타 + 카메라 아이콘 오버레이. 탭 시 이미지 선택 |
| C-4 | 닉네임 입력 필드 | placeholder: "닉네임 (2~12자)", 실시간 중복 체크 (debounce 500ms) |
| C-5 | 닉네임 규칙 안내 | "한글, 영문, 숫자 조합 가능. 특수문자 불가" (11px 회색) |
| C-6 | 닉네임 상태 표시 | 사용 가능(Green 체크) / 이미 사용 중(Red X) / 규칙 위반(Red 안내) |
| C-7 | 다음 버튼 | Primary Large 풀폭, 닉네임 유효 + 중복확인 통과 시 활성화 |

---

### D. Step 4 — 관심 카테고리 선택 (선택사항)

| ID | 요소 | 설명 |
|----|------|------|
| D-1 | 헤더 | 좌측: <- 뒤로가기, 중앙: "관심 카테고리" |
| D-2 | 스텝 인디케이터 | 4단계 중 4단계 활성 |
| D-3 | 안내 문구 | "관심 있는 카테고리를 선택하면 맞춤 알림을 받을 수 있어요" |
| D-4 | 카테고리 카드 — 애니/만화/소설 | Purple(#8B5CF6) 보더 카드, 아이콘 + 카테고리명. 탭하면 선택/해제 토글 |
| D-5 | 카테고리 카드 — 아이돌/아티스트 | Pink(#EC4899) 보더 카드 |
| D-6 | 카테고리 카드 — 배우/무대 | Amber(#F59E0B) 보더 카드 |
| D-7 | 카테고리 카드 — 게임 | Cyan(#06B6D4) 보더 카드 |
| D-8 | 완료 버튼 | Primary Large 풀폭, "시작하기". 카테고리 미선택도 가능 |
| D-9 | 건너뛰기 링크 | "나중에 설정할게요" Ghost 텍스트 링크 |

### E. 가입 완료 (확인 화면)

## API 명세 (요약)

| API | Method | Endpoint | 설명 |
|-----|--------|----------|------|
| 이메일 중복 확인 | GET | /api/auth/check-email?email={email} | 이메일 가용 여부 |
| 닉네임 중복 확인 | GET | /api/auth/check-nickname?nickname={nickname} | 닉네임 가용 여부 |
| 이메일 회원가입 | POST | /api/auth/signup | { email, password, nickname, profileImage?, categories? } |
| 카카오 회원가입 완료 | POST | /api/auth/kakao/complete | { kakaoId, nickname, profileImage?, categories? } |
| 프로필 이미지 업로드 | POST | /api/upload/profile | multipart/form-data, max 5MB, 리사이즈 200x200 |

## 비즈니스 로직

| # | 규칙 |
|---|------|
| 1 | 카카오 OAuth 경유 시 Step 2(계정 정보) 건너뜀 — Step 1 → Step 3 → Step 4 |
| 2 | 이메일 형식: RFC 5322 기본 검증 + 허용 도메인 제한 없음 |
| 3 | 비밀번호 정책: 최소 8자, 영문 + 숫자 + 특수문자 각 1개 이상 |
| 4 | 닉네임: 2~12자, 한글/영문/숫자만 허용, 특수문자/공백 불가 |
| 5 | 닉네임 금칙어 필터: 비속어, 관리자 사칭(admin, 운영자 등) 차단 |
| 6 | 프로필 이미지: 선택사항, 미등록 시 기본 아바타 자동 할당 |
| 7 | 이미지 업로드: JPEG/PNG만, 5MB 이하, Sharp로 200x200 리사이즈 후 S3 저장 |
| 8 | 관심 카테고리: 복수 선택 가능, 미선택도 허용, 마이페이지에서 나중에 변경 가능 |
| 9 | 가입 완료 시 JWT 발급 + 홈(HOME01) 이동 |
| 10 | 가입 후 24시간 내 웰컴 푸시 알림: "환영합니다! 교환글을 둘러보세요" |
| 11 | 만 14세 미만 가입 시도 → 법정대리인 동의 필요 안내 (PortOne 본인인증 시점에서 확인) |
| 12 | 동일 카카오 계정으로 재가입 시도 → "이미 가입된 계정입니다" + 로그인 유도 |
