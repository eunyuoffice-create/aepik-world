## 화면 개요

공동구매 진행자(총대)가 모든 참여자를 관리하고 배송 상태를 업데이트하는 관리 화면입니다. 참여자 목록 조회, 입금 확인, 배송 상태 관리 및 공지사항 발송 기능을 제공합니다.

---

## 진입 경로

- 공구 상세 화면 > "관리하기" 버튼 (진행자만 표시)
- 마이페이지 > 내 공구 > 관리

---

## 진입 조건

- 해당 공구의 진행자(총대)만 접근 가능
- 비진행자 접근 시 권한 없음 오류 메시지 표시

---

## 화면 구성 요소

### ① Header
- 뒤로가기 버튼
- "공구 관리" 제목
- 공지 아이콘 (알림 설정)

### ② 공구 요약 카드
- **상품명**: 공구의 정식 명칭
- **태그**: [공구] 또는 [세트공구] 구분
- **참여 현황**: 프로그레스바로 N/M명 표시 (예: 18/30)
- **마감일**: D-N 형식 (예: D-7)
- **상태 뱃지**: 모집중(초록) / 마감(주황) / 구매진행중(파랑) / 배송중(파랑) / 완료(회색)

### ③ 상태 스텝 인디케이터
```
[1. 모집중] → [2. 진행 결정] → [3. 진행 확정] → [4. 배송] → [5. 완료]
                    ↘ [무산]
```
- 현재 진행 단계를 하이라이트 표시
- "무산" 경로는 빨간색으로 분기 표시
- 진행 결정 단계에서 진행자가 "진행 확정" 또는 "무산 처리" 선택

### ③-B 진행 결정 패널 (마감 후 표시)

마감일 도달 또는 조기 마감 시 상단에 노란 배경 알림 패널 표시:

**목표 달성 시:**
```
[알림] 목표 인원이 달성되었습니다! 공구를 진행하시겠습니까?
[진행 확정] (Blue 버튼)   [무산 처리] (Red Outline 버튼)
```

**목표 미달 + 최소 인원 충족 시:**
```
[알림] 목표 미달 (18/30명), 최소 인원(20명) 충족. 진행하시겠습니까?
[진행 확정] (Blue 버튼)   [무산 처리] (Red Outline 버튼)
```

**목표 미달 + 최소 인원 미충족 시:**
```
[경고] 최소 인원 미충족 (12/20명). 진행 시 부족분은 본인이 부담합니다.
[그래도 진행] (Amber 버튼)   [무산 처리] (Red Outline 버튼)
```

**무산 처리 바텀시트:**
- 무산 사유 입력 (필수, 최소 10자)
- 사유 예시 칩: "판매처 품절" / "개인 사정" / "최소 인원 미달" / "기타"
- 환불 안내 체크박스: "입금자에게 환불 안내를 포함합니다" (기본 체크)
- [무산 확정] 버튼 (Red)
- 확인 모달: "무산 처리하면 참여자 전원에게 알림이 발송됩니다. 되돌릴 수 없습니다."

### ③-C 자동 무산 타이머 표시

진행 결정 대기 상태에서 진행자에게 타이머 표시:
```
[경고] 진행 결정까지 N일 남았습니다. 미결정 시 자동 무산됩니다.
D+1: 리마인더 알림
D+3: 참여자에게 "진행자 미응답" 알림 + 경고 배너 빨간색 전환
D+5: 자동 무산 처리
```

### ④ 공지글 영역
- **공지 작성**: Textarea 입력 필드 + 등록 버튼
- **기존 공지 목록**: 최신순 정렬
  - 작성일시 (M/D 형식)
  - 공지 내용 (최대 2줄 미리보기)
  - 더보기 링크

### ⑤ 참여자 목록 테이블

#### 일반 공구
| 항목 | 설명 |
|------|------|
| 닉네임 | 참여자 프로필 이미지 + 닉네임 |
| 참여수량 | 구매 수량 (개수) |
| 입금상태 | 미입금 / 입금확인 (토글) |
| 배송상태 | 대기 / 발송완료 / 수취확인 |
| 액션 | 채팅 바로가기, 상세 정보 보기 |

#### 세트 공구
| 항목 | 설명 |
|------|------|
| 닉네임 | 참여자 프로필 이미지 + 닉네임 |
| 선점 캐릭터 | 세트에서 선택한 캐릭터명 (복수 가능) |
| 입금상태 | 미입금 / 입금확인 (토글) |
| 배송상태 | 대기 / 발송완료 / 수취확인 |
| 액션 | 채팅 바로가기, 상세 정보 보기 |

#### 각 참여자 행 인터랙션
- **입금 확인 토글**: 진행자가 수동으로 입금 상태 체크
- **배송 상태 변경**: 발송전 → 발송완료 전환 (운송장 번호 입력)
- **채팅 바로가기**: CHAT02 화면으로 이동

#### 필터 탭
- 전체 (N명)
- 미입금 (N명)
- 입금완료 (N명)
- 배송완료 (N명)

### ⑥ 일괄 액션 바

상태별로 표시되는 버튼이 달라집니다:

**모집중 상태:**
- **[조기 마감]** (Amber) — 마감일 전에 수동 마감 → 진행 결정 단계로 이동
- **[일괄 발송처리]** (Gray, 비활성)

**진행 결정 대기 상태:**
- **[진행 확정]** (Blue) — 진행 확정 → 참여자에 알림
- **[무산 처리]** (Red Outline) — 무산 바텀시트 열기

**진행 확정 상태:**
- **[일괄 발송처리]** (Blue) — 선택한 참여자들의 배송상태 일괄 변경
- **[공구 취소]** (Red Outline) — 진행 확정 후에도 취소 가능 (사유 필수, 입금자 환불 안내 포함)

**배송중/완료 상태:**
- **[일괄 발송처리]** (미발송 건이 있을 때만 활성)

### ⑦ 배송 정보 입력
- **운송장 번호**: 일괄 입력 또는 개별 입력 선택
- **택배사 선택**: 드롭다운 (CJ대한통운, 우체국, 로젠 등)

#### PATCH /api/groupbuy/[id]/participants/[pid]
**참여자 입금확인 및 배송상태 변경**

Request:
```json
{
  "payment_status": "confirmed",
  "delivery_status": "shipped",
  "tracking_number": "1234567890"
}
```

Response: 업데이트된 참여자 정보

#### POST /api/groupbuy/[id]/notice
**공지사항 작성**

Request:
```json
{
  "content": "3/10 - 구매 진행 중입니다. 배송은 3/15 예정입니다."
}
```

Response:
```json
{
  "id": 2,
  "groupbuy_id": 12345,
  "content": "...",
  "created_at": "2026-03-06"
}
```

#### PATCH /api/groupbuy/[id]/status
**공구 상태 변경**

가능한 상태 전환:
```
recruiting → pending_decision (조기 마감 or 마감일 도달)
pending_decision → confirmed (진행 확정)
pending_decision → cancelled (무산 처리)
confirmed → shipping (배송 시작)
shipping → completed (전원 수취 확인)
confirmed → cancelled (진행 확정 후 취소)
pending_decision → cancelled (자동 무산, D+5 미응답)
```

Request (조기 마감):
```json
{
  "new_status": "pending_decision"
}
```

Request (진행 확정):
```json
{
  "new_status": "confirmed"
}
```

Request (무산 처리):
```json
{
  "new_status": "cancelled",
  "reason": "판매처 품절로 진행이 불가합니다",
  "notify_refund": true
}
```
- `reason`: 무산 사유 (필수, 최소 10자)
- `notify_refund`: 환불 안내 포함 여부 (입금자 있을 경우 true)

Response: 업데이트된 공구 정보 + 알림 발송 결과

## 비즈니스 로직

1. **진행자만 접근**: 공구 생성자(총대)만 이 화면 접근 가능
2. **공구 취소 알림**: 공구 취소 시 참여자 전원에게 푸시 알림 발송
3. **자동 완료**: 전원 수취확인 시 공구 상태 자동 완료 처리
4. **자동 수취 처리**: 배송 완료 후 5일 미확인 시 자동 수취 처리
5. **입금 확인**: 진행자가 수동으로 입금 상태 변경 (자동 연동 불가)
6. **배송 추적**: 운송장 번호 입력 시 택배 조회 가능하도록 연동

## 인터랙션 상세

### 입금 확인 토글
- 클릭 시 입금 상태 즉시 변경
- "입금확인" 상태 시 회색 배경에 흰 체크마크 표시
- "미입금" 상태 시 흰 배경에 빨강 아이콘 표시

### 배송상태 변경
- 드롭다운 또는 모달로 운송장 번호 입력
- 택배사 선택 후 저장
- 저장 시 참여자에게 "배송 알림" 푸시 발송

### 공지 작성
- Textarea 입력 후 "등록" 버튼 클릭
- 입력 제한: 500자
- 등록 후 즉시 목록 상단에 반영

### 공구 마감
- "공구 마감하기" 버튼 클릭
- 확인 모달: "마감하시겠습니까?"
- 마감 후 상태 변경 (모집중 → 마감)

### 공구 취소
- "공구 취소하기" 버튼 클릭
- 확인 모달: "공구를 취소하면 참여자 전원에게 알림이 발송됩니다. 계속하시겠습니까?"
- 취소 사유 입력 (선택)
- 확인 후 참여자 전원 푸시 알림 발송
