# BANR02 배너/팝업 등록

## 메타 정보
| 항목 | 내용 |
|------|------|
| **화면코드** | BANR02 |
| **화면명** | 배너/팝업 등록 |
| **라우트** | /banner/register |
| **이용자** | PC / Mobile |
| **버전** | 0.1 |

| **상태** | 초안 |

## 화면 개요
> BANR01(슬롯 현황)에서 [신청하기] 버튼을 탭하면 진입하는 배너/팝업 등록 폼 화면. 선택한 유형(메인 배너/피드 배너/팝업)에 맞는 통이미지 업로드와 부가 정보를 입력하여 신청한다. 배너/팝업은 모두 통이미지(full image)로만 구성되며, 텍스트는 이미지에 노출되지 않고 img alt 속성 용도의 description으로만 사용된다.

## 진입 경로
- BANR01(배너/팝업 슬롯 현황) > [신청하기] 버튼 (type 파라미터 전달)

## 화면 구성 요소

### (1) Header
- 좌측: `ChevronLeft` 뒤로가기 (BANR01로)
- 중앙: 유형에 따른 타이틀
  - type=main_banner: "메인 배너 등록"
  - type=feed_banner: "피드 배너 등록"
  - type=popup: "팝업 등록"
- 우측: 없음

### (2) 유형 정보 카드 (읽기 전용)
- BANR01에서 전달받은 유형 요약 카드
- 카드 구성:
  - 유형 아이콘 + 유형명
  - 노출 위치: "홈 캐러셀" / "목록 사이" / "접속 시 팝업"
  - 이미지 규격: "1200x300px" / "카드너비x180px" / "320x400px"
  - 잔여 슬롯: "3슬롯 남음" (Green)
  - 가격 안내 (Phase 1.5~): "5,000~10,000원/일"
- 유형 변경 불가 (변경하려면 BANR01로 돌아가야 함)

### (3) 대상 정보
- 작품/아이돌 선택 (필수, 드롭다운): 이벤트 캘린더와 동일한 작품 DB 연동
- 캐릭터/멤버 선택 (필수, 드롭다운): 선택한 작품에 따라 동적 로드
- 이벤트 종류 (필수, 칩 그룹): 생일카페 / 데뷔 기념 / 콘서트 / 팬미팅 / 전시 / 기타
  - 단일 선택

### (4) 게시 기간 선택
- 슬롯 달력: 날짜별 잔여 슬롯 상태 시각화
  - 초록 점: 여유 (>50% 잔여)
  - 노랑 점: 보통 (25~50% 잔여)
  - 빨강 점: 거의 매진 (1~2 슬롯)
  - 회색 "매진": 0슬롯 (비활성, 선택 불가)
- 시작일~종료일 범위 선택 (연속 날짜만 가능)
- 선택된 기간 요약: "3월 8일 ~ 3월 10일 (3일간)"
- 예상 비용 (Phase 1.5~): "예상 비용: 15,000원"
- Phase 1: 무료 안내 표시

### (5) 이미지 업로드
- 통이미지 업로드 영역 (Drag-and-drop 또는 탭)
- 유형별 권장 크기 안내:
  - 메인 배너: 1200x300px 통이미지
  - 피드 배너: 카드너비x180px 통이미지
  - 팝업: 320x400px 통이미지
- 파일 포맷: JPG, PNG, WebP (최대 5MB)
- 업로드 미리보기: 실제 비율로 미리보기 표시
- 저작권 가이드라인 (노란 박스):
  - O 팬아트, 이름 기반 디자인, 2차 창작
  - X 공식 사진, 앨범 이미지, 타인 팬아트 도용

### (6) 부가 정보
- 이미지 설명 (필수, 텍스트 인풋): img alt 속성에 들어갈 설명 텍스트 (최대 100자)
  - 안내: "이 텍스트는 화면에 노출되지 않으며, 접근성(alt)을 위해 사용됩니다"
  - 예시 placeholder: "쿠로코의 농구 쿠로코 생일 축하 카페 in 홍대"
- 외부 링크 (선택, URL 인풋): SNS 등 홍보 링크
  - placeholder: "https://twitter.com/..."
- 관련 이벤트 연결 (선택, 드롭다운): 이벤트 캘린더(EVNT01~03) 연동
  - 내가 등록한 이벤트 목록 표시
  - 미연결 시 배너/팝업 탭해도 상세 이동 없음 (이미지만 풀뷰)

### (7) 비용 안내
- Phase 1: "현재 무료 기간입니다" (초록 박스)
- Phase 1.5~: 유형별 일가격 x 선택 일수 = 총 비용
  - 예) 메인 배너 5,000원/일 x 3일 = 15,000원
  - 결제 수단: 카카오페이, 토스페이, 카드 (PortOne)

### (8) 동의 및 신청
- 저작권 동의 체크박스: "저작권 가이드라인을 확인했으며 동의합니다" (필수)
- [신청하기] Primary 버튼 (full-width)
  - 필수 항목 미입력 시: 비활성 + 미입력 필드 하이라이트
  - 신청 시: 확인 바텀시트 → "신청하시겠습니까?" + 요약 정보 + [신청] / [취소]
- 신청 완료 시: 완료 모달
  - "신청이 완료되었습니다"
  - "관리자 검수 후 게시됩니다 (24시간 이내)"
  - [내 신청 관리] 버튼 → USER02
  - [확인] 버튼 → BANR01로 돌아감

## 데이터 필드

### API: POST /api/banner/register
> 배너/팝업 신청 등록

**Request:**
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| type | string | Y | 유형 (main_banner/feed_banner/popup) |
| franchise_id | string | Y | 작품/아이돌 ID |
| character_id | string | Y | 캐릭터/멤버 ID |
| event_type | string | Y | 이벤트 종류 (birthday_cafe/debut/concert/fanmeeting/exhibition/other) |
| start_date | string | Y | 게시 시작일 (YYYY-MM-DD) |
| end_date | string | Y | 게시 종료일 (YYYY-MM-DD) |
| image_file | file | Y | 통이미지 파일 (JPG/PNG/WebP, max 5MB) |
| description | string | Y | 이미지 alt 텍스트 (max 100자) |
| external_link | string | N | 외부 홍보 링크 URL |
| event_id | string | N | 연결할 이벤트 ID |
| copyright_agreed | boolean | Y | 저작권 동의 여부 |

**Response:**
| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 신청 ID |
| status | string | 신청 상태 (pending) |
| created_at | string | 신청 시각 |
| estimated_review | string | 예상 검수 완료 시각 |

### API: GET /api/banner/slots (기존)
> 날짜별 슬롯 상세 (달력 렌더링용)

**Query:**
| 필드 | 타입 | 설명 |
|------|------|------|
| type | string | 유형 (main_banner/feed_banner/popup) |
| year | number | 연도 |
| month | number | 월 |

**Response:**
| 필드 | 타입 | 설명 |
|------|------|------|
| days | array | 날짜별 슬롯 정보 |
| days[].date | string | 날짜 (YYYY-MM-DD) |
| days[].total | number | 전체 슬롯 |
| days[].used | number | 사용 슬롯 |
| days[].available | number | 잔여 슬롯 |
| days[].is_full | boolean | 매진 여부 |

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 유형 고정 | BANR01에서 전달받은 type 파라미터로 유형 결정, 변경 불가 |
| 2 | 슬롯 실시간 확인 | 달력에서 날짜 선택 시 해당 날짜의 잔여 슬롯 실시간 체크 |
| 3 | 연속 날짜만 허용 | 시작일~종료일 사이 매진된 날짜가 있으면 선택 불가 |
| 4 | 이미지 = 통이미지 | 배너/팝업은 모두 통이미지(full image)로만 구성 |
| 5 | 텍스트 = alt 전용 | description 필드는 img alt 속성에만 사용, 화면에 텍스트로 노출되지 않음 |
| 6 | 이미지 검증 | 업로드 시 해상도, 비율, 파일 크기 서버 측 검증 |
| 7 | 중복 신청 방지 | 동일 유형 + 동일 날짜 범위에 이미 신청이 있으면 불가 |
| 8 | 검수 프로세스 | 신청 → 대기중 → 검수중 → 승인/수정요청/반려 |
| 9 | Phase 1 무료 | Phase 1에서는 결제 없이 바로 신청 |
| 10 | Phase 1.5 유료 | 결제 완료 후 신청 처리, PortOne 결제 연동 |
