## 화면 개요

> 로그인/소셜 로그인 화면. 카카오 OAuth와 이메일 로그인을 제공하며, 비로그인 둘러보기를 지원한다. 회원가입 시에는 AUTH02로 이동한다.

---

## 진입 경로

- 앱 최초 진입 (비로그인 상태)
- 기능 제한 동작 시 로그인 유도 바텀시트에서 [로그인하기] 탭
- 마이페이지 > 로그아웃 후 재접속
- 딥링크를 통한 로그인 필요 페이지 접근

## 화면 구성

### A. 로그인 메인 화면

| ID | 요소 | 설명 |
|----|------|------|
| A-1 | 로고 영역 | 애픽 월드 로고 + 서비스 슬로건 ("내 최애 굿즈, 교환하고 함께 사자") |
| A-2 | 카카오 로그인 버튼 | 카카오 노란색 (#FEE500) Primary CTA, 카카오 아이콘 포함 |
| A-3 | 이메일 로그인 버튼 | Ghost 스타일 Secondary CTA |
| A-4 | 둘러보기 링크 | "로그인 없이 둘러보기" 텍스트 링크, 홈으로 이동 |
| A-5 | 약관 안내 문구 | "로그인 시 이용약관 및 개인정보처리방침에 동의합니다" + 링크 |

### B. 이메일 로그인 모드

| ID | 요소 | 설명 |
|----|------|------|
| B-1 | 뒤로가기 버튼 | 헤더 좌측, A 화면으로 복귀 |
| B-2 | 헤더 타이틀 | "이메일 로그인" |
| B-3 | 이메일 입력 필드 | placeholder: "이메일 주소 입력", 이메일 형식 유효성 검사 |
| B-4 | 비밀번호 입력 필드 | placeholder: "비밀번호 입력", 비밀번호 표시/숨김 토글 |
| B-5 | 로그인 버튼 | Primary 버튼, 이메일+비밀번호 모두 입력 시 활성화 |
| B-6 | 보조 링크 | "비밀번호를 잊으셨나요?" / "회원가입" (AUTH02로 이동) |
| B-7 | 에러 메시지 | 인라인 에러: "이메일 또는 비밀번호가 일치하지 않습니다" |

### C. 비밀번호 찾기 모드

| ID | 요소 | 설명 |
|----|------|------|
| C-1 | 뒤로가기 버튼 | 헤더 좌측, B 화면으로 복귀 |
| C-2 | 헤더 타이틀 | "비밀번호 찾기" |
| C-3 | 안내 문구 | "가입한 이메일 주소를 입력하면 비밀번호 재설정 링크를 보내드립니다" |
| C-4 | 이메일 입력 필드 | placeholder: "이메일 주소 입력" |
| C-5 | 전송 버튼 | Primary 버튼, "재설정 링크 보내기" |
| C-6 | 성공 상태 | "메일이 전송되었습니다. 메일함을 확인해주세요." + 확인 버튼 |

### D. 로그인 유도 바텀시트 (비로그인 사용자)

| ID | 요소 | 설명 |
|----|------|------|
| D-1 | 딤 배경 | 터치 시 닫기 |
| D-2 | 드래그 핸들 | 상단 중앙 바 |
| D-3 | 안내 아이콘 + 문구 | "로그인이 필요한 기능입니다" (잠금 아이콘) |
| D-4 | 설명 문구 | "교환글 등록, 공구 참여, 채팅 등을 이용하려면 로그인해주세요" |
| D-5 | 카카오 로그인 버튼 | 카카오 노란색 Primary CTA |
| D-6 | 이메일 로그인 버튼 | Ghost 스타일 Secondary CTA |
| D-7 | 닫기 버튼 | "나중에" 텍스트 링크 |

## API 명세 (요약)

| API | Method | Endpoint | 설명 |
|-----|--------|----------|------|
| 카카오 OAuth | GET | /api/auth/kakao | 카카오 로그인 리다이렉트 |
| 카카오 콜백 | GET | /api/auth/kakao/callback | OAuth 콜백 처리, JWT 발급 |
| 이메일 로그인 | POST | /api/auth/login | { email, password } → JWT 토큰 |
| 비밀번호 재설정 요청 | POST | /api/auth/forgot-password | { email } → 재설정 링크 메일 발송 |
| 토큰 리프레시 | POST | /api/auth/refresh | refreshToken → 새 accessToken |

## 비즈니스 로직

| # | 규칙 |
|---|------|
| 1 | 카카오 로그인 시 기존 가입자면 JWT 발급 후 홈 이동, 신규면 AUTH02(회원가입)로 이동 |
| 2 | 이메일 로그인 실패 시 5회 초과 → 30분 계정 잠금 + "비밀번호를 잊으셨나요?" 안내 |
| 3 | 비밀번호 재설정 링크 유효기간: 1시간 |
| 4 | JWT accessToken 만료: 30분, refreshToken 만료: 14일 |
| 5 | 비로그인 상태에서 매물 검색/열람, 이벤트 캘린더 열람 가능 |
| 6 | 비로그인 상태에서 교환글 등록/공구 참여/채팅/찜/신고 시도 → 로그인 유도 바텀시트(D) 노출 |
| 7 | 카카오 OAuth 에러 시 → 에러 페이지 + "다시 시도" / "이메일로 로그인" 안내 |
| 8 | 휴면 계정 로그인 시 → 휴면 해제 안내 팝업 + 본인인증 후 복구 |
| 9 | 정지 계정 로그인 시 → 정지 안내 + 사유 + 해제일 표시, 기능 차단 |
| 10 | 둘러보기 선택 시 비로그인 세션 생성, 검색/열람만 가능 |
