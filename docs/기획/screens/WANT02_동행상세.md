# WANT02 동행 상세

## 메타 정보
| 항목 | 내용 |
|------|------|
| **화면코드** | WANT02 |
| **화면명** | 동행 상세 |
| **라우트** | /companion/[id] |
| **이용자** | PC / Mobile |
| **버전** | 0.4 |
| **작성일** | 2026-03-10 |
| **상태** | 초안 |

## 화면 개요
> 개별 동행글의 상세 정보를 확인하고, 동행 신청/채팅 문의 등의 액션을 수행하는 페이지.
> 연결 이벤트 정보, 장소 미니맵, 모집 현황, 작성자 프로필을 종합적으로 제공한다.

## 진입 경로
- 동행 목록(WANT01) > 동행 카드 탭
- 홈(HOME01) > 동행 모아보기 > 카드 탭
- 이벤트 상세(EVNT02) > "동행 찾기" 버튼 > 카드 탭
- 검색 결과(SRCH01) > 동행 탭 > 카드 탭
- 알림(NOTI01) > 동행 관련 알림 탭
- 채팅(CHAT02) > 동행글 링크 탭
- 마이페이지(USER02) > 내 동행글 > 카드 탭

## 화면 구성 요소

### (1) Header
- 좌측: `ArrowLeft` 뒤로가기
- 중앙: "동행 상세"
- 우측: 공유(`Share2`) + 더보기(`MoreVertical`)
  - 더보기 메뉴: 신고하기 / 차단하기 / URL 복사

### (2) 이미지 영역 (조건부)
- 이미지가 있는 경우: 이미지 슬라이더 (최대 3장)
  - 좌우 스와이프, 하단 인디케이터
  - 탭 시 전체 화면 이미지 뷰어
- 이미지 없는 경우: 카테고리 그라데이션 배경 + 아이콘 (높이 200px)

### (3) 상태 뱃지 + 동행 제목
- 상태 뱃지: 모집중(Green #16A34A) / 마감(Gray #6B7280) / 취소(Red #EF4444)
- D-day 뱃지 (동행일 기준, 예: "D-5")
- 동행 제목 (20px, 700)
- 등록일 `dot` 조회수

### (4) 연결 이벤트 카드 (조건부)
- 연결 이벤트가 있는 경우에만 표시
- 카드 형태 (배경 #F9FAFB, 라운딩 12px, 패딩 12px)
  - `Calendar` 아이콘 + 이벤트명 (14px, 600)
  - 이벤트 일정 (12px, #6B7280)
  - 이벤트 유형 태그 (생일카페/전시/콘서트 등)
- 탭 시: EVNT02 이벤트 상세 이동
- 우측: `ChevronRight` 아이콘

### (5) 동행 정보 섹션
- 카테고리 태그 (카테고리 컬러) + 작품명 `dot` 캐릭터명
- **동행 일시**: `Calendar` 아이콘 + 날짜 + 시간대 (예: "2026.03.15(토) 오후")
- **장소**: `MapPin` 아이콘 + 주소 텍스트
- **미니맵**: 카카오맵 임베드 (높이 150px, 라운딩 12px)
  - 장소 핀 마커
  - 탭 시 카카오맵 앱/웹 열기

### (6) 모집 현황
- 제목: "모집 현황" (16px, 600)
- 프로그레스 바 (N/M명, Blue #3B82F6)
- 신청자 목록 (모집 완료 시만 표시, 본인 글인 경우):
  - 아바타 + 닉네임 + 신청 메시지 미리보기
  - 수락/거절 버튼 (모집중일 때만)

### (7) 상세 설명
- 작성자가 입력한 상세 설명 텍스트
- 3줄 초과 시 접기/펼치기 (`ChevronDown`/`ChevronUp`)

### (8) 작성자 정보 박스
- 프로필 이미지 (40px 원형) + 닉네임
- 신뢰 등급 뱃지 (Lv.1~4 + 아이콘)
- 최근 활동: "N시간 전 접속"
- 탭 시: 타인 프로필 이동

### (9) 하단 고정 액션 바 (Sticky Bottom)
- 높이 56px, 흰 배경, `border-top: 1px solid #E5E7EB`
- **타인 글 (모집중)**:
  - 좌측: `MessageCircle` "채팅 문의" 버튼 (Secondary, 40%)
  - 우측: "동행 신청" 버튼 (Primary, 60%)
- **타인 글 (마감/취소)**:
  - 전체: 비활성 상태 "마감된 동행입니다" / "취소된 동행입니다"
- **타인 글 (이미 신청)**:
  - 전체: "신청 완료" (비활성) + "신청 취소" 텍스트 버튼
- **본인 글**:
  - 좌측: "끌어올리기" (`Flame`) 버튼
  - 중앙: "수정" 버튼 (신청 이력 0건일 때만 활성)
  - 우측: "마감하기" 또는 "삭제" 버튼
  - 신청 이력 1건 이상이면 "수정" 비활성 + 잠금 안내 배너 노출
  - 안내 문구: "신청 이력이 있어 핵심 조건을 수정할 수 없어요. 새 글로 다시 모집해 주세요."

### (10) 동행 신청 다이얼로그
- "동행 신청" 버튼 탭 시 센터 팝업 (COMM02 다이얼로그 패턴)
- 제목: "동행 신청"
- 필드: 한줄 메시지 (선택, placeholder: "동행에 대한 한마디를 남겨주세요", max 100자)
- CTA: "신청하기" (Primary) / "취소" (Secondary)
- 신청 완료 시:
  - 토스트: "동행 신청이 완료되었어요!"
  - 작성자에게 푸시 알림
  - 상태만 "신청 완료"로 전환 (채팅방은 수락 시 생성)
  - 하단 바 "신청 완료" 상태로 전환

### (11) 하단 네비게이션 (BottomNav)
- 5개 탭: 홈(`Home`) | 교환(`Repeat`) | 이벤트(`Calendar`) | 채팅(`MessageCircle`) | 마이(`User`)
- 상세 페이지에서는 네비게이션 숨김 (하단 액션 바가 대체)

## 데이터 필드

### API: GET /api/companion/:id (동행 상세)

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 동행글 ID |
| title | string | 동행 제목 |
| event | object/null | 연결 이벤트 {id, name, type, date_range, location, poster_url} |
| work | object | 작품 {id, name} |
| character | object | 캐릭터 {id, name} |
| category | string | 카테고리 |
| companion_date | string | 동행 일시 |
| time_slot | string | 시간대 |
| location | string | 장소 |
| location_coord | object/null | 좌표 {lat, lng} (미니맵용) |
| max_members | number | 모집 인원 |
| current_members | number | 현재 신청 인원 |
| applicants | array | 신청자 목록 (본인 글일 때만) [{id, nickname, avatar, message, status}] |
| description | string/null | 상세 설명 |
| images | string[] | 이미지 URL |
| status | string | 상태 (recruiting/closed/cancelled) |
| user | object | 등록자 {id, nickname, trust_level, avatar, last_active} |
| is_mine | boolean | 본인 글 여부 |
| can_edit_core_fields | boolean | 핵심 조건 수정 가능 여부 |
| application_count_total | number | 누적 신청 건수(상태 무관) |
| my_application | object/null | 내 신청 정보 {id, status, message} |
| created_at | string | 등록일 |
| view_count | number | 조회수 |
| is_bumped | boolean | 끌어올리기 여부 |

### API: POST /api/companion/:id/apply (동행 신청)

**Request Body:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| message | string | X | 한줄 메시지 (max 100자) |

### API: PATCH /api/companion/:id/apply/:applyId (신청 수락/거절)

**Request Body:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| action | string | O | accept / reject |

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 자동 마감 | 동행 일시 경과 시 자동으로 "마감" 상태 전환 |
| 2 | 인원 충족 마감 | 수락 인원 = 모집 인원이면 자동 "마감" |
| 3 | 중복 신청 불가 | 동일 동행글에 1회만 신청 가능 |
| 4 | 본인 신청 불가 | 본인 작성 동행글에 신청 불가 |
| 5 | 신청 수락 시 채팅 | 작성자가 신청 수락 시 1:1 채팅방 자동 생성 |
| 6 | 신청 취소 | 수락 전까지 신청자가 직접 취소 가능 |
| 7 | 수정 제한 | 신청 이력 1건 이상(상태 무관)이면 핵심 조건(동행일/장소/모집인원) 수정 불가 |
| 8 | 삭제 제한 | 수락된 신청이 있으면 삭제 불가 (마감 후 삭제 가능) |
| 9 | 이벤트 연동 | 연결 이벤트 삭제 시 동행글 유지 (이벤트명 텍스트 보존) |
| 10 | 본인인증 게이팅 | 동행 신청/채팅 문의는 로그인+본인인증 완료 계정만 가능 |
| 11 | 잠금 안내 노출 | 수정 시도 시 고정 안내 배너/토스트 노출 후 진입 차단 |

## 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 정상 (모집중) | status=recruiting | 전체 UI + 신청 버튼 활성 |
| 정상 (마감) | status=closed | 전체 UI + 하단 바 "마감된 동행입니다" |
| 정상 (취소) | status=cancelled | 전체 UI + 하단 바 "취소된 동행입니다" |
| 이미 신청 | my_application 존재 | 하단 바 "신청 완료" + 취소 버튼 |
| 본인 글 | is_mine=true | 하단 바 수정/끌어올리기/마감/삭제 |
| 본인 글(수정 잠금) | is_mine=true AND can_edit_core_fields=false | 하단 바 "수정" 비활성 + 잠금 안내 문구 |
| 로딩 | 데이터 페칭 중 | 스켈레톤 UI |
| 에러 | API 실패 | "잠시 후 다시 시도해주세요" + 재시도 버튼 |
| 삭제됨 | 404 | "삭제된 동행글입니다" + 목록으로 돌아가기 |
| 비로그인 | 미로그인 + 신청 시도 | 로그인 유도 팝업 |
| 미인증 | 로그인 + 미인증 + 신청/채팅 문의 시도 | 본인인증 유도 바텀시트 |
