## 화면 개요

> 작품, 캐릭터, 굿즈를 통합 검색하는 전용 페이지. 홈 검색 바 탭 시 진입하며, 자동완성 / 최근 검색어 / 인기 검색어 / 검색 결과 필터링을 한 화면에서 제공한다. 검색 결과는 거래 유형별(교환/공구/구함) 탭으로 분류되며, 필터 바텀시트를 통해 상세 조건을 적용할 수 있다.

---

## 진입 경로

- 홈 > 상단 검색 바 탭
- 교환/공구/구함 목록 > 헤더 검색 아이콘 탭
- 하단 네비게이션 어디에서든 검색 바 탭
- 딥링크 /search?q=키워드 (외부 공유)

## 화면 구성 요소

### A. 검색 입력 모드 (초기 진입 상태)

#### A-1 검색 헤더
- 좌측: <- 뒤로 버튼 (이전 화면 복귀)
- 중앙: 검색 인풋 (자동 포커스, 키보드 올라옴)
  - placeholder: "작품, 캐릭터, 굿즈 검색"
  - 높이 44px, 라운딩 10px, 배경 `#F3F4F6`
  - 좌측 검색 아이콘 (16px, `#9CA3AF`)
  - 포커스: 보더 2px `#3B82F6` + ring 2px
- 우측: X 클리어 버튼 (입력값 있을 때만 노출)

#### A-2 자동완성 드롭다운
- 트리거: 2자 이상 입력 시, 디바운스 300ms
- 최대 5개 항목
- 각 항목: 검색 아이콘 + 검색어 텍스트 (일치 부분 Bold Blue `#3B82F6`)
- 항목 높이: 44px
- 구분선: 1px `#F3F4F6`
- 보더: 1px `#E5E7EB`, 라운딩 10px, 그림자 shadow-md

#### A-3 최근 검색어
- 조건: 입력값 없을 때 표시
- 섹션 헤더: "최근 검색어" (13px Bold) + 우측 "전체 삭제" (12px `#9CA3AF`)
- 칩 형태: pill (라운딩 999px), 배경 `#F3F4F6`, 12px
- 칩 우측: X 삭제 아이콘 (개별 삭제)
- 최대 10개, FIFO
- 저장: 로컬 스토리지 (로그인 시 서버 동기화)
- 0건일 때: "최근 검색어가 없어요" 안내 텍스트

#### A-4 인기 검색어
- 섹션 헤더: "인기 검색어" (13px Bold) + 우측 갱신 시각 (11px `#9CA3AF`)
- 칩 형태: pill (라운딩 999px), 배경 `#EFF6FF`, 텍스트 `#2563EB`
- 순위 번호 포함 (1~10)
- 최대 10개, 서버 집계 24시간 기준 갱신
- 2열 레이아웃 (모바일), 가로 나열 (데스크톱)

---

### B. 검색 결과 모드 (검색 실행 후)

#### B-1 검색 결과 헤더
- 좌측: <- 뒤로 (검색 입력 모드로 복귀, 입력값 유지)
- 중앙: 검색 바 (검색어 표시, 탭 시 입력 모드 전환)
- 레이아웃: 입력 모드 헤더와 동일 구조

#### B-2 거래 유형 탭
- 전체(N) | 교환(N) | 공구(N) | 구함(N)
- 활성 탭: `#2563EB` 텍스트 + 하단 2px 인디케이터
- 비활성 탭: `#9CA3AF` 텍스트
- 탭 전환 시 인디케이터 슬라이드 애니메이션
- 각 탭에 건수 표시

#### B-3 필터 & 정렬 바
- 가로 스크롤 칩 바
- 첫 번째 칩: 필터 아이콘 + "필터" (탭 시 필터 바텀시트 올라옴)
- 이후 칩: 선택된 필터 표시 (예: "애니/만화", "최신순", "거래 가능")
- 선택된 칩: 보더 `#3B82F6` + 배경 `#EFF6FF` + 텍스트 `#2563EB`
- 미선택 칩: 보더 `#E5E7EB` + 텍스트 `#374151`

#### B-4 결과 건수
- "검색 결과 N건" (12px `#6B7280`, N은 Bold)

#### B-5 검색 결과 카드 리스트
- 카드 레이아웃: 리스트형 (좌측 이미지 80x80 + 우측 정보)
- 카드 구성:
  - 이미지 썸네일 (80x80px, 라운딩 8px)
  - 거래유형 태그 + 카테고리 태그
  - 상품명 (13px Bold, 1줄 ellipsis)
  - 작품명 / 캐릭터명 (11px `#9CA3AF`)
  - 등록 시간 / 조회수 (11px `#9CA3AF`)
- 카드 간격: 8px
- 무한 스크롤 (Intersection Observer)
- 스켈레톤 UI: 로딩 시 카드 형태 펄스 애니메이션

#### B-6 빈 검색 결과
- 검색 아이콘 (40px, `#D1D5DB`)
- "검색 결과가 없어요" (14px Bold `#374151`)
- "다른 키워드로 검색해 보세요" (12px `#9CA3AF`)
- CTA: "전체 목록 보기" 버튼 (Secondary, sm)

---

### C. 필터 바텀시트

#### C-1 바텀시트 구조
- 딤 배경: rgba(0,0,0,.4)
- 라운딩: 16px 16px 0 0
- 진입: 300ms translateY (하단 -> 상단)
- 드래그 핸들: 40x4px, `#D1D5DB`, 중앙
- 헤더: "필터" (18px Bold) + 우측 "초기화" (13px `#6B7280`)
- Desktop(768px+): 중앙 모달 전환, 최대 너비 480px

#### C-2 필터 그룹 (위 -> 아래 순서)
1. **카테고리**: 애니/만화/소설 | 아이돌/아티스트 | 배우/무대 | 게임
   - 선택 시 해당 카테고리 컬러 보더+배경 (Purple/Pink/Amber/Cyan)
   - 복수 선택 가능
2. **굿즈 타입**: 포토카드 | 아크릴 | 인형/플러시 | 앨범/CD | 기타
   - 선택 시 Blue 보더+배경, 복수 선택
3. **상품 상태**: S급 | A급 | B급 | C급
   - 복수 선택 가능
4. **거래 방법**: 택배 | 직거래 | 둘 다
   - 복수 선택 가능
5. **정렬**: 최신순 | 인기순 | 조회순
   - 단일 선택 (기본: 최신순)
6. **추가 옵션**: 체크박스 형태
   - "차액 협의 가능만"
   - "거래 가능만"

#### C-3 하단 버튼
- 초기화 (Ghost, flex:1): 모든 필터 해제, 기본값 복원
- "N건 보기" (Primary, flex:2): 필터 적용 + 바텀시트 닫기 + 결과 즉시 갱신
- N = 실시간 결과 건수 (0건이어도 버튼 활성)

## 데이터 필드

### API: GET /api/search

**Request (Query Parameters):**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| q | string | O | 검색 키워드 |
| type | string | X | 거래 유형 (all/exchange/group_buy/wanted), 기본 all |
| category | string | X | 카테고리 (anime_manga_novel/idol_artist/actor_stage/game), 복수 콤마 구분 |
| goods_type | string | X | 굿즈 타입 (photocard/acrylic/plush/album/etc), 복수 콤마 구분 |
| condition | string | X | 상품 상태 (S/A/B/C), 복수 콤마 구분 |
| trade_method | string | X | 거래 방법 (delivery/meetup/both), 복수 콤마 구분 |
| sort | string | X | 정렬 (latest/popular/views), 기본 latest |
| has_cash_diff | boolean | X | 차액 협의 가능만 |
| available_only | boolean | X | 거래 가능만 |
| page | number | X | 페이지 번호 (기본 1) |
| limit | number | X | 페이지당 개수 (기본 20) |

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| items | array | 검색 결과 목록 |
| items[].id | string | 매물 ID |
| items[].type | string | 거래 유형 (exchange/group_buy/wanted) |
| items[].images | string[] | 이미지 URL 배열 |
| items[].title | string | 상품명 |
| items[].work | object | 작품 정보 {id, name} |
| items[].character | object | 캐릭터 정보 {id, name} |
| items[].goods_type | string | 굿즈 타입 |
| items[].condition | string | 상품 상태 |
| items[].category | string | 카테고리 enum |
| items[].user | object | 등록자 {nickname, trust_level} |
| items[].created_at | string | 등록일 |
| items[].status | string | 상태 |
| items[].view_count | number | 조회수 |
| total | number | 전체 건수 |
| total_by_type | object | 유형별 건수 {all, exchange, group_buy, wanted} |
| has_next | boolean | 다음 페이지 존재 여부 |

### API: GET /api/search/autocomplete

**Request:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| q | string | O | 입력 중인 검색어 (2자 이상) |

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| suggestions | array | 자동완성 추천 (최대 5개) |
| suggestions[].text | string | 추천 검색어 |
| suggestions[].type | string | 매칭 유형 (work/character/goods) |

### API: GET /api/search/popular

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| keywords | array | 인기 검색어 (최대 10개) |
| keywords[].rank | number | 순위 |
| keywords[].text | string | 검색어 |
| keywords[].change | string | 변동 (up/down/new/same) |
| updated_at | string | 갱신 시각 |

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|

| 11 | SEO | 검색 결과 페이지 SSR 지원, Open Graph 메타태그 |
