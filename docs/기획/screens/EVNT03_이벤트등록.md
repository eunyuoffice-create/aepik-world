## 화면 개요

> 팬 이벤트(생일카페, LED 트럭 광고, 전광판 등) 정보를 등록하는 전용 폼 화면. 수정 모드에서도 동일 화면을 사용하며, 기존 데이터를 프리필(prefill)한다.

---

## 진입 경로

- 이벤트 캘린더(EVNT01) > FAB 탭
- 이벤트 상세(EVNT02) > 본인 글 [수정] 탭
- 홈(HOME01) > FAB 탭 (이벤트 등록 선택)

---

## 선행 조건

- 로그인 필수
- 본인인증 필수 (미인증 시 인증 유도 바텀시트)

---

## 화면 구성 요소

### ① Header
- 좌측: `ChevronLeft` 뒤로가기 (작성 중 이탈 시 확인 다이얼로그)
- 중앙: "이벤트 등록" (수정 모드: "이벤트 수정")
- 우측: 없음

### ② 카테고리 선택
- 4대 카테고리 칩: 애니/만화/소설(Purple #8B5CF6), 아이돌/아티스트(Pink #EC4899), 배우/무대(Amber #F59E0B), 게임(Cyan #06B6D4)
- 단일 선택 (토글 불가)
- 필수 항목 (*)
- API enum: `anime_manga_novel / idol_artist / actor_stage / game`

### ③ 이벤트 유형
- 드롭다운 선택
- 옵션: 생일카페 / 지하철 광고 / LED 트럭 / 전광판 / 버스 광고 / 카페 이벤트 / 기타
- 필수 항목 (*)

### ④ 작품/아이돌/아티스트/배우 선택
- 검색 인풋 + 자동완성 드롭다운 (COMM02 검색 컴포넌트 사용)
- 검색 결과 없을 시: "직접 추가" 버튼 → 관리자 승인 후 정식 등록
- 필수 항목 (*)

### ⑤ 캐릭터/멤버 선택
- 작품 선택 후 드롭다운 활성화
- 해당 작품의 등록된 캐릭터/멤버 목록 표시
- "직접 추가" 가능
- 필수 항목 (*)

### ⑥ 이벤트명
- 텍스트 인풋 (최대 50자)
- 플레이스홀더: "이벤트명을 입력해주세요"
- 예: "하이큐 히나타 생일카페"
- 필수 항목 (*)

### ⑦ 이벤트 기간
- 시작일 ~ 종료일 날짜 피커
- 날짜 선택 후 해당 기간이 캘린더에 표시
- 필수 항목 (*)

### ⑧ 장소
- 주소 검색 인풋 + 카카오맵 API
- 지도에서 핀 드래그로 미세 조정 가능
- 선택한 주소 텍스트 표시 (위도/경도 저장)
- 인라인 지도 높이: 200px
- 필수 항목 (*)

### ⑨ 운영 시간 (선택)
- 시작 시간 ~ 종료 시간 타임 피커
- 선택 항목

### ⑩ 상세 설명 (선택)
- 텍스트에리어 (최대 1000자)
- 플레이스홀더: "이벤트 상세 정보, 참고사항 등을 작성해주세요"
- 선택 항목

### ⑪ 대표 이미지 (선택)
- 최대 3장 (JPG/PNG, 10MB 이하)
- 드래그 정렬 가능 (첫 번째 이미지 = 대표 이미지)
- + 버튼으로 추가, ✕ 버튼으로 삭제
- 카메라 / 앨범 선택 시트
- COMM02 이미지 업로드 컴포넌트 사용
- 선택 항목

### ⑫ 외부 링크 (선택)
- URL 인풋 (유효성 검사)
- 플레이스홀더: "https://"
- 선택 항목

### ⑬ 등록하기 버튼
- 필수 항목 모두 입력 시 활성화 (Blue)
- 미입력 시 비활성 (Gray)
- 탭 시: API 호출 → 성공 시 캘린더로 이동 + 관심 유저 알림 발송
- 수정 모드: "[수정하기]" 텍스트
- 하단 고정

### ⑭ 하단 Safe Area
- iOS 하단 여백 고려

## 데이터 필드

### API: POST /api/events (등록) / PUT /api/events/[id] (수정)

**Request Body:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| title | string | O | 이벤트명 (최대 50자) |
| category | string | O | 카테고리 (anime_manga_novel / idol_artist / actor_stage / game) |
| event_type | string | O | 이벤트 유형 |
| work_id | string | O | 작품 ID |
| character_id | string | O | 캐릭터/멤버 ID |
| start_date | date | O | 이벤트 시작일 |
| end_date | date | O | 이벤트 종료일 |
| location_name | string | O | 장소명 (주소) |
| location_lat | number | O | 위도 |
| location_lng | number | O | 경도 |
| start_time | time | X | 운영 시작 시간 |
| end_time | time | X | 운영 종료 시간 |
| description | string | X | 상세 설명 (최대 1000자) |
| images | File[] | X | 대표 이미지 (최대 3장) |
| external_url | string | X | 외부 링크 (유효한 URL) |

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 이미지 리사이즈 | 업로드 시 Sharp로 max 1200px 리사이즈 + WebP 변환 |
| 2 | 중복 등록 방지 | 동일 사용자 같은 작품+이벤트+기간 5분 이내 재등록 불가 |
| 3 | 기간 유효성 | 시작일 ≤ 종료일 (같은 날짜 가능) |
| 4 | 임시 저장 | 폼 입력 중 localStorage에 자동 저장 (입력 후 2초 디바운스) |
| 5 | 수정 모드 | 기존 데이터 프리필, 이미지 추가/삭제 가능 |
| 6 | 관심 유저 알림 | 등록 완료 시 관심 설정된 유저에게 FCM 알림 발송 |
| 7 | URL 검증 | external_url 입력 시 정규식으로 유효한 URL 확인 |

## 인터랙션

| # | 트리거 | 동작 | 비고 |
|---|--------|------|------|
| 1 | `ChevronLeft` 탭 | 이탈 확인 다이얼로그 | "작성 중인 내용이 있습니다. 나가시겠습니까?" |
| 2 | 이미지 + 탭 | 카메라/앨범 선택 시트 | COMM02 이미지 업로드 컴포넌트 |
| 3 | 작품/아이돌 인풋 타이핑 | 자동완성 드롭다운 표시 | 디바운스 300ms |
| 4 | "직접 추가" 탭 | 모달 폼 → 사용자 입력 → 관리자 승인 대기 | 작품/캐릭터 DB에 별도 요청으로 저장 |
| 5 | 카테고리 칩 선택 | 작품 입풋 리셋 + 카테고리 필터 활성화 | 카테고리 변경 시 기존 선택 작품 초기화 |
| 6 | 날짜 피커 탭 | 달력 팝업 표시 | iOS/Android 네이티브 피커 우선 |
| 7 | 주소 인풋 + Enter | 카카오맵 검색 → 지도 이동 + 핀 표시 | 첫 결과를 기본 선택, 수동 조정 가능 |
| 8 | 지도 핀 드래그 | 좌표 업데이트 + 주소 역 지오코딩 | 실시간 주소 텍스트 갱신 |
| 9 | [등록하기] / [수정하기] 탭 | 전체 필수 검증 → API 호출 → 성공 시 캘린더 이동 | 로딩 스피너 표시 |

## 화면 상태

| 상태 | 설명 |
|------|------|
| 기본 | 빈 폼, 필수 항목 빈칸 → 등록 버튼 비활성 (Gray) |
| 입력 완료 | 모든 필수 항목 입력 → 등록 버튼 활성 (Blue) |
| 수정 모드 | URL param id 있으면 기존 데이터 프리필 + 제목 "이벤트 수정" |
| 로딩 | 등록/수정 처리 중 → 버튼 로딩 스피너 표시 |
| 에러 | 등록 실패 → 토스트 메시지 "등록에 실패했습니다. 다시 시도해주세요" |
| 미인증 | 로그인은 됐으나 본인인증 미완료 → 인증 유도 바텀시트 |
