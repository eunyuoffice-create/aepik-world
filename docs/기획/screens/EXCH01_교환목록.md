## 화면 개요

> 등록된 교환글을 필터링하고 탐색하는 메인 목록 화면. 작품/캐릭터/굿즈 타입 등 다양한 조건으로 필터링하여 원하는 교환 매물을 찾을 수 있다.

## 화면 구성 요소

### ① Header
- 좌측: 페이지 제목 "교환"
- 우측: 검색 아이콘(`Search`) — 탭 시 검색 페이지(SRCH01) 이동

### ② 필터 바 (FilterChips)
- 가로 스크롤 칩 형태
- 필터 항목: 작품 ▾ / 캐릭터 ▾ / 굿즈 타입 ▾ / 상품 상태 ▾ / 거래 방법 ▾ / 차액 여부 ▾
- 각 칩 탭 시: 바텀시트 → 옵션 선택 → 칩에 선택값 표시 (예: "작품" → "하이큐")
- 초기화 버튼: 전체 필터 초기화 (선택된 필터가 있을 때만 표시)

### ②-1 필터 미적용 (초기) 상태
- 어떤 필터도 선택되지 않은 상태로 진입
- 칩 라벨은 기본값(작품 ▾ / 캐릭터 ▾ / 굿즈 타입 ▾ ...)만 노출
- 초기화 버튼(`초기화`)은 숨김
- 결과 수는 전체 교환글 기준 집계값 표시

### ②-2 필터 선택 바텀시트 (옵션 선택 상태)
- 필터 칩 탭 시 화면 하단에서 바텀시트 노출
- 상단: 제목(예: `작품 선택`) + 닫기 아이콘(`X`)
- 본문: 검색 입력 + 옵션 리스트 (선택 상태 체크)
- 하단 액션: `초기화` / `적용` 버튼
- 적용 탭 시: 바텀시트 닫힘 → 칩에 선택값 반영 → 목록 재조회

### ②-3 PC 레이아웃 (Desktop)
- 768px 이상에서 데스크톱 레이아웃 적용
- 상단 TopBar + 필터 바 + 정렬/결과수 + 4열 목록 그리드
- 모바일 BottomNav는 PC에서 노출하지 않음
- 등록 진입은 우측 상단 `교환글 등록` 버튼 또는 고정 CTA로 제공

### ②-4 PC 필터 선택 팝오버 상태
- PC에서 필터 칩 클릭 시 해당 칩 하단에 팝오버 노출
- 팝오버 구성: 검색 입력 + 옵션 리스트 + `초기화`/`적용`
- 바텀시트 대신 드롭다운/팝오버로 빠른 선택 제공
- `적용` 시 목록/건수 즉시 갱신, `닫기` 시 미적용 유지

### ③ 정렬 + 결과 수
- 좌측: 전체 N개 (검색 결과 수)
- 우측: 정렬 드롭다운 — 최신순(기본) / 인기순 / 가까운순
- 가까운순은 위치 권한 허용 시에만 활성화

### ④ 교환글 그리드
- 2열 그리드 (모바일) / 4열 (데스크톱)
- 카드 구성: ProductCard 컴포넌트
  - 이미지 (1:1 비율, object-fit: cover)
  - 거래유형 태그 + 카테고리 태그
  - 상품명 (최대 2줄, text-overflow: ellipsis)
  - 작품명 · 캐릭터명 (Gray 400)
  - ★ 평점 · 거래 횟수
- 교환 완료 매물: "교환완료" 오버레이 + 반투명 처리
- 끌어올리기된 매물: `Flame` 아이콘 + "UP" 뱃지 표시
- 제안 받은 매물: "제안 N건" 뱃지 (본인 글에만 표시)
- 내 글 매물: 카드 우상단 더보기(`MoreHorizontal`) 메뉴 노출
  - 기본 메뉴: 수정 / 끌어올리기 / 삭제
  - 단, 받은 제안 이력이 1건 이상(상태 무관)이면 `수정/삭제` 비활성 + 안내 문구 노출
  - 안내 문구: "받은 제안 이력이 있어 수정할 수 없어요. 제안이 한 번이라도 도착한 글은 수정할 수 없어요."
  - 삭제 잠금 안내: "받은 제안 이력이 있어 삭제할 수 없어요. 분쟁 방지를 위해 마감으로 종료해 주세요."
  - 받은 제안 이력 0건일 때만 `수정` 선택 시 EXCH03 수정 모드 진입
- 무한 스크롤 (Intersection Observer)
- 스켈레톤 UI: 로딩 시 카드 형태 회색 박스 펄스 애니메이션

### ④-1 내 글 액션 플로우 (수정/끌어올리기/삭제)
- 더보기 메뉴(`MoreHorizontal`)에서 액션 선택 시 즉시 실행하지 않고 확인/분기 후 처리
- `수정`
  - `proposal_history_count=0`: EXCH03 수정 모드(`/exchange/[id]/edit`) 진입
  - `proposal_history_count>=1`: 수정 잠금, 안내 토스트/문구 노출
  - 안내 문구: "받은 제안 이력이 있어 수정할 수 없어요. 제안이 한 번이라도 도착한 글은 수정할 수 없어요."
- `끌어올리기`
  - 확인 팝업 노출: `취소 / 끌어올리기`
  - 확인 시 bump 처리 + 성공 토스트 + 목록 상단 재정렬 + `UP` 뱃지 갱신
  - 과금 정책: Phase 1 무료 1회, Phase 1.5부터 추가 끌어올리기 500원/회(1일 최대 3회)
- `삭제`
  - `proposal_history_count=0`: 확인 팝업 노출(`취소 / 삭제`) 후 delete 처리 + 카드 제거 + 결과 수 갱신
  - `proposal_history_count>=1`: 삭제 잠금 + 안내 토스트/문구 노출
  - 잠금 안내 문구: "받은 제안 이력이 있어 삭제할 수 없어요. 분쟁 방지를 위해 마감으로 종료해 주세요."

### ⑤ FAB (Floating Action Button)
- 우하단 고정 (bottom: 80px, right: 16px)
- 아이콘: `Edit` (+) — 교환글 등록
- 비로그인 시: 로그인 유도 팝업
- 스크롤 다운 시 축소/반투명

### ⑥ 하단 네비게이션 (BottomNav)
- 5개 탭: 홈(`Home`) | **교환(`Repeat`)** | 이벤트(`Calendar`) | 채팅(`MessageCircle`) | 마이(`User`)
- 교환 탭 활성 상태 (Blue)

### ⑦ Empty State
- 조건: 필터 결과 없음 or 아직 등록된 교환글 없음
- 메시지: "조건에 맞는 교환글이 없어요"
- 일러스트: 빈 상자 아이콘
- CTA: "교환글 등록하기" 버튼

## 화면 상태

- **필터 미적용(초기)**: 선택 칩 없음, 초기화 버튼 숨김, 전체 목록 노출
- **필터 적용**: 선택 칩(예: `하이큐 ✕`) + 초기화 버튼 노출, 조건 기반 결과 목록 노출
- **필터 선택 중(바텀시트 오픈)**: 배경 dim 처리 + 바텀시트 내 옵션 선택 가능
- **PC 기본 상태**: 4열 목록 + 상단 필터/정렬 고정 영역 노출
- **PC 필터 선택 상태**: 칩 하단 팝오버 오픈, 옵션 다중 선택/적용 가능
- **내 글 상태**: 카드에 `내 글` 표시 + 받은 제안 이력 0건일 때만 더보기 메뉴에서 수정/삭제 가능
- **수정 잠금 상태**: 받은 제안 이력 1건 이상(상태 무관)일 때 `수정` 비활성 + 안내 문구 노출
- **삭제 잠금 상태**: 받은 제안 이력 1건 이상(상태 무관)일 때 `삭제` 비활성 + 안내 문구 노출
- **끌어올리기 확인 상태**: `끌어올리기` 탭 후 확인 팝업에서 취소/진행 분기
- **삭제 확인 상태**: 받은 제안 이력 0건에서만 `삭제` 탭 후 확인 팝업 노출
- **로딩**: 스켈레톤 카드 표시
- **빈 상태**: 조건 일치 결과 없음 안내 + CTA
- **에러**: 네트워크 오류 + 다시 시도

## 데이터 필드

### API: GET /api/exchange

**Request (Query Parameters):**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| page | number | X | 페이지 번호 (기본 1) |
| limit | number | X | 페이지당 개수 (기본 20) |
| work_id | string | X | 작품 필터 |
| character_id | string | X | 캐릭터 필터 |
| goods_type | string | X | 굿즈 타입 필터 |
| condition | string | X | 상품 상태 (S/A/B/C) |
| trade_method | string | X | 거래 방법 (delivery/meetup/both) |
| has_cash_diff | boolean | X | 차액 여부 |
| sort | string | X | 정렬 (latest/popular/nearest) |

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| items | array | 교환글 목록 |
| items[].id | string | 교환글 ID |
| items[].images | string[] | 이미지 URL 배열 |
| items[].title | string | 상품명 |
| items[].work | object | 작품 정보 {id, name} |
| items[].character | object | 캐릭터 정보 {id, name} |
| items[].goods_type | string | 굿즈 타입 |
| items[].condition | string | 상품 상태 |
| items[].trade_type | string | 거래 유형 (exchange/exchange_cash) |
| items[].category | string | 카테고리 (anime_manga/idol_artist/actor_stage/game) |
| items[].user | object | 등록자 {nickname, rating, trades, trust_level} |
| items[].created_at | string | 등록일 |
| items[].status | string | 상태 (active/completed/hidden) |
| items[].pending_proposal_count | number | 대기중 제안 수 (본인 글, 뱃지 표시용) |
| items[].proposal_history_count | number | 누적 받은 제안 수 (본인 글, 수정 잠금 판단용) |
| items[].can_edit | boolean | 수정 가능 여부 (수정 잠금 정책 반영) |
| items[].can_bump | boolean | 끌어올리기 가능 여부 (쿨다운/상태 반영) |
| items[].can_delete | boolean | 삭제 가능 여부 (받은 제안 이력 0건일 때만 true) |
| items[].is_bumped | boolean | 끌어올리기 여부 |
| items[].bumped_at | string | 끌어올리기 시간 (nullable) |
| total | number | 전체 개수 |
| has_next | boolean | 다음 페이지 존재 여부 |

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 자동 비공개 | 등록 후 60일 미갱신 시 자동 비공개 |
| 2 | 교환 완료 매물 | 교환 완료된 매물은 "교환완료" 뱃지 표시 후 목록 하단 이동 |
| 3 | 차단 사용자 | 차단한 사용자의 매물은 표시하지 않음 |
| 4 | 신고 매물 | 3건 이상 신고된 매물은 관리자 검토 전까지 비공개 |
| 5 | 정렬 기본값 | 최신순 (created_at DESC) |
| 6 | 가까운순 | 위치 권한 허용 시에만 활성화, 직거래 가능 매물만 표시 |
| 7 | 끌어올리기 정렬 | 끌어올리기된 매물은 bumped_at 기준으로 목록 상단 노출 |
| 8 | 제안 뱃지 표시 | 본인이 등록한 매물에 대기중 제안이 있으면 "제안 N건" 뱃지 표시 |
| 9 | 수정 잠금 | 본인 글에 받은 제안 이력이 1건 이상(상태 무관)이면 수정 불가 (수정 메뉴 비활성) |
| 10 | 끌어올리기 확인 | 내 글 `끌어올리기`는 확인 팝업에서 확정 후 실행 (Phase 1.5 추가 500원/회) |
| 11 | 삭제 잠금 | 본인 글에 받은 제안 이력이 1건 이상(상태 무관)이면 삭제 불가 (삭제 메뉴 비활성) |
| 12 | 삭제 확인 | 받은 제안 이력이 0건일 때만 내 글 `삭제` 확인 팝업 노출 |
