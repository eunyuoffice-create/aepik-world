## 화면 개요

정적 페이지는 서비스의 법적 문서, 고객 지원, 서비스 소개를 제공하는 4개의 서브 페이지로 구성됩니다. 각 페이지는 로그인 없이 접근 가능하며, 앱 내 주요 진입점(회원가입 약관 동의, 설정 메뉴, 하단 푸터)을 통해 유입됩니다.

---

## 진입 경로

1. **이용약관 (/terms)**
   - AUTH02 회원가입 "약관 동의" 단계에서 링크
   - USER03 설정 > 정보 > "이용약관" 버튼

2. **개인정보처리방침 (/privacy)**
   - AUTH02 회원가입 "약관 동의" 단계에서 링크
   - USER03 설정 > 정보 > "개인정보처리방침" 버튼
   - 고객센터 고정 링크

3. **고객센터 (/support)**
   - 하단 Footer 링크 (모든 페이지)
   - USER03 설정 > 고객센터 버튼
   - 앱 내 헬프 버튼

4. **서비스 소개 (/about)**
   - AUTH01 비로그인 홈 > "서비스 소개" 섹션
   - 하단 Footer 링크
   - APP 온보딩 후 첫 방문 시

## 화면 구성 요소

### Sub-page 1: 이용약관 (/terms)

#### ① Header
- **레이아웃**: 상단 고정 바
- **요소**:
  - `ArrowLeft` 아이콘 버튼 (뒤로가기)
  - 타이틀: "이용약관" (가운데 정렬)
  - 우측: 공간 (추후 공유 기능 추가 시 활용)
- **스타일**: COMM01 Header 스펙 (배경 흰색, 테두리 아래 1px solid #eee)

#### ② 목차 (Table of Contents)
- **위치**: 콘텐츠 상단, 접을 수 있는 섹션
- **요소**:
  - "목차" 타이틀 + `ChevronDown` 토글 아이콘
  - 섹션 링크 목록 (스크롤 가능):
    1. 총칙
    2. 서비스 이용
    3. 교환 거래
    4. 공동구매
    5. 게시물
    6. 개인정보
    7. 책임 제한
    8. 분쟁 해결
  - 각 항목 클릭 시 해당 섹션으로 스크롤 이동 (anchor link)
- **스타일**: 배경 #f5f5f5, padding 16px, 항목 색상 #2563EB (링크)

#### ③ 콘텐츠 영역
- **레이아웃**: 스크롤 가능한 문서
- **섹션 구조**:
  - 각 섹션 헤더: 숫자 + 제목 (예: "1. 총칙")
  - 섹션 본문: 조항 목록 (조항 번호 포함)
  - 단락 간격: 16px (COMM01 스페이싱 규칙)
  - 폰트: 본문 14px / 섹션 헤더 16px 600 weight
  - 색상: 본문 #333, 헤더 #000
- **세부 항목 구조**:
  - 조항: 1.1, 1.2, ... (들여쓰기 구현)
  - 목록: • 불릿 포함 (색상 #2563EB)

#### ④ Footer (법률)
- **요소**:
  - 시행일: "시행일자: 2026-01-01"
  - 최종 수정일: "최종 수정일: 2026-03-07"
- **스타일**: 텍스트 색상 #999, 폰트 12px, 상단 테두리 1px solid #eee, padding 16px

### Sub-page 3: 고객센터 (/support)

#### ① Header
- 타이틀: "고객센터"
- 우측: 추가 아이콘 버튼 공간

#### ② 탭 네비게이션
- **2-탭 레이아웃**: FAQ | 1:1 문의
- **스타일**: COMM02 탭 컴포넌트 스펙 (밑줄 애니메이션)
- **기본 탭**: FAQ

---

#### **Tab 1: FAQ**

##### 검색 바
- **요소**:
  - 검색 아이콘 (`Search`) + 인풋 필드 (placeholder: "검색어를 입력하세요")
  - `X` 아이콘 (입력 시 나타남, 클릭 시 초기화)
- **스타일**: COMM02 검색 인풋 스펙 (배경 #f5f5f5, border-radius 8px, padding 12px)

##### 카테고리 칩 필터
- **구성**: "전체" + 4개 카테고리 (가로 스크롤)
  1. 전체 (기본값)
  2. 교환
  3. 공구
  4. 계정
  5. 이벤트
  6. 기타
- **선택 상태**: 배경 #2563EB, 텍스트 흰색 / 비선택: 배경 #e5e7eb, 텍스트 #333
- **스타일**: COMM02 칩 컴포넌트 스펙 (padding 8px 12px, border-radius 20px, 폰트 12px)
- **배치**: padding 12px (가로), 오버플로우 스크롤 가능

##### Accordion 목록
- **각 FAQ 아이템 (닫힘 상태)**:
  - 좌측: 카테고리 뱃지 (예: "교환" → 파란색 배경 #dbeafe, 텍스트 #0369a1)
  - 중앙: 질문 제목 (14px, 600 weight, 색상 #333)
  - 우측: `ChevronDown` 아이콘 (회색 #999)
  - 배경: 흰색, 테두리: 아래 1px solid #e5e7eb
  - padding: 16px

- **열린 상태**:
  - 아이콘: `ChevronUp`으로 변경 (회전 애니메이션)
  - 배경: 변경 없음 또는 미세 배경 (#fafafa)
  - 답변 콘텐츠 표시:
    - 답변 텍스트: 12px, 색상 #666, 줄높이 1.6
    - 답변 상단 padding: 12px 0
    - 리스트/링크 포함 가능

##### 상태 (FAQ)
- **로딩**: 스켈레톤 로더 (Accordion 형태)
- **빈 상태**: "검색 결과가 없습니다" (텍스트, 색상 #999, padding 40px)
- **에러**: 재시도 버튼 포함 에러 메시지

---

#### **Tab 2: 1:1 문의**

##### 로그인 체크
- **미로그인 상태**:
  - "로그인 후 1:1 문의를 작성할 수 있습니다" (메시지)
  - "로그인" 버튼 (COMM02 Primary 버튼) → AUTH02 로그인으로 이동
  - 하단: "문의 내역 조회는 로그인 필수입니다"

- **로그인 상태**: 다음 요소 표시

##### 문의 작성 폼
- **①** 카테고리 셀렉트 (필수)
  - 옵션: 교환 / 공구 / 계정 / 결제 / 이벤트 / 기타
  - 스타일: COMM02 Select 컴포넌트 스펙 (배경 흰색, 테두리 #e5e7eb)
  - 라벨: "문의 분류" (12px 600 weight)

- **②** 제목 입력 (필수)
  - 라벨: "제목" (max 100자)
  - 인풋: COMM02 Text Input (placeholder: "문제를 간단히 요약해주세요")
  - 글자 수 카운트: "0/100" (우측 하단, 12px, 색상 #999)

- **③** 내용 입력 (필수)
  - 라벨: "내용" (max 1000자)
  - 텍스트 에어리어: COMM02 Textarea (placeholder: "구체적으로 설명해주세요")
  - 글자 수 카운트: "0/1000"
  - 높이: 최소 120px, 리사이지 가능

- **④** 이미지 첨부 (선택)
  - 라벨: "이미지 첨부 (최대 3개, 5MB 이하)"
  - 드래그 앤 드롭 영역 또는 버튼: "+ 이미지 선택" (COMM02 Secondary 버튼)
  - 첨부된 이미지:
    - 썸네일 표시 (120px × 120px, border-radius 8px)
    - 각 이미지에 `X` 버튼 (제거)
    - 배경 #f5f5f5, 배치 grid (최대 3열)

- **⑤** 제출 버튼
  - 버튼 텍스트: "문의 제출"
  - 스타일: COMM02 Primary 버튼 (배경 #2563EB, 텍스트 흰색)
  - 비활성화: 필수 필드 미입력 시 (opacity 0.5)
  - 제출 후: 로딩 표시 + 성공 토스트 메시지

##### 내 문의 내역
- **위치**: 문의 작성 폼 하단
- **제목**: "내 문의 내역" (16px 600 weight)
- **목록 구성**:
  - 각 항목:
    - 카테고리 뱃지 + 제목 (14px, 600 weight)
    - 상태 뱃지: "접수" (주황 #f59e0b) / "답변완료" (초록 #10b981)
    - 작성일: "2026-03-07" (12px, 색상 #999)
    - 배경: 흰색, 테두리: 아래 1px solid #e5e7eb
    - padding: 16px
  - 클릭 시: 문의 상세 모달 또는 페이지 이동

- **상태**:
  - **빈 상태** (문의 없음): "아직 작성한 문의가 없습니다" (텍스트, 색상 #999)
  - **로딩**: 스켈레톤 리스트
  - **에러**: 재시도 버튼 포함

### Sub-page 4: 서비스 소개 (/about)

#### ① Header
- 타이틀: "서비스 소개"

#### ② Hero Section
- **배경**: 그라데이션 또는 단색 (추천: 라이트 블루 #f0f9ff)
- **콘텐츠**:
  - 애픽 월드 로고 (120px × 120px 또는 비율 유지)
  - 타이틀: "애픽 월드" (20px 600 weight)
  - 태그라인: "팬이 만드는 굿즈 세상" (14px, 색상 #666)
  - 설명: "좋아하는 작품, 아티스트, 배우의 굿즈를\n사람들과 교환하고 함께 구매하세요" (12px, 색상 #666, 줄높이 1.6)
- **배치**: 상단 padding 32px, 가운데 정렬, 하단 padding 32px

#### ③ 4대 카테고리 Showcase
- **구성**: 4개 카드 (가로 2열 그리드, 모바일 시 1열)
- **각 카드**:
  - 배경색: 해당 카테고리 컬러 (Purple/Pink/Amber/Cyan)
  - 아이콘: 카테고리별 아이콘 (SVG, 흰색, 48px)
    - 애니/만화/소설: `BookOpen` (Purple #8B5CF6)
    - 아이돌/아티스트: `Music` (Pink #EC4899)
    - 배우/무대: `Film` (Amber #F59E0B)
    - 게임: `Gamepad2` (Cyan #06B6D4)
  - 제목: "애니/만화/소설" 등 (14px 600 weight, 흰색)
  - 설명: 카테고리별 한 문장 설명 (12px, 색상 rgba(255,255,255,0.8))
  - 높이: 최소 120px
  - 테두리: 없음
  - 호버 효과: 배경 밝기 +10% (opacity 또는 shadow 추가)
  - padding: 20px

#### ④ 기능 소개 Section
- **제목**: "핵심 기능" (16px 600 weight)
- **카드 레이아웃** (가로 스크롤 또는 그리드, 모바일 시 세로):
  - 카드 1: "교환" + 설명 (예: "원하는 굿즈를 찾아 제안하세요")
  - 카드 2: "공동구매" + 설명 (예: "함께 구매해서 비용을 절감하세요")
  - 카드 3: "구함" + 설명 (예: "원하는 상품을 찾는 사람들에게 알려주세요")
  - 카드 4: "이벤트" + 설명 (예: "팬이벤트를 직접 알려보세요")
  - 각 카드: 배경 #f9fafb, 테두리 1px solid #e5e7eb, padding 16px, border-radius 12px

#### ⑤ 브랜드 스토리 Section
- **제목**: "우리의 이야기" (16px 600 weight)
- **콘텐츠**: 3~4문단 텍스트

  - 폰트: 12px, 색상 #555, 줄높이 1.8

| 7 | 1:1 문의 탭 진입 (로그인) | 문의 작성 폼 + 내 문의 내역 표시 | 폼 포커스 가능 |
| 8 | 이미지 선택 버튼 클릭 | 파일 피커 열기 | 이미지 선택 후 썸네일 표시 |
| 9 | 문의 제출 버튼 클릭 | 유효성 검사 → 제출 → 성공 토스트 | 로딩 상태 표시, 성공 후 폼 초기화 |
| 10 | "지금 시작하기" 버튼 클릭 | 로그인 상태에 따라 분기 | HOME01 또는 AUTH02로 이동 |
