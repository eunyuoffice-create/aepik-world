## 화면 개요
> 팬이 직접 디자인한 이벤트 홍보 배너 또는 팝업을 신청하는 화면. 슬롯 달력에서 날짜별 잔여 슬롯을 확인하고, 배너/팝업 이미지를 업로드하여 게시를 신청할 수 있다. Phase 1에서는 무료, Phase 1.5부터 유료 전환.

## 진입 경로
- 이벤트 캘린더(EVNT01) > 배너/팝업 신청 버튼
- 이벤트 상세(EVNT02) > 배너/팝업 신청
- 마이페이지(USER01) > 내 배너/팝업 관리
- 홈 > 배너 캐러셀 영역 > "배너 신청하기" 링크

## 화면 구성 요소

### (1) Header
- 좌측: `ChevronLeft` 뒤로가기
- 중앙: "배너/팝업 신청"
- 우측: 없음

### (2) 신청 유형 선택
- 카드 형태 선택 (복수 선택 가능):
  - **메인 배너**: 1200×300px, 홈 캐러셀, 5,000~10,000원/일, max 8슬롯/일 표시
  - **피드 배너**: 카드너비×180px, 목록 사이, 2,000~5,000원/일, max 4슬롯/일 표시
  - **팝업**: 400×500px, 접속 시 팝업, 3,000~7,000원/일, max 3슬롯/일 표시
- 각 카드: 미리보기 mockup 이미지 + 사이즈 + 위치 설명 + 가격
- 선택 시 파란 테두리 + 체크 아이콘

### (3) 대상 정보
- 작품/아이돌/아티스트/배우 선택 (필수, 드롭다운/검색)
- 캐릭터/멤버 선택 (필수)
- 이벤트 종류 선택 (필수, 칩: 생일카페/데뷔 기념/콘서트/팬미팅/전시/기타)

### (4) 슬롯 달력
- 월별 달력 UI (CalendarPicker)
- 날짜별 슬롯 잔여 표시:
  - 여유 (>50%): 초록 도트
  - 반이상 차지 (25~50%): 노랑 도트
  - 거의 매진 (<25%): 빨강 도트
  - 매진 (0): 회색 배경 + 선택 불가 + "매진" 텍스트
- 게시 기간 선택: 시작일~종료일 드래그 또는 탭
- 선택 기간 하이라이트 (파란 배경)
- 기간 내 매진 날짜 포함 시 선택 불가 안내
- 하단: 선택 기간 요약 (N일간, 예상 비용)

### (5) 이미지 업로드
- 선택한 유형별 가이드 표시:
  - 메인 배너: 1200×300px (모바일: 375×180px)
  - 피드 배너: 카드너비×180px
  - 팝업: 모바일 320×400px / 데스크톱 400×500px
- 이미지 업로드 영역 (드래그&드롭 또는 탭)
- 업로드 후 미리보기 (실제 노출 모습 시뮬레이션)
- 저작권 가이드라인 안내 박스:
  - O 허용: 팬아트, 이름·텍스트 기반 디자인, 2차 창작 일러스트, 로고 패러디
  - X 비허용: 공식 사진, 앨범/굿즈 공식 이미지, 타인 팬아트 도용, 공식 로고

### (6) 추가 정보
- 외부 링크 (선택, URL input: 트위터/인스타 등)
- 관련 이벤트 연결 (선택, 이벤트 캘린더에서 검색/선택)
- 배너에 표시할 텍스트 (선택, 이벤트명 오버레이용)

### (7) 비용 안내 (Phase 1.5~)
- Phase 1: "현재 무료로 이용 가능합니다" 안내
- Phase 1.5~: 유형별 일 가격 × 선택 기간 = 총 비용 표시
- 결제 수단 선택 (Phase 1.5~): 카카오페이/토스페이/카드

### (8) 하단 CTA
- "신청하기" Primary 버튼 (전체 너비)
- 신청 후: 확인 팝업 "관리자 검수 후 게시됩니다 (24시간 이내)" + 마이페이지로 이동

### (9) 내 배너/팝업 관리 (마이페이지 연계)
- 이 화면이 아닌 USER01/USER02에서 접근하지만 여기서 연관 화면으로 참조
- 신청 내역 리스트 (대기중/승인/수정요청/반려/게시중/종료)
- 수정요청 시 재편집 → 재검수 플로우

## 데이터 필드

### API: POST /api/banner/apply

**Request Body:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| type | string[] | O | 신청 유형 (main_banner/feed_banner/popup, 복수 가능) |
| work_id | string | O | 작품 ID |
| character_id | string | O | 캐릭터/멤버 ID |
| event_type | string | O | 이벤트 종류 (birthday_cafe/debut/concert/fanmeeting/exhibition/other) |
| images | object | O | 유형별 이미지 {main_banner?, feed_banner?, popup?} |
| start_date | string | O | 게시 시작일 (ISO date) |
| end_date | string | O | 게시 종료일 (ISO date) |
| external_link | string | X | 외부 링크 URL |
| related_event_id | string | X | 연결된 이벤트 ID |
| overlay_text | string | X | 배너 텍스트 오버레이 |

### API: GET /api/banner/slots

**Request:**
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| type | string | O | 배너 유형 (main_banner/feed_banner/popup) |
| month | string | O | 조회 월 (YYYY-MM) |

**Response:**
| 필드 | 타입 | 설명 |
|------|------|------|
| slots | array | 날짜별 슬롯 정보 |
| slots[].date | string | 날짜 |
| slots[].total | number | 전체 슬롯 수 |
| slots[].used | number | 사용 중 슬롯 수 |
| slots[].available | number | 잔여 슬롯 수 |
| slots[].is_full | boolean | 매진 여부 |

| 1 | 슬롯 확인 | 선택 기간의 모든 날짜에 잔여 슬롯 있어야 신청 가능 |

| 9 | 복수 신청 | 배너+팝업 동시 신청 가능, 각각 별도 슬롯 소진 |

## 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 정상 | 폼 표시 | 신청 폼 |
