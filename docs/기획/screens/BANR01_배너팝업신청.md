## 화면 개요
> 메인 배너, 피드 배너, 팝업의 슬롯 현황을 한눈에 보여주는 화면. 현재 진행 중인 배너/팝업 목록, 유형별 잔여 슬롯, 마감 여부, 다음 신청 기간을 안내한다. 신청 가능한 유형에 한해 [신청] 버튼이 활성화되며, 탭 시 등록 화면(BANR02)으로 이동한다.

## 진입 경로
- 이벤트 캘린더(EVNT01) > 배너/팝업 신청 스트립 > [신청] 버튼
- 이벤트 상세(EVNT02) > 배너/팝업 신청
- 마이페이지(USER01) > 내 배너/팝업 관리
- 홈(HOME01) > 팝업 오버레이 > "나도 배너/팝업 올리기"

## 화면 구성 요소

### (1) Header
- 좌측: `ChevronLeft` 뒤로가기
- 중앙: "배너/팝업"
- 우측: 없음

### (2) 현재 진행 중 배너/팝업
- 섹션 헤더: "지금 진행 중" + 전체 건수 뱃지
- 가로 스크롤 카드 리스트 (최대 10건, 최신순)
- 카드 구성:
  - 이미지 썸네일 (64x64, radius 8px)
  - 이벤트명 (1줄, ellipsis)
  - 기간: "3/8 ~ 3/14" (시작일~종료일)
  - 유형 뱃지: "메인 배너" / "피드 배너" / "팝업" (작은 pill)
  - 카테고리 컬러 도트 (Purple/Pink/Amber/Cyan)
- 카드 탭 시: 해당 배너/팝업 상세 보기 (이미지 풀뷰 + 정보)
- 진행 중 없을 시: "현재 진행 중인 배너/팝업이 없어요"

### (3) 슬롯 현황 카드 (유형별)
- 3개 카드 세로 나열 (메인 배너 / 피드 배너 / 팝업)
- 각 카드 구성:
  - 유형 아이콘 + 유형명
  - 슬롯 프로그레스 바: 사용/전체 (예: 5/8)
  - 잔여 슬롯 텍스트: "3슬롯 남음" (Green) / "1슬롯 남음" (Amber) / "마감" (Red)
  - 노출 위치 설명: "홈 캐러셀" / "목록 사이" / "접속 시 팝업"
  - 이미지 규격: "1200x300px 통이미지" / "카드너비x180px 통이미지" / "320x400px 통이미지"
  - 가격 (Phase 1.5~): "5,000~10,000원/일"
  - **신청 가능 시**: [신청하기] Primary 버튼 (활성) → BANR02 이동
  - **마감 시**: [마감] 버튼 (회색, 비활성) + "다음 신청: MM/DD~" 안내
- 신청 버튼 탭 시: BANR02 (배너/팝업 등록) 이동, 선택한 유형 파라미터 전달 (type=main_banner 등)

### (4) 다음 신청 기간 안내 (마감된 유형이 있을 때)
- 안내 카드 (파란 배경):
  - `Clock` 아이콘 + "다음 신청 기간"
  - 마감된 유형별 다음 신청 가능일 표시
  - 예: "메인 배너: 3/15(월)부터 신청 가능"
  - 알림 받기 체크: "신청 가능할 때 알려드릴까요?" → 토글/체크
- 모든 유형 여유 시: 이 섹션 미노출

### (5) 내 신청 내역 바로가기
- 하단 링크: "내 배너/팝업 관리 >" → USER02 (내 활동 관리)
- 최근 신청 1건 미리보기 (있을 경우):
  - 이미지 썸네일 + 유형 + 상태 뱃지 (대기중/검수중/승인/수정요청/반려/게시중/종료)

### (6) 하단 네비게이션
- 공통 하단 탭바 (이벤트 탭 활성)

## 데이터 필드

### API: GET /api/banner/status
> 슬롯 현황 전체 조회

**Response:**
| 필드 | 타입 | 설명 |
|------|------|------|
| active_banners | array | 현재 진행 중 배너/팝업 목록 |
| active_banners[].id | string | 배너 ID |
| active_banners[].type | string | 유형 (main_banner/feed_banner/popup) |
| active_banners[].title | string | 이벤트명 |
| active_banners[].image_url | string | 썸네일 URL |
| active_banners[].start_date | string | 게시 시작일 |
| active_banners[].end_date | string | 게시 종료일 |
| active_banners[].category | string | 카테고리 |
| slots | object | 유형별 슬롯 현황 |
| slots.main_banner | object | { total, used, available, is_full, next_available_date } |
| slots.feed_banner | object | { total, used, available, is_full, next_available_date } |
| slots.popup | object | { total, used, available, is_full, next_available_date } |

### API: GET /api/banner/slots (기존 유지)
> 날짜별 슬롯 상세 (BANR02에서 사용)

## 비즈니스 로직

| # | 규칙 | 설명 |
|---|------|------|
| 1 | 슬롯 잔여 계산 | 오늘 날짜 기준, 유형별 total - used = available |
| 2 | 마감 판정 | available = 0이면 마감, 버튼 비활성 |
| 3 | 다음 신청일 | 마감 시 가장 빠른 슬롯 여유 날짜를 next_available_date로 표시 |
| 4 | 알림 등록 | "신청 가능 알림"은 FCM으로 슬롯 여유 발생 시 push |
| 5 | 진행 중 목록 | 오늘 기준 start_date <= today <= end_date인 배너/팝업 |
| 6 | 이미지 = 통이미지 | 배너/팝업은 모두 통이미지(full image)로만 구성, 텍스트 오버레이 없음 |
| 7 | 이미지 설명 | 배너 텍스트는 이미지에 노출되지 않고, img alt 속성 용도의 description |
