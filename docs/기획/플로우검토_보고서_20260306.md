# 애픽 월드 유저 플로우 종합 검토 보고서

> 검토일: 2026-03-06
> 검토 대상: 유저 플로우 MD (v1.2), FLOW HTML (v1.3), IA MD (v1.0), 기획서 16개

---

## A. 버전 불일치 문제

| 문서 | 현재 버전 | 실제 내용 수준 | 문제 |
|------|----------|-------------|------|
| 유저 플로우 MD | v1.2 | AUTH01~02 + 마이페이지 반영 | FLOW HTML은 v1.3, MD는 v1.2에서 정지 |
| FLOW HTML | v1.3 | 마이페이지(11), 끌어올리기(14) 포함 | MD보다 최신이나 리비전 이력 없음 |
| IA MD | v1.0 | USER01~03 레이블 포함 | 기반 문서 "유저 플로우 v1.0" 표기 오래됨, 리비전 히스토리 없음 |

**즉시 조치:** 유저 플로우 MD를 v1.3으로 맞추고, IA MD에 리비전 히스토리 추가

## B. 심각도 높음 - 누락된 플로우 3건

### B-1. EXCH05(교환 신청함) 플로우 완전 누락

플로우 MD 어디에도 EXCH05 화면의 상세 플로우가 없음. FLOW HTML에는 섹션 4-3에 간단히 표시되어 있으나, 다음 내용이 빠짐:

- 받은 제안 탭: 내 교환글별 그룹핑, 제안 카드, 수락/거절 액션
- 보낸 제안 탭: 상태별 관리 (대기/수락/거절/만료/취소)
- 제안 수락 시 나머지 자동 거절 로직
- EXCH02 "받은 제안 패널" -> EXCH05 "전체보기" 연계 경로

### B-2. 공구 자동 무산 타이머 플로우 불명확

GRPB02 v0.2 기획서에 추가된 "진행 결정 대기 -> 자동 무산" 타이머(D+0/D+1/D+3/D+5)가 플로우 MD에 반영되지 않음:

- 마감일 도달 후 "진행 결정 대기" 상태 진입 조건
- 진행자 액션: 진행 확정 vs 무산 처리
- 진행자 미응답 시 D+1 리마인더, D+3 경고, D+5 자동 무산
- 자동 무산 시 참여자 알림 + 환불 안내

### B-3. 구함글 상세(WANT01) 플로우 미포함

IA에는 /wanted/[id] 라우트가 정의되어 있으나, 플로우 MD 섹션 6에는 "등록" 플로우만 있고 "상세 열람 + 매칭 제안" 플로우가 없음:

- 구함글 상세 페이지 구성 요소
- 관련 교환글 자동 매칭 표시
- "채팅으로 제안하기" 버튼 -> CHAT02 연계

## C. 심각도 중간 - 세부 플로우 부족 3건

### C-1. 후기 작성 & 이의제기 상세 플로우

교환/공구 완료 후 후기 시스템의 상세 플로우가 부족:

- 후기 작성 바텀시트 (긍정/부정 + 텍스트 200자)
- 7일 이내 작성 가능, 수정/삭제 불가
- 부정 후기 이의제기 (72시간 이내, 사유+증거 첨부)
- 운영팀 검토 결과 반영

### C-2. 신고 결과 처리 플로우

섹션 12(신고)에 "접수"까지만 있고, 결과 처리 플로우가 없음:

- 신청자 화면: 검토 중 -> 완료 (조치 결과 표시)
- 피신고자 조치: 경고/정지/삭제
- USER03 설정 > 신고 내역 확인 연계

### C-3. 플로우 MD와 FLOW HTML 간 AUTH01-D 표현 차이

MD에서 AUTH01-D(로그인 유도 바텀시트)가 상세하게 기술되어 있으나, FLOW HTML에서는 간략화됨

## D. 심각도 낮음 - 미세 불일치 3건

| 항목 | MD 표현 | 기획서 표현 | 차이 |
|------|---------|----------|------|
| USER02-A 카드 액션 | "롱프레스" | "롱프레스 또는 더보기(...)  액션시트" | MD 표현이 더 간략함 |
| USER02-B 공구 서브탭 | "서브탭: 진행중/참여한" | 동일 | 일치 |
| USER03-C 탈퇴 재가입 제한 | "3개월 대기" (FLOW HTML) | "30일 제한" (PRD/기획서) | FLOW HTML 정책 노트와 PRD 불일치 가능 |

## E. IA <-> 플로우 정합성

### 포함된 화면 (23개)

HOME01, AUTH01~02, EXCH01~03, GRPB01~04, SRCH01, WANT01(등록만), EVNT01~02, BANR01, CHAT01~02, NOTI01, USER01~03

### 플로우 MD에 누락된 화면 (2개)

| 화면 | IA 라우트 | 상태 |
|------|---------|------|
| EXCH05 교환신청함 | /exchange/proposals | 플로우 MD 부재 (FLOW HTML에 간략 포함) |
| WANT01 상세 | /wanted/[id] | 등록만 있고 상세 열람 플로우 없음 |

### 플로우에 있지만 IA에 없는 화면

없음 (정합성 양호)

## F. 수정 우선순위

### 즉시 조치 (이번 작업)

1. 유저 플로우 MD v1.2 -> v1.3 버전 업그레이드
2. EXCH05(교환 신청함) 상세 플로우 추가 (섹션 4-3)
3. 공구 자동 무산 타이머 플로우 명확화 (섹션 5-5 재작성)
4. 구함글 상세(WANT01) 열람 플로우 추가 (섹션 6-1)

### 후속 조치

5. 후기 작성 & 이의제기 플로우 상세화
6. 신고 결과 처리 플로우 추가
7. IA MD 리비전 히스토리 추가
8. FLOW HTML 해당 섹션 동시 업데이트 -> v1.4

## G. 긍정적 평가

- 전반적 정합성은 높음 (AUTH, 검색, 이벤트, 채팅, 마이페이지 플로우 일관적)
- 기획서의 상세도가 우수 (비즈니스 로직, API 스펙, 인터랙션 구체적)
- 화면 ID 네이밍 규칙 일관됨
- FLOW HTML v1.3의 마이페이지(flow11) 섹션은 MD와 높은 일치도
- COMM01/02 공통 디자인 시스템이 잘 정의되어 있어 향후 개발 시 참고 용이
