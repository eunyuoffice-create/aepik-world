# 애픽 월드 (Aepik World) — 화면 기획서 & 와이어프레임 템플릿 가이드

> **이 문서는 AI가 화면별 기획서(md)와 와이어프레임(HTML)을 작성할 때 일관성을 유지하기 위한 참고 템플릿입니다.**
> 모든 화면 기획과 와이어프레임은 이 가이드를 따릅니다.

---

## 1. 전체 작업 흐름

```
[IA 정보 구조] → [화면별 기획서 (md)] → [화면별 와이어프레임 (HTML)]
      ↓                 ↓                        ↓
 페이지 목록        기능 명세서              시각적 레이아웃
 라우트 정의        데이터 필드              번호 주석 + Description
 컴포넌트 목록      비즈니스 로직             인터랙션 표시
```

## 2. 화면 기획서 (md) 작성 규칙

### 2-1. 파일 명명 규칙

```
/docs/기획/screens/SCREEN_CODE_화면명.md
```

**화면 코드 체계:**

| 접두사 | 카테고리 | 예시 |
|--------|---------|------|
| `HOME` | 홈 | HOME01_메인홈.md |
| `EXCH` | 교환 | EXCH01_교환목록.md, EXCH02_교환상세.md, EXCH03_교환등록.md |
| `GRPB` | 공구 | GRPB01_공구목록.md, GRPB02_공구상세_일반.md, GRPB03_공구상세_세트.md |
| `WANT` | 구함 | WANT01_구함목록.md |
| `EVNT` | 이벤트 | EVNT01_이벤트캘린더.md, EVNT02_이벤트상세.md |
| `CHAT` | 채팅 | CHAT01_채팅목록.md, CHAT02_채팅방.md |
| `SRCH` | 검색 | SRCH01_검색.md |
| `NOTI` | 알림 | NOTI01_알림목록.md |
| `USER` | 마이/유저 | USER01_마이페이지.md, USER02_프로필편집.md, USER03_타인프로필.md |
| `BANR` | 배너 | BANR01_배너신청.md |
| `AUTH` | 인증 | AUTH01_로그인.md, AUTH02_회원가입.md |
| `STAT` | 정적 | STAT01_이용약관.md |

### 2-2. md 기획서 템플릿

```markdown
# [화면코드] 화면명

## 메타 정보

| 항목 | 내용 |
|------|------|
| **화면코드** | EXCH01 |
| **화면명** | 교환 목록 |
| **라우트** | /exchange |
| **이용자** | PC / Mobile |
| **버전** | 0.1 |
| **작성일** | YYYY-MM-DD |
| **상태** | 초안 / 검토중 / 확정 |

---

## 화면 개요

> 이 화면의 목적과 역할을 1~2문장으로 설명

---

## 진입 경로

- 홈 > 인기 교환 > 더보기
- 하단 네비게이션 > 교환 탭
- 검색 결과 > 교환 탭

---

## 화면 구성 요소

### ① Header
- 페이지 제목: "교환"
- 우측: 필터 아이콘

### ② 필터 바
- 가로 스크롤 칩 형태
- 필터 항목: 작품 ▾ / 캐릭터 ▾ / 굿즈 타입 ▾ / 상품 상태 ▾ / 거래 방법 ▾ / 차액 여부 ▾
- 각 칩 탭 시: 바텀시트 → 옵션 선택 → 칩에 선택값 표시
- 초기화 버튼: 전체 필터 초기화

### ③ 정렬
- 옵션: 최신순(기본) / 인기순 / 가까운순
- 가까운순은 위치 권한 허용 시에만 활성화

### ④ 교환글 그리드
- 2열 그리드 (모바일) / 4열 (데스크톱)
- 카드 구성: ProductCard 컴포넌트
  - 이미지 (1:1)
  - 거래유형 태그 + 카테고리 태그
  - 상품명 (최대 2줄)
  - 작품 · 캐릭터
  - ★ 평점 · 거래 횟수
- 무한 스크롤 (Intersection Observer)
- 스켈레톤 UI: 로딩 시 카드 형태 회색 박스 표시

### ⑤ FAB (Floating Action Button)
- 우하단 고정
- 탭 시: 교환글 등록 페이지 이동
- 비로그인 시: 로그인 유도 팝업

### ⑥ Empty State
- 검색 결과 없음: "조건에 맞는 교환글이 없어요" + 일러스트
- CTA: "교환글 등록하기" 버튼

## 데이터 필드

### API: GET /api/exchange

**Request (Query Parameters):**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| page | number | ❌ | 페이지 번호 (기본 1) |
| limit | number | ❌ | 페이지당 개수 (기본 20) |
| work_id | string | ❌ | 작품 필터 |
| character_id | string | ❌ | 캐릭터 필터 |
| goods_type | string | ❌ | 굿즈 타입 필터 |
| condition | string | ❌ | 상품 상태 (S/A/B/C) |
| trade_method | string | ❌ | 거래 방법 (delivery/meetup/both) |
| has_cash_diff | boolean | ❌ | 차액 여부 |
| sort | string | ❌ | 정렬 (latest/popular/nearest) |

**Response:**

| 필드 | 타입 | 설명 |
|------|------|------|
| items | array | 교환글 목록 |
| items[].id | string | 교환글 ID |
| items[].images | string[] | 이미지 URL 배열 |
| items[].title | string | 상품명 |
| items[].work | object | 작품 정보 {id, name} |
| items[].character | object | 캐릭터 정보 {id, name} |
| items[].goods_type | string | 굿즈 타입 |
| items[].condition | string | 상품 상태 |
| items[].trade_type | string | 거래 유형 (exchange) |
| items[].category | string | 카테고리 (anime_manga_novel/idol_artist/actor_stage/game) |
| items[].user | object | 등록자 {nickname, rating, trades, trust_level} |
| items[].created_at | string | 등록일 |
| total | number | 전체 개수 |
| has_next | boolean | 다음 페이지 존재 여부 |
```

## 3. 와이어프레임 (HTML) 작성 규칙

### 3-1. 파일 명명 규칙

```
/wireframe/SCREEN_CODE_화면명.html
```

예: `EXCH01_교환목록.html`, `HOME01_메인홈.html`

### 3-2. HTML 와이어프레임 구조

모든 와이어프레임 HTML은 다음 구조를 따릅니다:

```
┌─────────────────────────────────────────────────────────────────┐
│ [메타 정보 헤더]                                                  │
│  버전 | 화면코드 | 페이지명 | 라우트 | 이용자 | 페이지              │
├────────────────────────────────┬────────────────────────────────┤
│                                │                                │
│   [와이어프레임 영역]            │   [Description 영역]           │
│                                │                                │
│   ①②③ 번호 원 주석이           │   ① [Header]                  │
│   달린 실제 레이아웃             │     - 페이지 상단 고정           │
│                                │     - 로고 + 알림 + 메뉴        │
│   회색 박스로 영역 표시          │                                │
│   [IMG] 플레이스홀더            │   ② [검색바]                   │
│   텍스트는 실제 예시 데이터      │     - 플레이스홀더 텍스트        │
│                                │     - 탭 시 검색 페이지 이동     │
│                                │                                │
│                                │   ③ [배너 캐러셀]              │
│                                │     - 자동 슬라이드 5초          │
│                                │     - 인디케이터 ● ○ ○          │
│                                │                                │
├────────────────────────────────┴────────────────────────────────┤
│ [개정 이력]                                                      │
│  날짜 | 버전 | 내용                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 3-3. 와이어프레임 스타일 규칙

| 항목 | 규칙 |
|------|------|
| **전체 레이아웃** | 좌측 60% 와이어프레임 + 우측 40% Description |
| **와이어프레임 너비** | max-width 400px (모바일 기준) |
| **색상** | 흰 배경 + 회색(#E5E5E5) 박스 + 검은 텍스트 + 빨간(#E53935) 번호 원 |
| **이미지 플레이스홀더** | 회색 박스 + "IMG" 텍스트 중앙 |
| **번호 주석** | 빨간 원(20px) + 흰색 번호 텍스트, 절대 위치 |
| **Description** | 번호별 제목 + 불릿 리스트로 상세 설명 |
| **폰트** | system-ui, -apple-system, sans-serif |
| **보더** | 1px solid #ddd (영역 구분) |

### 3-4. 번호 주석 원칙

| 규칙 | 설명 |
|------|------|
| **순서** | 위에서 아래, 왼쪽에서 오른쪽 |
| **범위** | 시각적으로 구분되는 영역 단위 (컴포넌트 단위) |
| **번호 한도** | 한 화면에 최대 15개 이내 (초과 시 화면 분리 고려) |
| **고정 번호** | 같은 컴포넌트가 여러 화면에 등장하면 동일 설명 참조 |

### 3-5. Description 작성 규칙

각 번호에 대해:

```
[번호] [컴포넌트 타입] 컴포넌트명
  • 구성 요소 설명
  • 데이터 필드 명시
  • 인터랙션 동작
  • 조건부 로직
  • 에러/빈 상태
```

예시:
```
④ [Grid] 교환글 그리드
  • 2열 그리드 (모바일) / 4열 (데스크톱)
  • 카드 구성:
    └ 이미지 (1:1 비율, 없을 경우 Placeholder 노출)
    └ 태그: 거래유형 + 카테고리
    └ 상품명 (최대 2줄, 말줄임)
    └ 작품명 · 캐릭터명
    └ ★ 평점 · 거래 횟수
  • 무한 스크롤 (하단 도달 시 다음 페이지 로드)
  • 로딩 시 스켈레톤 UI 표시
  • 카드 탭 시 교환 상세(EXCH02) 이동
```

## 4. 공통 컴포넌트 Description 참조표

반복 사용되는 컴포넌트는 아래 참조표를 인용하여 Description에서 "공통 컴포넌트 참조" 링크로 대체 가능.

### ProductCard (상품 카드)

```
• 이미지 영역: 1:1 비율, object-fit: cover
  └ 이미지 없을 시 회색 플레이스홀더
• 태그 영역: [거래유형] [카테고리]
  └ 교환=#FFF7ED, 공구=#EFF6FF, 구함=#F0FDF4, 세트공구=#FAF5FF
  └ 애니·만화·소설=#F5F3FF, 아이돌·아티스트=#FDF2F8, 배우·무대=#FFFBEB, 게임=#ECFEFF
• 상품명: 최대 2줄, text-overflow: ellipsis
• 메타: 작품명 · 캐릭터명 (Gray 400)
• 하단: ★ 평점 · 거래 N회
• 호버: shadow + translateY(-2px)
• 탭: 해당 상세 페이지 이동
```

### GroupBuyCard (공구 카드)

```
• 이미지 영역: 16:9 비율
• 태그: [공구] 또는 [세트공구]
• 상품명: 1줄
• 참여 현황: N/M명 · 마감 D-N
• 프로그레스 바: (N/M)% — 80% 이상 시 Green, 미만 시 Blue
• 탭: 공구 상세 이동
```

### EventMiniCard (이벤트 미니 카드)

```
• 날짜: 소형 텍스트
• 이벤트 아이콘 + 이벤트명
• 📍 장소
• 카테고리 태그
• 탭: 이벤트 상세 이동
```

### BottomNav (하단 네비게이션)

```
• 5개 탭: 홈(🏠) | 교환(🔄) | 이벤트(📅) | 채팅(💬) | 마이(👤)
• 비활성: Gray 400, 활성: Blue (#3B82F6) (#3B82F6)
• 채팅/마이: 비로그인 시 로그인 유도
• 채팅: 안 읽은 메시지 뱃지 표시
```

### SearchBar (검색바)

```
• 플레이스홀더: "상품명, 작품, 캐릭터 검색"
• 좌측 🔍 아이콘
• 탭 시 검색 페이지(SRCH01) 이동
• 홈에서는 폼이 아닌 탭 영역으로 동작
```

### FilterChips (필터 칩 바)

```
• 가로 스크롤 가능
• 각 칩: pill 형태, 기본 Gray 배경
• 선택 시: Blue 배경 + 흰색 텍스트
• 탭 시: 바텀시트 → 옵션 목록 → 선택 시 칩 업데이트
• 우측 끝: 초기화 버튼 (선택된 필터 있을 때만 표시)
```

### TrustBadge (신뢰 뱃지)

```
• Lv.1 🌱 새싹 (가입)
• Lv.2 ⭐ 일반 (거래 5회+)
• Lv.3 💎 우수 (거래 20회+ & 평점 4.0+)
• Lv.4 👑 VIP (거래 50회+ & 평점 4.5+)
```

### FAB (플로팅 액션 버튼)

```
• 위치: 우하단 고정 (bottom: 80px, right: 16px)
• 크기: 56×56px, border-radius: 50%
• 색상: Blue 배경, 흰색 + 아이콘
• 탭 시: 해당 등록 페이지 이동
• 비로그인 시: 로그인 유도 팝업
• 스크롤 다운 시: 축소 or 반투명 (선택)
```

## 5. 화면 상태 체크리스트

모든 화면 기획서에 아래 상태를 고려해야 합니다:

| 상태 | 설명 | 와이어프레임에 표시 |
|------|------|------------------|
| **기본 상태** | 데이터가 정상 로드된 상태 | ✅ 필수 |
| **로딩 상태** | 데이터 로드 중 (스켈레톤/스피너) | Description에 명시 |
| **빈 상태** | 데이터 없음 (Empty State) | 별도 표시 권장 |
| **에러 상태** | API 실패, 네트워크 오류 | Description에 명시 |
| **비로그인 상태** | 로그인 필요 기능 접근 시 | Description에 명시 |
| **첫 방문 상태** | 온보딩, 튜토리얼 | Phase 1.5 이후 |

## 6. 작업 순서 가이드

### 새 화면 기획 시

```
1. IA 문서에서 해당 화면 확인 (라우트, 구성 요소)
2. 유저 플로우에서 해당 화면의 진입/이탈 경로 확인
3. md 기획서 작성 (이 템플릿의 §2 참조)
4. HTML 와이어프레임 작성 (이 템플릿의 §3 참조)
5. 기획서와 와이어프레임의 번호 매칭 확인
```

### 기획 수정 시

```
1. md 기획서의 개정 이력에 변경 내용 기록
2. 와이어프레임 HTML 업데이트
3. 메타 정보의 버전 번호 증가
4. 연관 화면 영향 범위 확인
```
