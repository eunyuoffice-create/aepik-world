SRCH01 — 검색
라우트: /search  |  비로그인 접근: 가능  |  Phase 1
1
[Header] 검색 헤더
  • 좌측: 뒤로 가기 버튼 (‹), 이전 화면 복귀
  • 중앙: 검색 입력 필드
    • placeholder: "작품, 캐릭터, 굿즈 검색"
    • 높이 44px, border-radius 10px, 배경 #F3F4F6
    • 포커스 시: border 2px #2563EB + ring 2px
    • 입력값 있을 때: 우측 X 클리어 버튼 노출
  • 초기 진입 시 자동 포커스 + 키보드 올라옴
  • 결과 모드에서는 입력값 표시 + 수정 가능
2
[Chips] 최근 검색어
  • 입력값 없을 때만 표시 (결과 모드에서는 숨김)
  • 섹션 헤더: "최근 검색어" + 우측 "전체 삭제" 버튼
  • 칩 형태: pill (border-radius 999px), 배경 #F3F4F6
  • 각 칩 우측: X 삭제 아이콘 (개별 삭제)
  • 저장: 로컬 스토리지 (FIFO, 최대 10개)
  • 로그인 시 서버 동기화
  • 0건 시: "최근 검색어가 없어요" 안내 텍스트
3
[Chips] 인기 검색어
  • 입력값 없을 때만 표시
  • 섹션 헤더: "인기 검색어" + 우측 갱신 시각 (예: "방금 갱신")
  • 칩 형태: pill, 배경 #EFF6FF, 텍스트 #2563EB
  • 순위 번호 포함 (1~10)
  • 데이터: 서버 집계 (24시간 기준)
  • 칩 탭 시: 해당 검색어로 즉시 검색 실행
  • 2열 레이아웃 (모바일), 가로 나열 (데스크톱)
4
[Tabs] 거래 유형 탭
  • 표시: 전체(N) | 교환(N) | 공구(N) | 구함(N)
  • 활성 탭: #2563EB 텍스트 + 하단 2px 인디케이터
  • 비활성 탭: #9CA3AF 텍스트
  • 탭 전환 시 인디케이터 슬라이드 애니메이션
  • 각 탭 내 건수 표시 (결과 모드에서만 노출)
  • 탭 전환 시: 해당 유형 결과만 필터링 (다른 필터는 유지)
5
[Card] 검색 결과 카드
  • 레이아웃: 리스트형 (좌측 이미지 80×80 + 우측 정보)
  • 카드 구성:
    • 이미지 썸네일 (80×80px, border-radius 8px, object-fit: cover)
    • 태그: [거래유형] + [카테고리]
    • 상품명 (13px Bold, 1줄 ellipsis)
    • 작품명 · 캐릭터명 (11px #9CA3AF)
    • 등록 시간 / 조회수 또는 공구 진행률
  • 카드 간격: 8px
  • 카드 탭 시: 해당 상세 페이지로 이동
  • 로딩 시: 스켈레톤 UI (펄스 애니메이션)
6
[BottomSheet] 필터 바텀시트
  • 진입: "필터" 칩 탭 시, 300ms translateY 애니메이션
  • UI 요소:
    • 드래그 핸들: 40×4px, #D1D5DB (중앙)
    • 헤더: "필터" (18px Bold) + 우측 "초기화" (13px #6B7280)
    • 필터 그룹: 6개 (카테고리, 굿즈 타입, 상품 상태, 거래 방법, 정렬, 추가 옵션)
    • 하단 버튼: [초기화] (Ghost) + [N건 보기] (Primary)
  • Desktop (768px+): 중앙 모달 전환, 최대 너비 480px
  • 닫기: 딤 배경 탭 또는 드래그 아래로
화면 상태
  • 초기 진입: 검색 입력 모드, 최근/인기 검색어 표시, 키보드 올라옴
  • 검색 중 (디바운스): 자동완성 드롭다운 표시, 최대 5개 항목
  • 결과 모드: 거래 유형 탭 + 필터 칩 바 + 카드 리스트
  • 필터 바텀시트 열림: 딤 배경 + 바텀시트, 스크롤 고정
  • 빈 결과: 검색 아이콘 + "검색 결과가 없어요" + "다른 키워드로 검색해 보세요" + CTA 버튼
  • 로딩: 각 카드 스켈레톤 UI (펄스 애니메이션)
  • 에러: "데이터를 불러올 수 없습니다" + 재시도 버튼
