[Header] 공통 헤더
  • Step 1: X 닫기 버튼 (가입 취소, AUTH01로 이동)
  • Step 2~4: 뒤로가기 버튼 (이전 단계로 이동)
  • 제목: "회원가입" (Step 1~2) / "프로필 설정" (Step 3) / "관심 카테고리" (Step 4)
  • 높이: 56px, padding 12px 16px
2
[Stepper] 스텝 인디케이터
  • 4단계 진행 표시
  • 현재 단계: 파란색 배경(#2F8EC1) + 흰색 숫자
  • 완료 단계: 파란색 배경 + 흰색 체크마크
  • 미래 단계: 회색 배경(#DEE8F2) + 회색 숫자
  • 라인: 완료 구간은 파란색, 나머지는 회색
  • padding: 16px, flex with gap 8px
3
[Section] Step 1 - 약관 동의
  • 제목: "서비스 이용을 위해 약관에 동의해주세요" (14px, weight 600)
  • 구성:
    • └ 전체 동의 (체크박스) - 파란색 배경 토글
    • └ [필수] 이용약관 동의 + 보기 링크 → STAT01
    • └ [필수] 개인정보처리방침 동의 + 보기 링크 → STAT01
    • └ [선택] 마케팅 수신 동의
    • └ [필수] 만 14세 이상 확인
  • 동작: 모든 필수 항목(3개)이 체크되면 "다음" 활성화
  • 전체 동의 체크 → 하위 모든 항목 체크, 하나라도 미체크 → 전체 동의 자동 미체크
4
[Checkbox] 체크박스 (필수/선택 뱃지 포함)
  • 크기: 20px x 20px, border-radius 4px
  • 기본: 회색 테두리(#DEE8F2)
  • 체크됨: 파란색 배경(#2F8EC1) + 흰색 체크마크
  • 뱃지: 필수(빨강 #fecaca), 선택(보라 #e0e7ff)
  • 폰트: 13px, color #163958
5
[Link] 약관 보기 링크
  • 텍스트: "+보기" (13px, color #2F8EC1, text-decoration none)
  • 위치: 체크박스 라벨 우측에 우정렬
  • 동작: 클릭 시 STAT01 (이용약관/개인정보처리방침 페이지) 인앱 웹뷰 오픈
  • Hover: 배경 #EAF7FF, border-radius 4px
6
[Section] Step 2 - 계정 정보 입력
  • 이메일 가입 전용 (카카오/소셜 로그인 경유 시 Step 3으로 스킵)
  • 구성:
    • └ 이메일 필드 (RFC 5322 형식 검증)
    • └ 비밀번호 필드 (강도 바 표시)
    • └ 비밀번호 확인 필드 (일치 여부 표시)
  • 동작: 모든 필드 유효하면 "다음" 활성화
7
[Input] 이메일 필드
  • 플레이스홀더: "example@email.com"
  • 검증: RFC 5322 형식 체크
  • 중복 확인: 입력 후 500ms debounce 자동 확인
  • 상태:
    • └ 기본: 회색 테두리
    • └ 포커스: 파란색 테두리
    • └ 검증 실패: 빨강 테두리 + 힌트 "유효하지 않은 이메일" (빨강)
    • └ 중복: 빨강 테두리 + 힌트 "이미 사용 중인 이메일입니다" + "로그인" 링크
    • └ 성공: 초록색 힌트 "중복 확인됨"
8
[Input] 비밀번호 필드
  • 플레이스홀더: "8자 이상, 영문+숫자+특수문자"
  • 요구사항: 최소 8자, 영문 + 숫자 + 특수문자 포함
  • 강도 표시 바:
    • └ 약함 (빨강 #EA5D86): 요구사항 일부만 만족
    • └ 보통 (호박색 #f59e0b): 8~11자, 조건 충족
    • └ 강함 (초록색 #10b981): 12자 이상 또는 조건 초과
  • 강도 라벨: 해당 색상으로 표시
9
[Section] Step 3 - 프로필 설정
  • 구성:
    • └ 프로필 이미지 (선택)
    • └ 닉네임 (필수)
  • 제목: "프로필 설정"
  • 동작: 닉네임만 필수 입력 시 "다음" 활성화
10
[Avatar] 프로필 이미지
  • 크기: 80px x 80px 원형
  • 기본: 회색 배경(#EEF3F8) + 초기 닉네임 첫 글자
  • 오버레이: 카메라 아이콘(파란 배경, 우하단)
  • 동작: 클릭 시 갤러리 열기 (이미지 선택 후 자동 업로드 및 크롭)
  • 지원 포맷: JPEG, PNG (최대 5MB)
11
[Input] 닉네임 필드
  • 길이: 2~12자
  • 허용 문자: 한글, 영문(대소문자), 숫자
  • 중복 확인: 입력 후 500ms debounce 자동 확인
  • 상태:
    • └ 기본: 회색 테두리
    • └ 포커스: 파란색 테두리
    • └ 검증 실패: 빨강 테두리 + 힌트 "2~12자 한글/영문/숫자만 사용 가능"
    • └ 중복: 빨강 테두리 + "이미 사용 중인 닉네임입니다" (빨강)
    • └ 성공: 초록 테두리 + 초록 힌트 "사용 가능한 닉네임입니다" + 체크마크
12
[Section] Step 4 - 관심 카테고리
  • 제목: "관심 카테고리"
  • 설명: "좋아하는 카테고리를 선택해주세요"
  • 구성: 4개 카테고리 카드 (2x2 그리드)
  • 선택: 복수 선택 가능 / 미선택도 허용
  • 버튼:
    • └ "시작하기" (Primary, 전체 항상 활성)
    • └ "나중에 설정할게요" (Ghost Link, 나중에 마이페이지에서 설정 가능)
13
[Card] 카테고리 카드
  • 크기: ~160px x 100px
  • 기본 상태: 흰색 배경 + 1px 회색 테두리
  • 선택 상태: 2px 컬러 테두리 + 연한 배경
  • 카테고리별 색상:
    • └ 애니/만화/소설: Purple #8B5CF6
    • └ 아이돌/아티스트: Pink #EC4899
    • └ 배우/무대: Amber #F59E0B
    • └ 게임: Cyan #06B6D4
  • 구성: 아이콘(위) + 카테고리명(아래) + 체크마크(선택 시만 우상단에 표시)
  • border-radius: 8px
14
[Screen] 가입 완료
  • 구성: 중앙 정렬
    • └ 축하 아이콘(80px 원형, 초록 배경)
    • └ 제목 "{닉네임}님, 환영합니다!" (20px, weight 700)
    • └ 부제 "이제 애픽 월드에서 멋진 굿즈를 교환해보세요" (13px, 회색)
    • └ "시작하기" 버튼 (Primary)
  • 동작: "시작하기" 클릭 → HOME01로 이동
화면 상태 (States)
로딩 중: "다음" / "시작하기" 버튼에 로딩 스피너 표시, 버튼 비활성
이메일 중복: 이메일 필드 빨강 테두리 + 인라인 에러 "이미 사용 중인 이메일입니다" + "로그인" 링크 (AUTH01)
닉네임 중복: 닉네임 필드 빨강 테두리 + 인라인 에러 "이미 사용 중인 닉네임입니다" (빨강)
네트워크 오류: 토스트 메시지 (아래 중앙) "일시적인 오류가 발생했습니다" + 재시도 버튼
이미지 업로드 실패: 토스트 메시지 "이미지 업로드에 실패했습니다" (기본값 유지, 재시도 가능)
