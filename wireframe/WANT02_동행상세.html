1
[Header] 앱 헤더
  • 뒤로가기(← 아이콘) 버튼
  • 가운데 타이틀 "동행 상세"
  • 우측 아이콘 버튼 2개
    • └ Share icon (공유)
    • └ MoreVertical icon (더보기 메뉴: 신고 / 차단 / URL복사)
  • 높이: 50px, 흰 배경, 하단 border 1px #DEE8F2
2
[Image] 이미지 영역
  • 높이: 200px
  • Swipe 슬라이더, 최대 3장
  • 하단 indicator dots (현재 페이지 흰색, 비활성 회색)
  • 우측상단 "N/M" 카운트 배지 (검은 반투명 배경, 흰 글자)
  • 이미지 없는 경우: 카테고리별 gradient 배경 + 아이콘
  • 탭 시: fullscreen 이미지 뷰 (COMM02 참고)
3
[Badge] 상태 뱃지 + D-day
  • 상태별 뱃지 (inline-flex, padding 4px 10px, border-radius 12px)
    • └ 모집중: 녹색 (#16A34A) 배경, 흰 글자
    • └ 마감: 회색 (#45617E) 배경
    • └ 취소: 빨강 (#EA5D86) 배경
  • D-day 뱃지: 빨강 배경, 흰 글자 (예: D-5)
  • 타이틀: 20px 700weight, 색상 #163958
  • 메타 정보: "YYYY.MM.DD · 조회 N" (12px, 색상 #45617E)
4
[Card] 연결 이벤트 카드
  • 조건: 동행이 연결된 이벤트가 있을 때만 표시
  • 배경: #F7FAFD, 테두리 1px #DEE8F2, border-radius 12px
  • 레이아웃: flex, 세로 방향
    • └ 좌: Calendar 아이콘 (18px, 회색 #45617E)
    • └ 중: 이벤트명 (13px 600) + 날짜 (11px 회색)
    • └ 우: ChevronRight 아이콘 (20px, 회색 #8EA2B7)
  • 탭 시: EVNT02 화면으로 이동
5
[Info] 동행 정보 섹션
  • 카테고리 태그 + 작품명 · 캐릭터
    • └ 태그: COMM01 카테고리 색상 (자주 #8B5CF6, 분홍 #EC4899 등)
    • └ 플레이스홀더: "장르/팬덤명 · 캐릭터"
  • 동행 날짜: Calendar 아이콘 + "YYYY.MM.DD(요일) 시간대"
  • 동행 장소: MapPin 아이콘 + 지역명 (Kakao API 주소)
  • 모든 아이콘 좌측 배치, 간격 8px
6
[Map] 카카오 지도 Embed
  • 높이: 150px, border-radius 12px
  • Kakao Map API 마커 표시
  • 마커 탭 시: 카카오맵 앱 또는 웹 열기 (커스텀 URI)
7
[Progress] 모집 현황
  • 제목: "모집 현황" (16px 600)
  • 프로그레스 바: 높이 8px, 배경 #DEE8F2, fill 색상 #2F8EC1
  • 텍스트: "N/M명" (12px 600, 색상 #45617E)
  • 본인 글인 경우: 신청자 목록 표시 (신청자 프로필 + 수락/거절 버튼)
8
[Text] 상세 설명
  • 최대 3줄 표시, 초과 시 "더보기" 링크
  • 글자 크기: 13px, 높이: 1.6
  • "더보기" 탭 시: 전체 텍스트 펼침 (접기 / 펼치기 토글)
9
[Profile] 작성자 정보 박스
  • 배경: #F7FAFD, 테두리 1px #DEE8F2, border-radius 12px, padding 12px
  • 레이아웃: flex
    • └ 좌: 40px 원형 아바타
    • └ 중: 닉네임 (13px 600) + 신뢰도 뱃지 Lv.1~4 (10px, 배경 #F59E0B) + 마지막 접속 시간 (11px 회색)
    • └ 우: ChevronRight (선택사항)
  • 탭 시: USER01 (해당 유저 프로필 페이지)
10
[StickyBar] 하단 고정 액션 바
  • 위치: position sticky, bottom 0
  • 높이: 56px, 흰 배경, 상단 border 1px #DEE8F2
  • 패딩: 12px
  • 4가지 상태
    • └ 타인의 모집중 글: "채팅 문의" (secondary 40%) + "동행 신청" (primary 60%)
    • └ 타인의 마감/취소 글: 전체 disabled gray bar + "마감된 동행입니다" 텍스트
    • └ 이미 신청함: "신청 완료" 비활성 + "신청 취소" 텍스트 버튼
    • └ 본인 글: "끌어올리기" + "수정"(신청 이력 0건일 때) + "마감하기"
    • └ 본인 글(수정 잠금): "수정 불가" 비활성 버튼 + 잠금 안내 배너
11
[Dialog] 동행 신청 다이얼로그
  • COMM02 Dialog/Modal 패턴 참고
  • 레이아웃: center popup (반투명 오버레이 배경 rgba(0,0,0,0.4))
  • 박스: 흰 배경, border-radius 16px, padding 24px
  • 제목: "동행 신청" (16px 700)
  • 메시지 필드 (optional):
    • └ Textarea, placeholder "동행에 대한 한마디를 남겨주세요"
    • └ 최대 100자, 글자 수 카운트 "N/100" 표시
  • 버튼: "취소" (secondary) + "신청하기" (primary)
  • 신청 완료 시: Toast 메시지 + 다이얼로그 닫힘 + 하단바 "신청 완료" 상태 전환
  • 채팅방은 신청 즉시 생성되지 않고, 작성자 수락 시 생성

화면 상태

상태 설명
정상 (모집중) 초대값 로드됨, 모집중 뱃지 표시, 녹색 배경
정상 (마감) 모집 완료, 회색 뱃지, 하단 bar 비활성화
정상 (취소) 작성자가 글 삭제, 빨강 뱃지, 하단 bar 비활성화
이미 신청 사용자가 이미 동행 신청 함, 하단 bar "신청 완료" + "신청 취소"
본인 글 작성자 본인, 하단 bar 3개 버튼 (끌어올리기/수정/마감하기)
본인 글(수정 잠금) 신청 이력 1건 이상이면 "수정 불가" 비활성 + 잠금 안내 문구
로딩 skeleton 로딩 상태, 콘텐츠 영역 회색 배경
에러 네트워크 오류, 재시도 버튼
삭제됨 글이 삭제된 상태, "삭제된 동행입니다" 메시지
비로그인 로그인 하지 않은 사용자, 로그인 유도 팝업
미인증 로그인 + 미인증 상태에서 신청/채팅 문의 시 본인인증 유도 바텀시트
