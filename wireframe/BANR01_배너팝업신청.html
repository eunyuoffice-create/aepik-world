Frame 3: 추가 정보 + 비용 + CTA
1
[Header] 앱 헤더
  • 뒤로가기 버튼 (ChevronLeft SVG)
  • 중앙 정렬 타이틀 "배너/팝업 신청"
  • 우측 예약 공간 (버튼 또는 빈 공간)
2
[Selection] 신청 유형 선택
  • 3가지 신청 유형 카드 (메인 배너 / 피드 배너 / 팝업)
  • 각 카드: 스크린샷 미리보기 + 제목 + 상세 정보 (위치, 슬롯수, 가격)
  • 복수 선택 가능 (멀티 체크박스 UX)
  • 선택 시 파란 테두리 + 체크마크 표시
  • 선택된 유형별로 이후 폼에서 필드 노출 (선택적 로직)
3
[Form] 대상 정보
  • 작품/아이돌 선택 (필수, 드롭다운)
  • 캐릭터/멤버 선택 (필수, 드롭다운)
  • 이벤트 종류 (필수, 칩 그룹): 생일카페 / 데뷔 기념 / 콘서트 / 팬미팅 / 전시 / 기타
  • 최대 1개 선택 권장
4
[Calendar] 슬롯 달력
  • 달력 그리드: 날짜별 잔여 슬롯 상태 시각화
  • ┗ 초록 점: 여유 있음 (6+ 슬롯)
  • ┗ 노랑 점: 반 이상 (3~5 슬롯)
  • ┗ 빨강 점: 거의 매진 (1~2 슬롯)
  • ┗ 회색 배경 "매진": 매진 상태 (비활성)
  • 기간 선택: 시작일~종료일 (이전 프레임과 다른 유형 시 독립 선택)
  • 선택된 기간 요약: "N월 M일 ~ N월 X일 (Y일간)" + 예상 비용
5
[Upload] 이미지 업로드
  • Drag-and-drop 또는 탭 업로드 영역
  • 유형별 권장 크기:
    • └ 메인 배너: 1200×300px
    • └ 피드 배너: 320×240px (예정)
    • └ 팝업: 540×720px (예정)
  • 저작권 가이드라인 (노란 박스):
    • └ O 팬아트, 이름 기반 디자인, 2차 창작
    • └ X 공식 사진, 앨범 이미지, 타인 팬아트 도용
  • 업로드 미리보기 (이미지 선택 시)
6
[Form] 추가 정보
  • 외부 링크 (선택): SNS 등 홍보 링크 입력
  • 관련 이벤트 연결 (선택): 이벤트 캘린더 기능과 연동
  • 배너 텍스트 (선택): 추가 설명 텍스트 (최대 50자)
7
[Info] 비용 안내
  • Phase 1: 배너 신청 무료
  • Phase 1.5 이후: 유형별 일가격 × 선택 일수 = 총 비용
  • 예) 메인 배너 5,000원/일 × 4일 = 20,000원
  • 결제 수단: 카카오페이, 토스페이, 카드
8
[CTA] 신청 + 완료
  • 저작권 동의 체크박스: "저작권 가이드라인을 확인했으며 동의합니다"
  • 신청하기 버튼: full-width 파란색
  • 클릭 시 완료 팝업 노출: "관리자 검수 후 게시됩니다 (24시간 이내)" + "확인" 버튼
  • 신청 상태 흐름:
    • └ 대기: 신청 직후
    • └ 검토 중: 관리자 검수
    • └ 승인: 게시
    • └ 수정 요청: 이미지/정보 재업로드
    • └ 반려: 가이드라인 위반
