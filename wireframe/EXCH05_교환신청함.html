[Header] 헤더
  • 좌측: ← 뒤로가기 버튼 (이전 화면으로 이동)
  • 중앙: "교환 신청함" 제목
  • 우측: 빈 공간
2
[TabBar] 탭 바
  • 2개 탭: 받은 제안 (N) | 보낸 제안 (N)
  • 각 탭에 뱃지: 대기중인 제안 수만 표시
  • 활성 탭: 하단 Blue 밑줄 + 텍스트 Blue
  • 비활성 탭: Gray 텍스트
  • 탭 전환: URL 변경 (/exchange/proposals ↔ /exchange/my-proposals)
3
[CardGroup] 내 교환글 요약
  • 그룹 헤더 역할: 내 교환글을 그룹핑
  • 구성:
    • 이미지 (60×60px, 정사각형)
    • 교환글 상품명 (1줄)
    • "받은 제안 N건" (Gray)
  • 탭 시: 해당 교환글 상세(EXCH02)로 이동
  • 받은 제안이 있는 글만 표시, 내림차순 (최신순)
4
[ProposalCard] 받은 제안 카드
  • 제안자 정보 (우상단 상태 뱃지):
    • 32×32px 아바타 + 닉네임 + 신뢰등급(Lv.N) + 평점(★ N.N)
    • 상태 뱃지: 대기중(Amber) / 수락됨(Green) / 거절됨(Gray) / 취소됨(Gray) / 만료(Gray)
  • 제안 굿즈 (좌측 60×60px 이미지):
    • 상품명 + 작품명 · 캐릭터명
    • 상태등급 뱃지 (S/A/B/C, 배경 컬러 구분)
  • 제안 메타 요약:
    • 제안 방식: 내 교환글 매칭 / 빠른 제안 등록
    • 거래 구조: 1:1 / 1:N / N:1 + 제공/희망 수량
    • 차액: 없음 / +금액 / -금액
    • 오늘 직교환 ON 시 이벤트명/시간대/대략 위치 표시
  • 첨부 메시지 (Gray 말풍선, 있는 경우만)
  • 제안 시간: "1시간 전" / "3일 전"
  • 액션 버튼 (대기중 상태일 때만):
    • [수락] Blue 버튼
    • [거절] Gray 아웃라인 버튼
    • [] 채팅 아이콘 버튼
  • 수락/거절됨/취소됨/만료 상태: 버튼 미표시
  • 받은 제안 1건 이상 생성 시 대상 교환글의 proposal_history_count가 1 이상이 되어 EXCH02/EXCH03 수정 잠금 조건 충족
5
[SentProposalCard] 보낸 제안 카드
  • 좌측: 상대방 교환글 요약
    • 이미지 (정사각형) + 상품명
    • 탭 시: 해당 교환글(EXCH02)로 이동
  • 우측: 내가 제안한 굿즈
    • 이미지 (정사각형) + 상품명
    • 상태 뱃지 + 제안 시간
    • 제안 메타: 방식/거래 구조/수량/차액/오늘 직교환 여부
    • 액션 버튼: 상태별 다름
      • 대기중: [제안 취소] 버튼
      • 수락됨: [채팅으로 이동] Blue 버튼
      • 거절됨/취소됨/만료: 버튼 미표시
6
[EmptyState] 빈 상태
  • 받은 제안 없음: "아직 받은 교환 제안이 없어요" + 일러스트
  • 보낸 제안 없음: "아직 보낸 교환 제안이 없어요" + [교환글 둘러보기] CTA 버튼
  • 버튼 탭 시: EXCH01(교환목록)으로 이동
화면 상태
  • 로딩 상태: 제안 카드 스켈레톤 UI (회색 박스 펄스 애니메이션)
  • 빈 상태: Empty State 메시지 + CTA 버튼
  • 대기 제안 > 수락/거절/취소/만료 정렬: 받은 제안은 상태 우선순위 반영
  • 상태 이력 표시 정책: 받은/보낸 제안 모두 pending/accepted/rejected/cancelled/expired 상태 유지 표시
  • 제안 메타 표시 상태: 제안 방식/거래 구조/수량/차액이 카드 요약으로 노출
  • 오늘 직교환 상태: 이벤트명·시간대·대략 위치 요약 노출, 시간 만료 시 일반 직거래 제안으로 전환
  • 수정 잠금 연동 상태: 받은 제안이 1건 이상인 내 글은 EXCH02에서 [수정] 비활성 + 안내 문구 노출
  • 에러 상태: "데이터를 불러올 수 없습니다" + 재시도 버튼
