화면 목적

공동구매(공구) 상세 정보를 확인하고 참여를 진행하는 화면입니다.

  • 일반 공구: 수량 선택 후 참여
  • 세트 공구: 캐릭터 선택 후 선점

진입 경로

  • 공구 목록 > 카드 탭
  • 홈 > 인기 공구 카드 탭
  • 알림 > 공구 알림 탭
  • 채팅 > 공구 링크 탭

이용자

PC / Mobile (주로 Mobile)

라우트

/groupbuy/[id]

① Header

뒤로가기 + 타이틀 + 공유/신고 아이콘

② 이미지 갤러리

상품 이미지 좌우 스와이프, 페이지 인디케이터

③ 공구 정보

  • 태그: [공구]/[세트공구] + 카테고리
  • 상품명 (bold, large)
  • 가격: ₩N,000 / 1인 또는 /캐릭터
  • 배송비 포함 여부

④ 진행자 정보

프로필 이미지 + 닉네임 + 신뢰배지(Lv) + 거래횟수

탭시 진행자 프로필로 이동

⑤ 참여 현황

일반 공구: 프로그레스 바 + N/M명 + 최소인원 + 마감일 + 배송예정

세트 공구: 캐릭터별 선점 상태 리스트

⑥ 상세 설명

진행자가 작성한 공구 상세 내용

⑦ 참여자 목록

프로필 아이콘 최대 5개 + "+N명 더보기"

⑧ 하단 CTA

찜 버튼 + 참여하기/캐릭터 선점하기 버튼

API 호출

GET /api/groupbuy/[id]

응답: 공구 전체 정보 + 진행자 + 참여자 + 캐릭터 슬롯(세트 타입)

참여 흐름

  • 참여하기 버튼 클릭 → 바텀시트 오픈
  • 수량/캐릭터 선택 → 금액 계산
  • 참여 확정 → 참여 목록에 추가
  • ※ 결제는 진행자와 직접 정산

상태 관리

  • 마감 전: 참여 가능
  • 마감 후: 버튼 비활성 "마감된 공구입니다"
  • 이미 참여: "참여 취소" 버튼
  • 비로그인: 로그인 유도

세트 공구 특징

  • 캐릭터별 제한 수량 설정
  • 선착순 선점 (동시성 제어)
  • 마감 캐릭터는 회색 처리 및 클릭 불가
  • 선택한 캐릭터의 합계로 결제금액 계산

공구 상태별 표시

상태 배지 CTA 버튼
모집중 Green [참여하기] 활성
마감/진행 결정 대기 Amber 비활성 "진행자가 진행 여부를 결정 중입니다"
진행 확정 Blue 비활성 "공구가 확정되었습니다"
배송중 Blue 비활성 "배송이 진행 중입니다"
완료 Gray [후기 작성]
무산 Red Red 배너 + 사유 + 환불 안내 + [진행자에게 채팅하기]

자동 무산 타이머

마감일 도달 후 다음 흐름으로 진행됩니다:

  • D+0 (마감일): 진행 결정 대기 상태 진입 → 진행자에게 결정 알림
  • D+1: 진행자 리마인더 알림 (결정 요청)
  • D+3: 참여자 "진행자 미응답" 알림 표시
  • D+5: 자동 무산 처리 → Red 배너 + 무산 사유 자동 입력 (예: "진행자 미응답")
