[Screen] 이메일 로그인 모드

  • 보조 링크: "비밀번호를 잊으셨나요?" + "회원가입" (중앙 정렬)
  • 에러 메시지 영역 (인라인, 빨강 #EF4444)

[Input] 이메일/비밀번호 필드
  • 높이: 40px, border-radius: 6px, padding: 10px 12px
  • 테두리: 1px #ddd (기본), 1px #2563EB (포커스)
  • 포커스 시: box-shadow 0 0 0 3px rgba(37,99,235,0.1) 추가
  • 에러 시: 테두리 1px #EF4444
  • 이메일 필드:
    • 플레이스홀더: "user@example.com"
    • 유효성: /^[^\s@]+@[^\s@]+\.[^\s@]+$/ (기본 형식 검증)
  • 비밀번호 필드:
    • 플레이스홀더: "비밀번호를 입력하세요"
    • 타입: password (마스킹)
    • 우측 눈 아이콘: 토글 시 평문 표시

OAuth 에러에러 팝업 + 이메일 로그인 자동 제안

정지 계정로그인 불가, "계정 정지" 안내 + 사유 + 해제 예정일
비밀번호 재설정 링크이메일 발송, 유효기간 1시간
비로그인 세션로컬 스토리지 쿠키, 만료 없음 (브라우저 닫기 시 유지)
화면 상태
  • 로딩 상태: 버튼 텍스트 "로그인 중..." + 좌측 스피너, 버튼 비활성화
  • 로그인 실패: 인라인 에러 메시지 (빨강 #EF4444) "이메일 또는 비밀번호가 일치하지 않습니다"
  • 계정 잠금: "로그인 시도 초과. 30분 후 다시 시도해주세요" 에러 팝업
  • OAuth 에러: 에러 팝업 + "다시 시도" 또는 "이메일 로그인" 버튼
  • 휴면 계정: "휴면 계정입니다. 휴면을 해제하시겠습니까?" 팝업 → 비밀번호 확인
  • 정지 계정: "이 계정은 신고로 인해 2026-03-20까지 정지되었습니다" (사유+날짜 포함)
  • 비밀번호 찾기 성공: "비밀번호 재설정 링크가 이메일로 발송되었습니다. 이메일을 확인해주세요." (Green 배경)
