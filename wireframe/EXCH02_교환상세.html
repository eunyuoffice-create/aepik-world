EXCH02 교환 상세
개별 교환글의 상세 정보 확인 + 찜/채팅 + EXCH04 교환 제안 진입 + 본인 글 수정 잠금 정책
1
[Header] 상세 헤더
  • 좌측: ← 뒤로가기 (이전 페이지)
  • 우측: 공유(↗) + 더보기(⋯)
    • 더보기 메뉴: 신고하기 / 차단하기 / URL 복사
  • 스크롤 시 상단 고정
2
[ImageSlider] 이미지 슬라이더
  • 상품 이미지 최대 5장 (1:1 비율)
  • 좌우 스와이프로 이동
  • 인디케이터: 하단 중앙 ● ○ ○ + 우하단 "1/3" 카운터
  • 이미지 탭 시: 전체 화면 뷰어 (핀치 줌 지원)
  • 이미지 없는 경우: 회색 플레이스홀더 1장
3
[ProductInfo] 태그 + 상품 정보
  • 태그: [거래유형] + [카테고리]
    • 거래유형: 교환 (#FFF7ED) / 교환+차액 (#FFF7ED)
    • 카테고리: 애니·만화·소설 / 아이돌·아티스트 / 배우·무대 / 게임
  • 상품명: 16px, Bold
  • 작품 · 캐릭터: Blue 텍스트 (탭 시 해당 작품 필터 목록으로 이동)
  • 등록일 · 조회수: Gray 텍스트
4
[DetailBox] 상세 정보 박스
  • 회색 배경 카드에 레이블-값 쌍으로 표시
  • 데이터 필드:
    • 상태: S급/A급/B급/C급 + 설명 (미개봉/거의새것/사용감있음/하자있음)
    • 굿즈 타입: 아크릴/포토카드/캔뱃지/피규어 등
    • 보유 수량: N개
    • 희망 교환: Blue 하이라이트 텍스트 (핵심 정보)
    • 차액: ±금액 또는 "차액 없음"
    • 거래 방식: 택배/직거래/둘 다
5
[Map] 직거래 지도 (조건부)
  • 거래 방식이 "직거래" 또는 "둘 다"일 때만 표시
  • 카카오맵 API 임베드 (120px 높이)
  • 거래 희망 위치 핀 마커
  • 하단: 주소 텍스트
  • 지도 탭 시: 카카오맵 앱으로 열기
6
[TextBox] 추가 조건
  • 등록자가 작성한 자유 텍스트
  • 3줄 초과 시 "더보기" 버튼으로 접기/펼치기
  • 작성 안 한 경우 섹션 자체 미표시
7
[SellerCard] 판매자 정보
  • 프로필 이미지 (40px 원형) + 닉네임
  • 신뢰 등급: TrustBadge (Lv.1~4 + 아이콘)
  • ★ 평점 · 거래 횟수 · 최근 접속 시간
  • 전체 영역 탭 시: 타인 프로필(USER03) 이동
8
[HScroll] 관련 교환글
  • 같은 작품/캐릭터의 다른 교환글
  • 가로 스크롤 (110px 미니 카드)
  • 카드: 이미지 + 상품명 + 평점/거래수
  • 최대 6개 표시
  • 카드 탭 시: 해당 교환 상세(EXCH02) 이동
  • 관련 교환글 없으면 섹션 미표시
9
[BottomCTA] 하단 고정 CTA
  • 하단 고정 (sticky bottom)
  • 구성:
    • 찜: 토글 (빈 하트 ↔ 꽉 찬 하트), 찜 수 표시
    • 채팅하기: 판매자와 1:1 채팅 (CHAT02)
    • 교환 제안: EXCH04(교환 제안 전용 페이지) 이동 → 방식 선택(내 글 매칭/빠른 제안 등록) → 조건 입력 → 제안 발송
  • 비로그인 시: CTA 탭 → 로그인 유도 바텀시트
  • 본인 글일 경우 CTA가 변경됨:
    • [수정] — 받은 제안 이력 0건일 때만 EXCH03(수정 모드) 이동
    • [끌어올리기] — 목록 상단 노출 갱신
    • [삭제] — 삭제 확인 다이얼로그
  • 받은 제안 이력 1건 이상(상태 무관)이면 [수정] 비활성 + 안내 문구 노출
    • "받은 제안 이력이 있어 수정할 수 없어요. 제안이 한 번이라도 도착한 글은 수정할 수 없어요."
  • 교환 완료 상태: 전체 CTA 비활성(회색)
10
[PageFlow] 교환 제안 페이지 진입 (EXCH04)
  • CTA 탭 시 EXCH04 교환 제안 전용 페이지로 이동
  • 구성:
    • 헤더: "교환 제안하기" 제목 + X 닫기 버튼 (EXCH04 내부)
    • 1단계: 제안 방식 선택
      • 내 교환글 매칭 (`proposal_mode=listing`)
      • 빠른 제안 등록 (`proposal_mode=manual`)
    • 매칭 제안: 내 교환글 선택(라디오), 이미 제안한 글은 "제안 중" 뱃지 + 비활성
    • 빠른 제안: 사진 1~3장, 굿즈명, 상태(S/A/B/C), 수량 직접 입력
    • 다수거래/차액: 거래 구조(`1:1`/`1:N`/`N:1`) + 제공/희망 수량 + 차액(+/-) 입력
    • 오늘 직교환: 이벤트명/시간대/대략 위치 입력 (당일만 가능)
    • 메시지 입력: 선택사항, 거래 조건 등 추가 정보 (100자)
    • CTA: "교환 제안 보내기" (Blue, full-width)
  • 와이어프레임 상 프레임 2/3은 EXCH04 입력 화면 미리보기
  • 실제 제출 성공 후 EXCH05(교환 신청함)에 받은/보낸 제안이 반영됨
11
[ProposalPanel] 받은 제안 패널 (본인 글)
  • 본인이 작성한 교환글에서만 표시
  • 헤더:
    • "받은 제안" 레이블 + 숫자 뱃지 (Blue background)
    • 최대 2건 카드 표시, 초과 시 "전체보기 →" 링크
  • 제안 카드 구성:
    • 아바타(24px 원형) + 닉네임 + Lv.N + 시간
    • 제안 굿즈: 이미지(48px) + 상품명 + 상태 뱃지
    • [수락] [거절] 액션 버튼 (대기중 상태에서만, 등록자 권한)
    • 거절/취소/만료 상태는 상태 배지만 노출되고 액션은 비활성
  • "전체보기 →" 탭 시: EXCH05(교환신청함) 이동
12
[OwnerCTA] 본인 글 CTA 바
  • 본인이 작성한 교환글에서만 표시 (받은 제안 패널 하단에서 함께 노출)
  • 구성: 3개 버튼 (flex 1/3 width)
    • [수정] — 받은 제안 이력 0건일 때만 EXCH03(수정 모드) 이동
    • [끌어올리기] — 1일 1회 무료, 추가는 유료
    • [삭제] — 삭제 확인 다이얼로그
  • 받은 제안 이력 1건 이상(상태 무관)이면 [수정] 비활성 + 잠금 안내 문구 노출
  • 하단 고정 (sticky bottom)
