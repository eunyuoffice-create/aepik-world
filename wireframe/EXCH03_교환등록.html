EXCH03 교환글 등록/수정
3단계 폼으로 보유 굿즈 + 희망 교환 + 거래 방식을 등록/수정
Step 1: 보유 굿즈
1
[Header] 등록 헤더
  • 좌측: ← 뒤로가기 (작성 중 이탈 시 "작성 중인 내용이 있습니다. 나가시겠습니까?" 확인 다이얼로그)
  • 중앙: "교환글 등록" (수정 모드: "교환글 수정")
2
[StepIndicator] 스텝 인디케이터
  • 3단계: ① 보유 굿즈 → ② 원하는 굿즈 → ③ 거래 방식
  • 현재 스텝: Blue 원 + Bold 텍스트
  • 완료 스텝: Green 체크(✓), 탭으로 되돌아가기 가능
  • 미진행 스텝: Gray (탭 불가)
3
[PhotoUpload] 사진 업로드
  • 최대 5장 (JPG/PNG, 10MB 이하)
  • 첫 번째 이미지 = 대표 이미지 (뱃지 표시)
  • 드래그로 순서 변경 가능
  • + 버튼: 카메라/앨범 선택 시트
  • ✕ 버튼: 이미지 삭제
  • 업로드 시 Sharp로 max 1200px 리사이즈
4
[SearchInput] 작품 선택 *
  • 검색 인풋 + 자동완성 드롭다운 (디바운스 300ms)
  • 작품 DB에서 검색 (이름 + alias 매칭)
  • 검색 결과 없을 시: "직접 추가" → 관리자 승인 후 등록
  • 필수 항목
5
[Dropdown] 캐릭터 선택 *
  • 작품 선택 후 활성화 (작품 미선택 시 비활성)
  • 해당 작품의 등록된 캐릭터 목록
  • "직접 추가" 가능 → 관리자 승인 후 등록
  • 필수 항목
6
[Dropdown] 굿즈 타입 *
  • 포토카드 / 아크릴 / 캔뱃지 / 피규어 / 인형 / 엽서·포스터 / 의류 / 기타
  • 필수 항목
7
[RadioGroup + Stepper] 상품 상태 + 수량 *
  • 상태 라디오: S급(미개봉) / A급(거의 새 것) / B급(사용감) / C급(하자)
    • C급 선택 시: 하자 설명 텍스트 필드 추가 (필수)
  • 수량: 숫자 스테퍼 (−/+ 버튼, 기본 1, 최대 99)
  • 둘 다 필수 항목
8
[TextInput] 상품명 *
  • 최대 50자 (글자 수 카운터 표시)
  • 플레이스홀더: "예: 하이큐 히나타 아크릴 스탠드"
  • [다음] 탭 시 필수 항목 검증 → 미입력 시 빨간 테두리 + 에러 텍스트
Step 2: 원하는 굿즈
9
[SearchInput + Dropdown] 원하는 작품/캐릭터
  • "아무거나 괜찮아요" 체크 시: 작품/캐릭터 필드 비활성
  • 작품: Step 1과 동일 검색 UI
  • 캐릭터: 작품 선택 후 드롭다운
  • 선택 항목 (미입력 시 "희망 교환 미지정" 표시)
10
[Dropdown] 원하는 굿즈 타입
  • Step 1과 동일 옵션
  • "아무 타입 괜찮아요" 체크 가능
  • 선택 항목
11
[Checkbox + Toggle + Input] 차액 설정
  • 체크박스: "차액 협의 가능"
  • 체크 시 활성화:
    • 방향 토글: "+받을게요" / "−줄게요"
    • 금액 인풋: 숫자만, 1,000원 단위 포맷
  • 선택 항목
Step 3: 거래 방식
12
[RadioGroup] 거래 방법 *
  • 택배 / 직거래 / 둘 다 가능
  • 필수 항목
13
[MapPicker] 직거래 위치 (조건부)
  • "직거래" 또는 "둘 다" 선택 시에만 표시
  • 주소 검색 인풋 + 카카오맵 핀 선택
  • 선택 시 주소 텍스트 표시
  • 직거래 선택 시 필수 항목
14
[Textarea] 추가 조건
  • 최대 500자 (글자 수 카운터)
  • 플레이스홀더: "추가 교환 조건이나 참고사항을 적어주세요"
  • 선택 항목
15
[Button] 등록하기 버튼
  • Step 3의 필수 항목 모두 입력 시 활성화 (Blue)
  • 미입력 시: 비활성(Gray) + 미입력 필드 스크롤 이동
  • 탭 시: API 호출 → 로딩 스피너 → 성공 → 교환 상세(EXCH02) 이동
  • 수정 모드: "수정하기" 텍스트
  • 에러 시: 토스트 메시지 "등록에 실패했습니다. 다시 시도해주세요."
16
[EditMode] 수정 모드 전용 프레임
  • 헤더 타이틀: "교환글 수정"
  • 기존 입력값 프리필(이미지/상품명/조건/거래 방식)
  • 변경 항목은 `변경됨` 표시로 구분 가능
  • 하단 CTA: `수정하기` (받은 제안 이력 0건일 때만 활성)
  • 받은 제안 이력 1건 이상(상태 무관)이면 진입/저장 차단 + 정책 안내 문구 노출
  • 성공 시 EXCH02 상세로 복귀 + 최신 수정일 반영
화면 상태
등록 모드: 빈 폼 + 제목 "교환글 등록" + 버튼 "교환글 등록하기"
수정 모드: 기존 데이터 프리필, 제목 "교환글 수정"
수정 잠금 상태: 받은 제안 이력 1건 이상(상태 무관)이면 진입/저장 차단 + 안내 문구 노출
필드 에러: 빨간 테두리 + 하단 에러 텍스트
제출 중: 버튼 로딩 스피너 + 폼 비활성
이미지 업로드 중: 썸네일 위 프로그레스 오버레이
이탈 시도: "작성 중인 내용이 있습니다" 확인 다이얼로그
임시 저장: 각 스텝 완료 시 localStorage 자동 저장, 재진입 시 복원 안내
