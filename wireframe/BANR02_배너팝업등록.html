BANR02 배너/팝업 등록
선택한 유형에 맞는 통이미지 업로드와 부가 정보를 입력하여 배너/팝업 신청
1
[Header] 앱 헤더
  • 좌측: `ChevronLeft` 뒤로가기 (BANR01로)
  • 중앙: 유형별 타이틀 ("메인 배너 등록" / "피드 배너 등록" / "팝업 등록")
  • 우측: 없음
2
[Info] 유형 정보 카드 (읽기 전용)
  • BANR01에서 전달받은 유형 요약 (변경 불가)
  • 유형 아이콘 + 유형명 + 노출 위치 + 이미지 규격
  • 잔여 슬롯 표시 ("3슬롯 남음", Green)
  • 가격 안내 (Phase 1.5~)
  • 유형 변경하려면 뒤로가기로 BANR01 복귀 필요
3
[Form] 대상 정보
  • 작품/아이돌 선택 (필수, 드롭다운): 이벤트 캘린더와 동일 DB
  • 캐릭터/멤버 선택 (필수, 드롭다운): 작품 선택 시 동적 로드
  • 이벤트 종류 (필수, 칩 그룹): 생일카페/데뷔 기념/콘서트/팬미팅/전시/기타
  • 단일 선택만 허용
4
[Calendar] 게시 기간 선택
  • 슬롯 달력: 날짜별 잔여 슬롯 상태 시각화
    • 초록 점: 여유 (>50% 잔여)
    • 노랑 점: 보통 (25~50% 잔여)
    • 빨강 점: 거의 매진 (1~2 슬롯)
    • 회색 비활성: 매진 (0슬롯, 선택 불가)
  • 시작일~종료일 범위 선택 (연속 날짜만 가능)
  • 선택 기간 요약 + 예상 비용 (Phase 1.5~)
  • Phase 1: "무료" 표시
5
[Upload] 이미지 업로드
  • 통이미지 업로드: Drag-and-drop 또는 탭
  • 유형별 권장 크기:
    • 메인 배너: 1200x300px
    • 피드 배너: 카드너비x180px
    • 팝업: 320x400px
  • 파일 포맷: JPG, PNG, WebP (최대 5MB)
  • 업로드 시 실제 비율 미리보기 표시
  • 저작권 가이드라인 (노란 박스):
    • O 팬아트, 이름 기반 디자인, 2차 창작
    • X 공식 사진, 앨범 이미지, 타인 팬아트 도용
  • 통이미지 정책: 텍스트 오버레이 없이 이미지 1장으로만 구성
6
[Form] 부가 정보
  • 이미지 설명 (필수): img alt 속성에 들어갈 텍스트 (최대 100자)
    • 안내: "화면에 노출되지 않으며, 접근성(alt)을 위해 사용됩니다"
    • 이미지에 텍스트가 들어가지 않으므로 설명 용도로만 사용
  • 외부 링크 (선택): SNS 등 홍보 URL
  • 관련 이벤트 연결 (선택): 이벤트 캘린더 연동
    • 내가 등록한 이벤트 목록에서 선택
    • 미연결 시 배너 탭해도 이미지만 풀뷰
7
[Info] 비용 안내
  • Phase 1: 무료 (초록 배경 안내)
  • Phase 1.5~: 유형별 일가격 x 일수 = 총 비용
  • 결제 수단: 카카오페이, 토스페이, 카드 (PortOne)
8
[CTA] 동의 및 신청
  • 저작권 동의 체크박스 (필수)
  • [신청하기] Primary 버튼 (full-width)
  • 필수 미입력 시: 비활성 + 미입력 필드 하이라이트
  • 신청 시: 확인 바텀시트 -> 요약 + [신청] / [취소]
  • 완료 모달:
    • "신청이 완료되었습니다" + "관리자 검수 후 게시 (24시간 이내)"
    • [내 신청 관리] -> USER02
    • [확인] -> BANR01 복귀
  • 신청 상태 흐름: 대기중 -> 검수중 -> 승인/수정요청/반려
